## 1. Overview

### 1.1 배경
프로젝트 템플릿 기능이 **상세 화면(간트)** 에 흩어져 있었고, "템플릿 적용"은 사실상 새 프로젝트를 만드는데도 기존 프로젝트 상세에 버튼이 있어 혼란스러웠다. 또한:
- 템플릿으로 새 프로젝트 생성 시 자원 배정에서 **500 에러**(XOR 제약 위반) — 자원-모델-분리 때 누락된 잔재 버그.
- 템플릿을 **수정(카테고리 변경)·삭제**할 관리 화면이 프론트에 없었다(백엔드 PATCH/DELETE는 존재).

### 1.2 목표
템플릿 관련 기능을 **프로젝트 목록 화면으로 일원화**:
1. **템플릿 저장** — 목록 툴바 버튼 → 원본 프로젝트 선택 후 템플릿 저장
2. **템플릿 적용** — "새 프로젝트" 모달에 "템플릿으로 만들기" 옵션 통합 (TemplateWizard)
3. **템플릿 관리** — 목록 툴바 버튼 → 모달에서 이름/카테고리 수정·추천·삭제
4. **버그 수정** — 템플릿 인스턴스화 시 polymorphic FK 채워 XOR 제약 충족
5. **상세 화면 정리** — 중복 템플릿 버튼/모달 제거

### 1.3 비목표
- 템플릿 버전관리/공유범위 고도화
- 태스크 단위 부분 템플릿화 UI

## 2. 성공 기준
1. 목록에서 저장/적용/관리 모두 가능
2. 템플릿으로 프로젝트 생성이 500 없이 동작(자원 배정 포함)
3. 상세 화면에 템플릿 버튼 없음(중복 제거)
