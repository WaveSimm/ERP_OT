## 1. 개요 (Overview)

ERP/OT 플랫폼에서 **개별 실무자(Operator/Manager)가 매일 접속하는 개인 업무 허브**.
지휘센터 대시보드가 팀장/임원의 전체 현황 파악 도구라면,
개인 대시보드는 **"오늘 나는 무엇을 해야 하는가"** 에 집중한 실무자 중심 뷰다.

두 가지 핵심 축으로 구성된다:
- **개인 업무 현황**: 내가 배정된 세그먼트 기반 칸반·캘린더·알림
- **경량 근태 관리**: 출퇴근 기록·연차 잔여·OT 집계 (결재 워크플로 포함)

### 배경 (Background)

현재 설계된 시스템에서 Operator는 프로젝트 상세 화면에 들어가야만 자신의 태스크를
확인할 수 있다. 수십 개 프로젝트에 분산 배정된 실무자 입장에서는 **어느 프로젝트,
어느 세그먼트부터 집중해야 할지** 한눈에 알기 어렵다.
근태 역시 별도 시스템(종이, 엑셀, 별도 SW)으로 관리되어 프로젝트 투입 현황과 연동이 없다.

## 2. 문제 정의 (Problem Statement)

| 문제 | 영향 |
|------|------|
| 내 오늘 할 일을 확인하려면 프로젝트별로 각각 들어가야 함 | 업무 파악에 시간 낭비 |
| 여러 프로젝트에 배정됐을 때 우선순위를 알 수 없음 | 중요 마감 누락 위험 |
| @멘션·마감 임박 알림이 흩어져 있어 놓치기 쉬움 | 협업 단절 |
| 진행률 업데이트를 오랫동안 안 해도 본인이 인지 못함 | 팀장이 Stale 이슈 감지 → 사후 알림 |
| 출퇴근을 별도 시스템에 기록 → 프로젝트 실투입 시간과 불일치 | 자원 계획 오차 |
| 연차 잔여·OT 현황을 HR팀에 물어봐야 알 수 있음 | 휴가 계획 어려움 |
| 휴가·OT 신청을 메일/구두로 → 이력 관리 안 됨 | 분쟁 발생 시 증빙 불가 |
| 내 이번 달 근무 패턴(조기출근, 결근 등)을 한눈에 볼 수 없음 | 자기 관리 어려움 |

## 3. 목표 및 성공 지표 (Goals & Success Metrics)

### 목표
- 개인이 매일 아침 대시보드 1개로 당일 업무를 즉시 파악
- 진행률 업데이트 누락 0건 (본인이 먼저 알림 받음)
- 근태 기록을 출퇴근 버튼 하나로 처리 (별도 시스템 이탈 없음)

### 성공 지표 (KPI)

| 지표 | 현재 | 목표 |
|------|------|------|
| 개인 할 일 파악 시간 | 15~30분 | < 1분 |
| 진행률 Stale 경고 발생률 (3일 미업데이트) | ~40% | < 5% |
| 근태 기록 누락률 | ~30% (별도 시스템 미기록) | < 1% |
| 휴가·OT 신청 처리 시간 | 1~2일 (메일 대기) | < 4시간 |

## 4. 사용자 및 역할 (Users & Roles)

| 역할 | 사용 방식 |
|------|-----------|
| **Operator** | 주 사용자. 칸반 진행률 업데이트, 출퇴근 기록, 휴가 신청 |
| **Manager** | 칸반 + 팀원 근태 요약 + 휴가·OT 승인 (관리자 역할) |
| **Admin** | 근태 설정 관리 (연차 기준, OT 단가, 공휴일 등) |

## 5. 사용자 스토리 (User Stories)

### A. 개인 업무 현황

**US-P01**: Operator로서 대시보드에 접속하면 **내가 배정된 모든 세그먼트**가 칸반 형태로 보인다.
- 컬럼: `이번 주 예정` / `진행 중` / `완료` / `마감 임박(D-3 이내)`
- 카드: 프로젝트명 > 태스크명 > 세그먼트명, 기간, 내 배정률, 진행률

**US-P02**: Operator로서 칸반 카드에서 **진행률을 직접 업데이트**하고 변경 사유를 입력한다.
프로젝트 상세 화면으로 이동하지 않아도 된다.

**US-P03**: Operator로서 **주간 캘린더 뷰**로 전환하면 이번 주 내 배정 세그먼트가 날짜별로 표시된다.
> 월~일, 각 날짜 칸에 해당일에 활성인 세그먼트 바(bar) 표시

**US-P04**: Operator로서 **참여 프로젝트 목록**을 확인한다. 각 프로젝트의 내 배정 세그먼트 수,
내 평균 진행률, 프로젝트 전체 RAG 상태가 함께 표시된다.

**US-P05**: 시스템이 **진행률 미업데이트 세그먼트(3일 이상)**를 자동 감지하여
대시보드 상단에 경고 배너와 해당 세그먼트 목록을 표시한다.
> "배관 설치 > 지상 배관: 5일째 업데이트 없음 [지금 업데이트]"

**US-P06**: Operator로서 **@멘션 알림**, **세그먼트 마감 임박 알림**, **이슈 할당 알림**을
대시보드 알림 패널 한 곳에서 확인하고 읽음 처리한다.

**US-P07**: Manager로서 **내 팀원 업무 현황 요약**을 볼 수 있다.
- 팀원별: 이번 주 활성 세그먼트 수, 평균 진행률, 미업데이트 건수
- 클릭 시 해당 팀원의 칸반으로 이동 (읽기 전용)

---

### B. 경량 근태 관리

**US-P08**: Operator로서 **출근 버튼 한 번**으로 오늘 출근을 기록한다. 퇴근도 동일.
- 기록 항목: 출근시각, 퇴근시각, 근무 유형 (정상/재택/외근)
- 외출/복귀도 별도 버튼으로 기록 (중간 외출 시간 자동 계산)

**US-P09**: Operator로서 **이번 달 근태 달력**을 본다.
- 날짜별: 출근 ✓ / 결근 ✗ / 휴가 🏖 / OT ⏰
- 정상 근무 시간(기준: 8h) 대비 실근무 시간 표시

**US-P10**: Operator로서 **연차 현황**을 확인한다.
- 연도별 부여 일수, 사용 일수, 잔여 일수
- 예정 휴가(승인된 것) 포함한 잔여 일수 표시

**US-P11**: Operator로서 **휴가를 신청**한다.
- 유형 선택: 연차 / 반차(오전/오후) / 병가 / 특별휴가
- 기간 선택 + 사유 입력
- 신청 즉시 담당 Manager에게 알림 발송
- 잔여 연차 초과 시 경고

**US-P12**: Operator로서 **OT(초과근무)를 신청**한다.
- 날짜, 예상 OT 시간, 관련 프로젝트 연결 (선택)
- Manager 사전 승인 후 실제 OT 기록 활성화
- OT 완료 후 실제 시간 입력

**US-P13**: Manager로서 팀원의 **휴가·OT 신청을 승인/반려**한다.
- 신청 목록: 이름, 유형, 기간, 사유
- 승인 시 → 신청자에게 알림 + 달력에 반영
- 반려 시 → 사유 입력 필수

**US-P14**: Manager로서 **팀원 근태 요약**을 확인한다.
- 이번 달: 팀원별 출근일, 지각 횟수, 누적 OT 시간, 잔여 연차
- 근태 이상(무단 결근, 지각 3회 이상) 자동 강조

**US-P15**: Admin으로서 **근태 기준을 설정**한다.
- 정상 근무 시간: 기본 8h/일 (조정 가능)
- 출근 인정 시각 범위: ex) 08:30~09:30
- 연차 부여 기준: 입사일 기준 / 회계연도 기준 선택
- OT 가산율: 150% / 200% (야간·휴일) — 향후 급여 연동용 저장
- 공휴일 설정 (수동 입력 또는 공공 API 연동)

**US-P16**: Operator로서 **OT와 프로젝트 세그먼트를 연결**하면
해당 세그먼트의 실투입 시간이 OT 시간만큼 자동으로 증가한다.
> 경량 연동: 별도 집계 없이 OT 레코드에 projectId + segmentId 메타 저장 → 향후 집계

---

### C. 알림 통합

**US-P17**: 모든 알림(댓글 @멘션, 마감 임박, 진행률 Stale, 휴가 승인 결과, OT 승인 결과)이
**단일 알림 센터**에서 관리된다.
- 읽음/안 읽음 구분, 우선순위별 정렬
- 알림 클릭 시 관련 화면으로 즉시 이동

type KanbanColumn = 'UPCOMING' | 'IN_PROGRESS' | 'DUE_SOON' | 'DONE';

function classifySegment(seg: TaskSegment, today: Date): KanbanColumn {
  if (seg.progressPercent >= 100) return 'DONE';
  const daysUntilEnd = dateDiff(today, seg.endDate);
  if (daysUntilEnd <= 3 && daysUntilEnd >= 0) return 'DUE_SOON';
  if (seg.startDate <= today && seg.endDate >= today) return 'IN_PROGRESS';
  return 'UPCOMING';
}
