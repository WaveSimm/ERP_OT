## 1. 개요 (Overview)

기존 자연어검색(임베딩 기반)에 **키워드 매칭 신호**를 결합한 하이브리드 검색을 도입한다. 영문 약어·고유명사·짧은 키워드에 대한 임베딩 모델의 약점을 보완하고, 사용자 체감 정확도를 높인다.

### 1.1 배경 — 현재 문제

`DAAU` 검색 사례 (사용자 보고, 2026-04-28):

| 결과 | 단어 포함 여부 | 임베딩 점수 |
|------|:---:|:---:|
| #1 ekdma "프로젝트 진행 현황 보고서…DAAU 메인보드…" | ✅ 포함 | 41% |
| #2 test1 / second | ✗ 무관 | 37% |
| #3 test1 / hello | ✗ 무관 | 36% |
| #4 ekdma / 완료 | ✗ 무관 | 35% |
| #5 ekdma | ✗ 무관 | 27% |

**원인 분석**:
1. **bge-m3는 코사인 유사도 분포가 좁음** — 거의 모든 글이 0.3~0.7 사이로 압축됨
2. **영문 약어("DAAU")는 의미 신호 약함** — 한국어 임베딩 모델이 anchor를 잡지 못함
3. **점수 컷오프 없음** — 무관한 결과도 top-K에 포함됨
4. **데이터 적을 때 더 심함** — 현재 7+5건의 작은 데이터셋에서 차이 흐려짐

### 1.2 핵심 가치

1. **정확도 향상**: 영문 약어·고유명사·짧은 키워드의 매칭 누락·오매칭 감소
2. **임베딩 약점 보완**: 의미 검색의 강점 유지하면서 키워드 신호 추가
3. **운영 중 튜닝 가능**: 환경변수로 가중치·컷오프 조정 (재배포 불필요)
4. **무관 결과 제거**: 컷오프로 노이즈 자동 거름
5. **부담 작은 인프라 변경**: pg_trgm extension 1개 추가 (별도 컨테이너 없음)

## 3. 핵심 도메인 모델

### 3.1 변경 사항 — DB 스키마

```
[추가] PostgreSQL extension: pg_trgm (postgres contrib)

[추가] GIN trigram 인덱스 3개:
  - public.board_posts.title  (gin_trgm_ops)
  - public.board_posts.content (gin_trgm_ops)
  - project.work_logs.content (gin_trgm_ops)

[변경 없음] embedding 컬럼·HNSW 인덱스는 유지
```

### 3.2 점수 계산 공식 (단일 SQL 내)

```text
embed_score   = 1 - (embedding <=> query_vec)
              ∈ [0, 1]

exact_bonus   = (title   ILIKE '%' || keyword || '%') ? TITLE_EXACT   : 0
              + (content ILIKE '%' || keyword || '%') ? CONTENT_EXACT : 0
              # 누적, 최대 0.45

trgm_score    = GREATEST(
                  similarity(title,   keyword) * TITLE_TRGM,
                  similarity(content, keyword) * CONTENT_TRGM
                )

keyword_score = GREATEST(exact_bonus, trgm_score)   # 두 신호 중 강한 것

final_score   = embed_score   * EMBED_WEIGHT
              + keyword_score * KEYWORD_WEIGHT

# WHERE 절: final_score >= MIN_SCORE
# ORDER BY: final_score DESC
```

### 3.3 데이터 흐름 (변경점만)

```
[검색 쿼리]                        ← 변경
  → 사용자 입력 "DAAU"
  → 쿼리 임베딩 (~50-100ms)         ← 동일
  → SQL: pgvector + ILIKE + pg_trgm 결합 ← 변경
       · embed_score  = 1 - (embedding <=> q)
       · keyword_score = max(exact_bonus, trgm)
       · final_score = 0.6*e + 0.4*k
       · WHERE final_score >= 0.45
  → 권한 필터링 (변경 없음)
  → 결과 반환

[글 작성/수정] (변경 없음)
```

- [ ] **기존 검색 회귀 테스트** — 자연어검색에서 동작하던 케이스 유지

### 5.2 Out of Scope (별도 PDCA / 후속)

- 한국어 형태소 분석기(mecab-ko, pg_bigm) 도입 — 운영 부담↑, 효과는 trgm로 충분 가설
- 검색 결과 카드의 "키워드 일치" 배지 — 사용자 결정으로 미포함
- 본문 하이라이트(검색어 노란색 강조) — 별도 PDCA
- BM25 / Rank Fusion (RRF) — 장기 검토
- 검색 히스토리·자동완성·재검색 추천
- 임베딩 모델 교체 (bge-m3 → 다른 모델)
- 댓글·첨부 OCR 검색 (자연어검색 Out-of-Scope 그대로 유지)

## 10. Convention Prerequisites

### 10.1 기존 컨벤션 활용

- ✅ `$queryRaw` 템플릿 리터럴 패턴 (자연어검색에서 검증됨)
- ✅ 환경변수는 services 컨테이너 env 명시
- ✅ Prisma schema 변경 없음 (인덱스만 SQL로)

### 10.2 신규 정의

| 카테고리 | 정의 | 우선순위 |
|---|---|:---:|
| 점수 합산 식 | `final = embed*W_e + max(exact, trgm*W_t)*W_k` | High |
| 점수 컷오프 | `final >= SEARCH_MIN_SCORE` (WHERE 절) | High |
| trigram 함수 | `similarity(col, q)` (pg_trgm 제공) | High |
| 가중치 합 | `EMBED_WEIGHT + KEYWORD_WEIGHT = 1.0` (코드 검증) | Medium |
| sanity 회귀 매트릭스 | 5쿼리 × (이전/이후) 비교표 | Medium |

### 10.3 환경 변수 (신규 8개 — v1.1)

| 변수 | 기본값 | 용도 |
|---|---|---|
| `SEARCH_EMBED_WEIGHT` | `0.6` | 임베딩 점수 가중치 |
| `SEARCH_KEYWORD_WEIGHT` | `0.4` | 키워드 점수 가중치 |
| `SEARCH_MIN_SCORE` | `0.35` | 최종 점수 컷오프 (v1.1: 0.45→0.35) |
| `SEARCH_TITLE_EXACT_BONUS` | `0.30` | title ILIKE 포함 시 가산 |
| `SEARCH_CONTENT_EXACT_BONUS` | `0.15` | content ILIKE 포함 시 가산 |
| `SEARCH_TITLE_TRGM_WEIGHT` | `0.20` | title similarity() 가중치 |
| `SEARCH_CONTENT_TRGM_WEIGHT` | `0.10` | content similarity() 가중치 |
| `SEARCH_KEYWORD_MATCH_BONUS` | `0.20` | **v1.1 추가**: ILIKE 매칭 시 절대 가산 |

> 적용 범위: auth-service, project-service 둘 다 동일 변수 사용. WorkLog는 title 없으므로 content만 적용.
