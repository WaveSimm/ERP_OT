## 1. 개요 (Overview)

작업(Task) 단위의 **시계열 작업 일지**를 기록·축적하고, 별도 "프로젝트 게시판"에서 모든 WorkLog를 프로젝트 단위로 자동 조회할 수 있는 도메인. 작업 진행 상황을 누가·언제·무엇 했는지 추적하고, 후속 분석·보고·인수인계를 위한 1차 정보원으로 활용.

### 1.1 배경

- **현재 상태**: TaskDrawer에 `Task.description` (작업 정의) + `Comment` (토론) 모델만 존재. 시계열 진행 기록·일지 모델이 없음
- **수요**:
  - "내가 이 작업에 무엇을 했는지" 시계열 자동 추적
  - 동료 인수인계·휴가 시 진행 상황 빠른 파악
  - 프로젝트 단위 통합 일지 = 사실상 프로젝트 진행 보고서
  - 작업 분류·검색 (디버깅·현장·기술 자료와 별개의 "작업 단위" 보존)
- **게시판 PDCA 의존성**: 게시판 도메인의 컴포넌트 패턴(BoardSidebar, PostList, MarkdownView 등) 재활용 가능 → **게시판 [Check] 94% 완료 직후 진입**

### 1.2 핵심 가치

1. **시계열 누적**: 작성자별 + 작업일별 다수 기록, 덮어쓰기 없음
2. **소급 입력**: `workedAt` 별도 필드로 어제·지난주 작업 기록 가능
3. **세그먼트 단위 옵션**: Task 단위가 기본, 세그먼트 단위 분리 기록도 가능
4. **자동 집계**: 별도 "프로젝트 게시판" 메뉴에서 모든 WorkLog가 프로젝트별로 자동 리스트 (수동 발행 불필요)
5. **소유권 명확**: 본인 작성건만 수정·삭제 (ADMIN 오버라이드)

## 3. 핵심 도메인 모델

### 3.1 WorkLog (작업비고)

```
WorkLog
├── id, taskId, segmentId (옵션)
├── authorId
├── content (Markdown)
├── workedAt (실제 작업 일자 — 소급 입력 가능)
├── createdAt, updatedAt
└── isDeleted (소프트 삭제)
```

### 3.2 데이터 흐름

```
Task                     WorkLog                    프로젝트 게시판
 │                         │                              │
 ├─ TaskDrawer "비고" 탭 ──┤                              │
 │   ↓                     ├─ author × workedAt ──────┐  │
 │   create/update/delete  │  for taskId              │  │
 │                         │                          │  │
 │   list (taskId 기준)   ◄┤                          │  │
 │                         │                          │  │
                           └──────────────────────────┴──► /work-logs
                                                          (또는 /board/project)
                                                            ├─ 사이드바: 프로젝트 목록
                                                            └─ 본문: 시계열 카드 리스트
```

### 3.3 게시판 vs 작업비고 구분

| 항목 | 게시판 (Board) | 작업비고 (WorkLog) |
|---|---|---|
| 도메인 | 회사 공지·자료실·자유 게시 | 작업 단위 진행 일지 |
| 구조 | BoardCategory → Board → Post | Task → WorkLog |
| 작성 단위 | 보드별 1글 | 작업당 N개 누적 |
| 공유 범위 | audience (전사/부서/Role) | 프로젝트 멤버 (배정자) |
| 주요 가치 | 발행·아카이빙 | 진행 추적·인수인계 |
| 알림 | 공지 발행 시 자동 | (1차 미발생) |
| UI 위치 | `/board/...` | TaskDrawer 비고 탭 + `/work-logs` 페이지 |

- [x] markdown 본문 + 안전 렌더 (rehype-sanitize)
- [x] 작성자/ADMIN 권한 가드 (canEdit, canDelete)
- [x] 배정자/ADMIN/MANAGER 작성 가드 (canCreate)
- [x] 소프트 삭제 (`isDeleted`)
- [x] segmentId 옵션 연결 (TaskDrawer 비고 탭에 "구간 선택" 드롭다운)
- [x] 별도 메뉴 "프로젝트 게시판" (`/work-logs` 또는 `/board/project`) — 사이드바 프로젝트 + 본문 시계열

- 첨부파일 업로드 (필요 시 게시판 첨부 시스템 재활용)

- 알림 자동 발송 (수요 보고 결정)
- 풀텍스트 검색
- WorkLog → 게시판 글 발행 (수동 가져오기 버튼)
- 통계/그래프 (작성 빈도·작성자별 누적)
- 태스크 단위 자동 분류 그룹 (생성 데이터 보고 결정)
- 협업 편집 (다중 작성자 동시 편집)
