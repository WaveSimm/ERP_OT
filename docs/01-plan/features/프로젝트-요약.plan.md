## 1. Overview

### 1.1 배경
프로젝트 목록(`/projects`)에서 프로젝트를 클릭하면 곧바로 간트 상세로 들어간다. **프로젝트의 개요(작성자·참여자·참여부서·자원 현황)를 한 화면에 모아 보는 곳이 없다.** 태스크는 상세 드로어(TaskDrawer)가 있는데 프로젝트엔 그런 요약 화면이 없다.

### 1.2 목표
프로젝트 목록 행에 **"요약" 버튼**을 추가하고, 누르면 **프로젝트 요약 드로어**가 열려 다음을 한눈에 보여준다:
1. **기본 정보** — 이름·상태·진도율·기간·작성자·소유자·생성일·설명
2. **참여 인원** — 이 프로젝트 태스크에 배정된 사람/외부인력/장비 목록 (이름·부서·참여 태스크 수·평균 투입/진척)
3. **참여 부서** — 참여자 부서별 인원 수
4. **자원 종류 요약** — 직원/외부/장비 인원 수

### 1.3 비목표
- 참여자 편집(여기서 배정 변경 X — 조회 전용)
- 프로젝트 전체 자원 부하 히트맵(자원 화면에 별도 존재)
- 예산/비용 (예산 개념 미사용 — 2026-06-24 폐기)

## 2. User Intent
- 핵심: "이 프로젝트에 누가(어느 부서가) 참여하고, 각자 얼마나 투입/진행 중인지 한 번에 보고 싶다."
- 주 사용자: PM·관리자 (프로젝트 현황 파악)

## 3. 성공 기준
1. 목록에서 버튼 한 번으로 요약 드로어 진입
2. 작성자·참여자·참여부서·참여자 자원현황이 한 화면에 표시
3. 조회 전용(편집 없음), 추가 페이지 이동 없이 드로어로 처리

---

## 4. 범위 / 영향 파일
- **백엔드**: `services/project` — 신규 `GET /api/v1/projects/:id/summary` 집계 엔드포인트 (project.service + route)
  - 참여자 = 프로젝트 태스크의 SegmentAssignment 집계(distinct 자원)
  - 부서/이름 = auth `/internal/users/all-with-departments` 1회 호출로 PERSON 해석, EXTERNAL=ExternalPerson, EQUIPMENT=EquipmentResource
- **프론트**: `apps/web` — `projectApi.getSummary`, `ProjectSummaryDrawer` 컴포넌트, `/projects` 행에 요약 버튼

## 5. 리스크
| 리스크 | 완화 |
|--------|------|
| 참여자 많을 때 성능 | 부서 조회는 all-with-departments 1회 + 인메모리 매핑, 자원 집계는 프로젝트 단위 1쿼리 |
| 외부인력/장비는 부서 없음 | 종류별로 분리 표시(부서는 PERSON만) |
| createdBy=auth userId → 이름 해석 | ownerName 해석 패턴 재사용(같은 bulk/all 맵) |
