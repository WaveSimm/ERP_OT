## 1. Overview

## 5. Risks and Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| Layer 3 Refresh 마이그레이션으로 모든 활성 세션 무효화 | High | Certain | 사내 공지 + 작업 시간 외 배포 (전사 배포 전이라 영향 적음) |
| Layer 2 미들웨어 통합 시 일부 라우트 401 회귀 | High | Medium | 각 서비스 sanity 후 다음 layer, k6 baseline 재실행 |
| Internal API JWT 서비스 토큰 도입 (FR-30)이 기존 X-Internal-Token 흐름과 충돌 | High | Medium | 점진 마이그레이션 — 두 방식 병행 → 검증 → 구방식 제거 |
| Layer 5 rate-limit이 정상 사용자 차단 | Medium | Low | 임계값을 보수적으로 설정 (login 5/min은 일반 사용 영향 없음), Redis 기반 슬라이딩 윈도우 |
| Refresh reuse detection 오탐으로 정상 사용자 강제 로그아웃 | Medium | Low | 첫 출시 시 alert만 (자동 무효화 OFF), 1주 모니터링 후 enforcement |
| CSRF 토큰 도입으로 기존 form/API 호출 깨짐 | Medium | High | apps/web의 모든 fetch에 CSRF 헤더 자동 추가 (api.ts 통합 지점) |
| 보안 헤더 (CSP) 너무 엄격해 일부 화면 깨짐 | Medium | High | `Content-Security-Policy-Report-Only`로 1주 모니터링 → 정식 적용 |
| H2 로그인 실패 로그 변경으로 audit consumer 깨짐 | Low | Medium | activity-log consumer 동시 업데이트 |
| 시크릿 회전 (INF-1) 후 일부 서비스 부팅 실패 | High | Low | docker-compose down → secret 갱신 → up. 단계별 실행 |
| Layer 3 작업 중 사용자 신규 로그인이 옛 모델로 발생 | Low | Medium | 마이그레이션 시점에 web 컨테이너 일시 정지 (1~2분) |
| OWASP 매핑 보고서가 13일 전이라 추가 회귀 발견 가능 | Medium | Medium | Plan 작성 시점(5/2)에 spot check 완료, Design 단계에서 한 번 더 코드 grep |

## 6. Architecture Considerations

### 6.1 Project Level Selection

| Level | Selected |
|---|:---:|
| Enterprise | ☑ |

### 6.2 Key Architectural Decisions

| Decision | Selected | Rationale |
|---|---|---|
| Access Token 저장소 | **httpOnly cookie + CSRF 더블 서브밋** | XSS 면역. CSRF 비용은 SameSite=strict + 토큰으로 |
| Access TTL | **1h + sliding refresh** | CLAUDE.md 명세 일치 + 권한 회수 응답성 |
| Refresh 저장 | **SHA-256 hash** | DB 유출 시 활성 세션 제로 |
| 다중 디바이스 | **허용 + "다른 기기 로그아웃"** | UX 개선 + 보안 균형 |
| JWT 알고리즘 | **HS256 + verify에 명시** | 모노레포 + 단일 발급자, 향후 외부 IdP 시 RS256 |
| Refresh rotation | **매 refresh + reuse detection** | 탈취 즉시 감지 |
| 미들웨어 | **services/shared/middleware/** | drift 방지 + 단일 출처 |
| Internal API | **JWT 서비스 토큰 (5분 TTL)** | 토큰 누출 시 영향 5분 한정 |
| Rate Limit 저장소 | **Redis 슬라이딩 윈도우** | 이미 Redis 7 가동 중 |
| 보안 헤더 | **Helmet + Next.js next.config.js 동시** | 백엔드+프론트 양쪽 |
| 에러 포맷 | **`{error: {code, message, details?}}`** | 일관된 클라이언트 처리 |
| CSP 도입 | **Report-Only 1주 → 정식** | 화면 깨짐 사전 감지 |
