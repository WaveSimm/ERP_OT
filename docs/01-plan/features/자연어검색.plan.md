## 1. 개요 (Overview)

게시판(공지·자료실)과 작업비고(WorkLog) 전체 영역에서 **자연어 의미 검색**을 제공. 사용자가 태그나 카테고리를 사전에 붙이지 않아도 "디버깅 메모리 누수", "지난주 영업팀 활동" 같은 자연어 쿼리로 관련 글을 찾을 수 있다.

### 1.1 배경

- **현재 한계**: 게시판은 `LIKE` 검색만 지원 — 정확한 단어 일치만 매칭됨. "디버깅"으로 검색해도 "트러블슈팅"·"메모리 누수 해결" 글은 못 찾음.
- **수요**:
  - 키워드 안 떠올라도 의미로 검색
  - 게시판 + 작업비고 통합 검색 (한 곳에서 다 찾기)
  - 태그·카테고리 수동 분류 부담 제거
- **로컬 운영 환경 활용**: RTX 5060 (8GB VRAM) + 32GB RAM 보유 → **완전 로컬 임베딩 모델 운영 가능**, 회사 데이터 외부 노출 0

### 1.2 핵심 가치

1. **의미 기반 검색**: 같은 단어가 없어도 관련 의미로 매칭
2. **100% 로컬 운영**: 회사 데이터 외부 API에 안 보냄 (보안 ⭐⭐⭐)
3. **태그 불필요**: 사용자가 분류·태그 안 붙여도 자동 의미 인덱싱
4. **통합 검색 범위**: 게시판 Post + 작업비고 WorkLog 모두
5. **무료 운영**: 외부 API 비용 0, 1회 GPU 사용량만

### 1.3 관련 문서

- 게시판 PDCA: 1차 완료 (Match 94%)
- 작업비고 PDCA: 1차 완료 (Match 94%)
- LLM 옵션 검토 (회의 메모): RTX 5060 기준 옵션 A·B·C 검토, **A 선택**

## 3. 핵심 도메인 모델

### 3.1 임베딩 저장 구조

기존 모델에 vector 컬럼 추가:

```
Post (board_posts)
├── ... 기존 컬럼
└── embedding: vector(1024)  ← bge-m3 차원
└── embedded_at: timestamp

WorkLog (work_logs)
├── ... 기존 컬럼
└── embedding: vector(1024)
└── embedded_at: timestamp
```

### 3.2 데이터 흐름

```
[글 작성/수정]
  → DB 저장 (즉시)
  → 비동기 임베딩 큐 (fire-and-forget)
       ↓
       [Embedding Worker] → Ollama/Local 모델
       ↓
       embedding 컬럼 UPDATE

[검색 쿼리]
  → 사용자 입력 "디버깅 메모리"
  → 쿼리 임베딩 (~50ms)
  → pgvector cosine 유사도 top K (~50ms)
  → 권한 필터링 + 결과 정렬
  → 결과 리스트 반환
```

## 5. 범위 (Scope)

### 5.1 In Scope (1차)

- [x] **로컬 임베딩 모델 컨테이너**: Ollama 또는 sentence-transformers Python service (Docker)
- [x] 모델: **BAAI/bge-m3** (한국어 우수, 1024차원, 다국어 지원)
- [x] **pgvector 확장 설치** (PostgreSQL 16)
- [x] Post + WorkLog 테이블에 `embedding vector(1024)` + `embedded_at` 컬럼 추가
- [x] **HNSW 인덱스** (검색 속도 ms 단위)
- [x] 임베딩 워커 (auth-service / project-service에서 fire-and-forget hook)
- [x] **백필 스크립트** — 기존 글·비고 일괄 임베딩 (1회성)
- [x] 검색 API: `GET /api/v1/search?q=...&scope=all|posts|worklogs&limit=20`
- [x] 권한 필터링 (audience + 프로젝트 멤버십)
- [x] 게시판 랜딩 (`/board`) 상단에 검색 바 추가
- [x] 검색 결과 페이지 (`/board/search?q=...`)
- [x] 결과 카드: 타입 라벨(공지·자료실·비고) + 제목 + 스니펫 + 작성자/일자 + 유사도 점수
- [x] 검색 결과 클릭 → 원본 글로 이동
- [x] 빈 본문/짧은 본문에 대한 안전 처리

### 5.2 Out of Scope (2차)

- LLM 답변 생성 (RAG) — 옵션 B/C는 2차에 검토
- 글로벌 검색 (헤더 검색 아이콘) — 1차는 게시판 페이지 한정
- 댓글 검색
- 첨부파일 OCR 텍스트 검색
- 검색 결과 그룹화 (게시판/비고 분리 탭)
- 검색 히스토리·자동완성
- 인덱싱 실패 재시도 (1차 단순 fire-and-forget)
- 다중 모델 비교 (bge-m3 외 e5/MiniLM 옵션)
