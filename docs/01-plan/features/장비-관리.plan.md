## 1. 개요 (Overview)

ERP/OT 플랫폼 내에서 USV(무인수상정), AUV(자율무인잠수정), 멀티빔(MBES) 등 수중·해양 조사 장비의 **전 생애주기**를 관리하는 모듈. 장비 보유 현황, 정비 이력, 프로젝트별 센서 구성(Configuration), 운영 일정을 통합 관리하여 장비 가동률 극대화와 프로젝트 투입 계획 최적화를 달성한다.

### 배경 (Background)

해양 조사 업무에서는 다양한 장비(USV, AUV, MBES, SSS, SBP 등)가 프로젝트별로 서로 다른 센서 조합으로 투입된다. 현재는 장비 상태, 정비 이력, 프로젝트 투입 일정이 스프레드시트나 개별 문서로 분산 관리되어 있어 다음과 같은 문제가 발생한다:

- 장비 가용 여부를 실시간으로 파악하기 어려움
- 정비 주기를 놓쳐 현장에서 장비 고장 발생
- 프로젝트마다 어떤 센서 구성으로 투입했는지 이력이 남지 않음
- 동일 기간에 장비가 중복 배정되는 사고 발생

## 2. 문제 정의 (Problem Statement)

| 문제 | 영향 |
|------|------|
| 장비 보유 현황이 통합되지 않아 한눈에 파악 불가 | 장비 확인을 위해 여러 문서 조회 필요 |
| 정비 이력이 체계적으로 기록되지 않음 | 정비 주기 누락 → 현장 고장 리스크 |
| 프로젝트별 센서 구성(장착 센서 목록)이 기록되지 않음 | 과거 프로젝트 구성 재현 불가, 노하우 유실 |
| 장비 운영 일정을 달력 형태로 볼 수 없음 | 가용 장비 파악 지연, 중복 배정 |
| 장비와 프로젝트 자원 배정이 연결되지 않음 | 자원 계획과 실제 장비 투입 불일치 |
| 센서 교정(Calibration) 주기 관리가 수동 | 교정 만료 상태로 현장 투입 위험 |
| 장비별 가동률/고장률 통계가 없음 | 장비 교체/추가 구매 의사결정 근거 부족 |

## 3. 목표 및 성공 지표 (Goals & Success Metrics)

### 목표
- 모든 장비의 현재 상태를 단일 화면에서 확인
- 정비 이력 + 센서 교정 주기를 체계적으로 관리하여 장비 고장 예방
- 프로젝트 투입 시 센서 구성을 기록·재현할 수 있는 구성 관리 체계
- 장비 운영 일정을 시각적으로 관리하여 중복 배정 방지
- 프로젝트 자원 배정(project-service)과 장비 일정 연동

### 성공 지표

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 장비 등록율 | 100% | 보유 장비 전수 등록 |
| 정비 일정 누락율 | 0% | 예정 정비 대비 미실시 건 |
| 중복 배정 건수 | 0건/월 | 동일 기간 동일 장비 중복 |
| 투입 구성 기록율 | 100% | 프로젝트 투입 시 센서 구성 기록 |
| 장비 가동률 | 측정 가능 | 일정 기반 자동 계산 |

## 4. 사용자 스토리 (User Stories)

### US-01: 장비 등록 및 현황 조회
> **As a** 장비 관리자  
> **I want to** 모든 장비를 등록하고 현재 상태를 한눈에 볼 수 있다  
> **So that** 보유 장비 현황을 즉시 파악할 수 있다

**인수 조건**:
- 장비 기본 정보(이름, 종류, 일련번호, 제조사, 취득일, 사진) 등록
- 장비 종류: USV, AUV, MBES, SSS(Side Scan Sonar), SBP(Sub-Bottom Profiler), GNSS, INS, DVL, 기타
- 현재 상태: 가용(AVAILABLE), 운용중(IN_OPERATION), 정비중(IN_MAINTENANCE), 고장(BROKEN), 퇴역(RETIRED)
- 목록 필터: 종류별, 상태별, 검색

### US-02: 센서 독립 자산 관리 (Sensor Pool)
> **As a** 장비 관리자  
> **I want to** 센서를 독립 자산으로 등록하고 현재 상태·위치를 추적한다  
> **So that** 어떤 센서가 가용한지, 어디에 투입 중인지 즉시 파악한다

**인수 조건**:
- 센서를 독립 자산으로 등록 (예: ADCP-001, MBES-R2Sonic-2020, DVL-Nortek-003)
- 센서 기본 정보: 이름, 종류, 일련번호, 제조사, 모델, 취득일
- 센서 상태: 가용(AVAILABLE), 투입중(DEPLOYED), 정비중(IN_MAINTENANCE), 고장(BROKEN), 퇴역(RETIRED)
- 현재 위치/소속 표시: "창고", "USV-A에 장착 중 (○○ 프로젝트)", "정비센터"
- 센서별 교정(Calibration) 주기 및 최근 교정일 기록
- 교정 만료 경고 (D-30, D-7)
- 센서 투입 이력 전체 조회: 언제, 어떤 장비에, 어떤 프로젝트로 투입되었는지

### US-02-1: 장비-센서 호환성
> **As a** 장비 관리자  
> **I want to** 각 장비(USV, AUV 등)에 장착 가능한 센서 종류를 정의한다  
> **So that** 프로젝트 투입 시 호환되지 않는 조합을 방지한다

**인수 조건**:
- 장비-센서 호환성 매트릭스 관리 (예: USV-A에 ADCP 장착 가능, AUV-B에는 불가)
- 호환성은 장비 종류(카테고리) 레벨 또는 개별 장비 레벨에서 정의 가능
- 투입 구성 시 호환되지 않는 센서 선택하면 경고

### US-03: 정비 이력 관리 (장비 + 센서 공통)
> **As a** 장비 관리자  
> **I want to** 장비와 센서 모두의 정비 이력을 기록하고 조회한다  
> **So that** 정비 패턴을 파악하고 예방 정비 일정을 수립한다

**인수 조건**:
- **장비와 센서 모두** 동일한 정비 이력 체계 적용
- 정비 유형: 예방정비(PREVENTIVE), 수리정비(CORRECTIVE), 교정(CALIBRATION), 업그레이드(UPGRADE)
- 정비 기록: 일시, 유형, 설명, 수행자, 비용, 소요 시간, 교체 부품, 첨부파일(사진/문서)
- 예방 정비 주기 설정 (운용시간 기반 또는 기간 기반)
- 다음 정비 예정일 자동 계산 + 알림
- 센서 교정(Calibration) = 정비 유형 중 하나로 통합 관리

### US-04: 프로젝트 투입 구성 (Deployment Configuration)
> **As a** 프로젝트 매니저  
> **I want to** 프로젝트에 장비를 투입할 때 센서 구성을 지정하고 기록한다  
> **So that** 프로젝트별 장비 구성 이력이 남고, 유사 프로젝트에 재활용할 수 있다

**인수 조건**:
- 프로젝트 투입 시 장비 선택 → 센서 풀에서 센서 **체크아웃** → 구성 확정
- 장착 센서 목록 + 각 센서별 설정 파라미터 기록
- 센서 체크아웃 시 해당 센서 상태 자동 변경: AVAILABLE → DEPLOYED
- 프로젝트 완료/장비 반납 시 센서 상태 복귀: DEPLOYED → AVAILABLE
- 구성 템플릿 저장/불러오기 (예: "표준 수심측량 구성", "정밀 해저면 조사 구성")
- 과거 프로젝트의 투입 구성 조회 및 복제
- project-service의 Resource/SegmentAssignment와 연동

### US-05: 장비 운영 일정 관리
> **As a** 장비 관리자  
> **I want to** 장비별 운영 일정을 달력/타임라인으로 관리한다  
> **So that** 장비 가용 여부를 파악하고 중복 배정을 방지한다

**인수 조건**:
- 장비별 일정 타입: 프로젝트 투입(PROJECT), 정비(MAINTENANCE), 교정(CALIBRATION), 교육(TRAINING), 대기(STANDBY)
- 타임라인(간트) 뷰: 모든 장비의 일정을 한 화면에서 조회
- 달력 뷰: 월간 장비 스케줄 조회
- 일정 충돌 감지: 동일 장비 동일 기간 중복 배정 시 경고
- 프로젝트 일정 변경 시 장비 일정 자동 연동 (project-service 이벤트)

### US-06: 장비 통계 및 리포트
> **As a** 경영진 / 부서장  
> **I want to** 장비 가동률, 정비 비용, 고장 빈도 등의 통계를 본다  
> **So that** 장비 투자 및 교체 의사결정에 활용한다

**인수 조건**:
- 장비별 가동률 (운용일수 / 전체일수)
- 장비 종류별 정비 비용 합계
- 고장 빈도 Top-N
- 기간별 투입 현황 (장비 × 프로젝트 매트릭스)

## 6. 아키텍처 개요 (Architecture Overview)

### 서비스 배치

```
┌─────────────┐    ┌──────────────────┐    ┌─────────────────┐
│  apps/web   │───→│ equipment-service │←──→│ project-service  │
│  (Next.js)  │    │     :3005        │    │     :3003        │
└─────────────┘    └──────┬───────────┘    └─────────────────┘
                          │                         ↕
                   ┌──────┴───────┐          (RabbitMQ 이벤트)
                   │  PostgreSQL  │
                   │  schema:     │
                   │  "equipment" │
                   └──────────────┘
```

- **equipment-service** (Port 3005): 독립 마이크로서비스, 자체 Prisma schema ("equipment")
- **project-service 연동**: 내부 HTTP API + RabbitMQ 이벤트 (`project.schedule.changed`, `equipment.deployed`)
- **auth-service 연동**: JWT 검증 미들웨어 공유

### 핵심 엔티티 관계

```
Equipment (장비: USV, AUV 등 시스템)
├── EquipmentCategory (종류 분류)
├── MaintenanceRecord (정비 이력) [N]        ← 장비+센서 공용
├── EquipmentSchedule (운영 일정) [N]
├── EquipmentSensorCompat (호환 센서) [N:M]
└── DeploymentConfig (프로젝트 투입 구성) [N]
    └── DeploymentSensor (투입 시 장착 센서 + 체크아웃) [N]

Sensor (독립 자산: ADCP, MBES, DVL, GNSS 등)
├── SensorCategory (센서 종류 분류)
├── 상태: AVAILABLE → DEPLOYED → AVAILABLE (체크아웃/반납 순환)
├── 현재 위치: 창고 / 장비-X에 장착 중 (프로젝트-Y)
├── MaintenanceRecord (정비/교정 이력) [N]   ← 장비와 공용 테이블
├── SensorSchedule (센서 일정) [N]
├── EquipmentSensorCompat (호환 장비) [N:M]
└── DeploymentSensor (투입 이력 역추적) [N]
```

> **핵심 개념**: 센서는 "장비의 부품"이 아니라 **독립 자산 풀**에서 관리된다.
> 프로젝트에 USV를 투입할 때, 센서 풀에서 필요한 센서(ADCP, MBES 등)를
> 체크아웃하여 장착하고, 프로젝트 완료 후 반납하면 다시 풀로 돌아간다.

### API 구조 (RESTful)

```
# 장비 (시스템)
/api/v1/equipment                       # 장비 등록/조회/수정/삭제
/api/v1/equipment/:id/maintenance       # 장비 정비 이력
/api/v1/equipment/:id/schedules         # 장비 운영 일정
/api/v1/equipment/:id/compatible-sensors # 호환 센서 목록
/api/v1/equipment/:id/deployments       # 투입 구성 목록
/api/v1/equipment/categories            # 장비 카테고리 관리
/api/v1/equipment/schedules/timeline    # 전체 장비 타임라인 조회

# 센서 (독립 자산 풀)
/api/v1/sensors                         # 센서 등록/조회/수정/삭제
/api/v1/sensors/:id/maintenance         # 센서 정비/교정 이력

/api/v1/sensors/categories              # 센서 카테고리 관리
/api/v1/sensors/available               # 현재 가용 센서 목록
```

## 7. 기술적 고려사항 (Technical Considerations)

### project-service 연동 방식
- project-service의 `Resource` 모델에서 `type = EQUIPMENT`인 자원은 equipment-service의 장비와 1:1 매핑
- `Resource.externalId` 필드 추가 → equipment-service의 `Equipment.id` 참조
- 프로젝트에서 장비 배정 시 equipment-service에 일정 등록 API 호출
- 일정 변경 이벤트는 RabbitMQ로 비동기 동기화

### 일정 충돌 감지
- 장비 일정 등록/수정 시 동일 장비의 기존 일정과 날짜 범위 겹침 검사
- 겹침 허용 타입 조합 정의 가능 (예: STANDBY + PROJECT는 충돌 아님)
- 충돌 시 경고(soft) 또는 차단(hard) 모드 선택 가능

### 정비 주기 관리
- 기간 기반: lastMaintenanceDate + intervalDays → nextDueDate 자동 계산
- 운용시간 기반: 별도 운용시간 입력 or 일정 기반 자동 누적 → 임계값 도달 시 알림
- Cron Job으로 매일 정비 예정일 스캔 → 알림 발송

### 센서 구성 기록
- DeploymentConfig = 특정 프로젝트에 특정 장비를 투입할 때의 스냅샷
- 장착 센서 목록 + 각 센서별 설정 파라미터 (JSON)
- 템플릿으로 저장 시 프로젝트 무관한 재사용 가능 구성
