## 1. 개요 (Overview)

회사 차원에서 관리하는 달력 시스템. 자체 휴일·국가 공휴일·회사 행사·특별 근무일을 통합 관리하고, 향후 프로젝트 일정·근태 계산에 자동 반영할 수 있는 기반을 마련.

### 1.1 배경

- **현재 상태**: 휴일·행사 관리가 부재. 프로젝트 일정·근태 계산에 휴일 개념 없이 운영
- **수요**:
  - 자체 휴일(창립기념일, 단체 휴가) 등록·공유
  - 국가 공휴일 표시
  - 회사 행사(워크숍, 분기 회의) 일정 공유
  - 향후 프로젝트 Gantt/CPM 휴일 자동 제외
  - 향후 근태 OT/특근 자동 판별
- **의존 도메인**: auth(부서·사용자 정책 묶음에 자연 추가), project(미래 통합), attendance(미래 통합)

### 1.2 핵심 가치

1. **회사 정책 단일 출처**: 휴일·행사 = SSOT, 모든 시스템이 참조
2. **단순한 1차**: 일회성 항목 + 종일 + 전사 → 빠른 출시
3. **확장 가능한 데이터 모델**: recurrence·time·department 컬럼은 더미로 두고 2차에 활성화
4. **관리자 한정 입력**: 데이터 정합성 보장
5. **read-only 소비**: attendance/project가 internal API로 조회

### 1.3 관련 문서

- 게시판 Plan/Design (의사결정 패턴 참고)
- 작업비고 Plan/Design (UI 패턴 참고)

## 3. 핵심 도메인 모델

### 3.1 CompanyCalendarEntry

```
CompanyCalendarEntry
├── id, type (PUBLIC_HOLIDAY/COMPANY_HOLIDAY/EVENT/WORKDAY)
├── title, description
├── startDate, endDate (date-only, 동일이면 단일일자)
├── isAllDay = true (1차 고정)
├── color (UI 표시 색)
├── recurrence (null, 컬럼만 — 2차 활성화)
├── targetDepartmentId (null, 컬럼만 — 2차 활성화)
├── createdBy, createdAt, updatedAt
```

### 3.2 타입 분류

| 타입 | 의미 | 근무일 영향 (2차) |
|---|---|:---:|
| `PUBLIC_HOLIDAY` | 국가 공휴일 | ✅ 휴무 |
| `COMPANY_HOLIDAY` | 자체 휴일 (창립기념일 등) | ✅ 휴무 |
| `EVENT` | 회사 행사 (비휴일) | ❌ |
| `WORKDAY` | 특별 근무일 (휴일이지만 출근) | ❌ (강제 근무) |
