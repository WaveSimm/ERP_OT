## 1. 개요 (Overview)

회사 서버 이관 전 erp-ot-platform이 60명 동시 사용자를 안정적으로 지원하는지 검증한다. k6로 ERP 실사용 패턴(출근 burst·평상 mix·검색 burst)을 시뮬레이션하고, dev vs production 빌드 모드 차이를 측정한다.

### 1.1 배경

- **이관 시 사용자 규모**: 약 60명 내부 사용자 (오션테크). 동시 접속 피크는 출근 시간(09:00-09:30) 일제 로그인.
- **현재 검증 부재**: 12개 기능 PDCA 완료(평균 Match 92%)했으나 동시 부하 검증은 단발 sanity test 수준
- **신규 변수**: 자연어검색·검색개선(2026-04-28) 도입으로 Ollama 임베딩이 글 작성 시 fire-and-forget으로 실행됨 → 동시 부하 시 throughput 검증 필요
- **검증 환경**: 로컬 단계 (외부 노출 없음). 회사 서버 이관 후 동일 시나리오 재실행으로 환경 비교 가능

### 1.2 핵심 가치

1. **이관 전 안정성 검증**: 60명 정상 부하에서 p95 < 500ms 달성 가능 여부
2. **임계점 식별**: 90명 스트레스에서 어떤 컴포넌트(Ollama·DB·web·JWT)가 먼저 무너지는지 파악
3. **dev vs prod 차이 정량화**: production 빌드 효과 수치화
4. **재현 가능한 시나리오**: 회사 서버 이관 후 동일 매트릭스로 환경 차이 비교
5. **격리된 시뮬레이션**: 부하 테스트 사용자가 운영 데이터에 노출되지 않음 (환경변수 toggle)

### 1.3 관련 문서

- 종합 검토 보고서: `docs/04-operation/2026-04-24-종합검토보고서.md`
- 이관 가이드: `docs/04-operation/개발환경-이전-가이드.md`
- 자연어검색/검색개선 PDCA: 검색 시스템 부하 영향 분석 근거

## 3. 핵심 도메인 모델

### 3.1 부하 시뮬레이션 사용자 격리 (결정 1)

**명명 규칙**:
- ID: `loadtest-001` ~ `loadtest-090`
- Email: `loadtest-001@erp-ot.load` (운영 도메인 `@oceant.com`/`@erp-ot.local`과 분리)
- Name: `[LOAD] 테스트 사용자N`
- Role: `OPERATOR`

**자동 필터**:
- 환경변수 `HIDE_LOAD_TEST` (default `true`)
- 일반 사용자 조회 API에서 `WHERE NOT email LIKE '%@erp-ot.load'` 적용
- 작성자/결재자/멤버 select 옵션에 자동 제외

**부하 테스트 모드**:
- `HIDE_LOAD_TEST=false`로 시작하면 일반 화면에서도 보임 (디버그 용)
- 기본은 항상 숨김

### 3.2 부하 시나리오 모델 (결정 3)

```
[Scenario A: 출근 burst]                  [Scenario B: 평상 mix]                [Scenario C: 검색 burst]
30초 → 60 VU 램프업                       60 VU constant 10분                    30 VU constant 5분
순서:                                      행동 비율:                              행동:
  1. POST /auth/login                       30%: 게시판 조회 + 검색                  60%: 자연어검색
  2. GET /me                                25%: 작업비고 조회/작성                  20%: 본문 키워드 검색
  3. GET /posts/me/unread-count             20%: 결재 inbox/처리                    20%: 작업비고 검색
  4. GET /attendance/today                  15%: 프로젝트/태스크 CRUD               임베딩 동시 부하 검증
  5. GET /home/dashboard                    10%: 근태/대시보드
  사용자 사이 sleep(1)                      사이 sleep(2~7)                        사이 sleep(0.5~1.5)
```

### 10.2 신규 정의

| 카테고리 | 정의 | 우선순위 |
|---|---|:---:|
| 부하 사용자 도메인 | `@erp-ot.load` (예약, 일반 사용자에게 미노출) | High |
| 격리 필터 위치 매트릭스 | 사용자 select 노출 모든 API 사전 grep + 체크리스트 | High |
| k6 시나리오 명명 | `scenario-{a/b/c}-{morning-burst/mixed/search-burst}.js` | Medium |
| 메트릭 보고서 형식 | round별 표 + 응답시간 그래프 + 병목 분석 | Medium |
| Cleanup 검증 SQL | "0건 잔존" 검증용 SELECT 패턴 | Medium |

### 10.3 환경 변수

| 변수 | 용도 | 범위 | 신규 |
|---|---|---|:---:|
| `HIDE_LOAD_TEST` | 일반 조회에서 부하 사용자 자동 제외 (default `true`) | 모든 백엔드 서비스 | ☑ |
| (선택) `LOAD_TEST_DOMAIN` | 격리 도메인 변경 가능성 (default `@erp-ot.load`) | 모든 백엔드 | ☑ |
