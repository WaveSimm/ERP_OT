## 1. 개요 (Overview)

전사원의 근태 현황을 한눈에 파악할 수 있는 **주간 캘린더 뷰** 페이지.
두 가지 UI로 구성된다:

1. **자원관리 > 근태현황** (조회 전용): 전사원 부서별 주간 캘린더
2. **내 대시보드** (입력): 기존 출퇴근 위젯에 근태 유형(외근/교육/출장) 선택 기능 추가

### 배경 (Background)

- 현재 근태관리는 **개인별** 근태 신청 및 관리에 한정
- 관리자/팀장이 전사 근태를 한눈에 파악할 방법이 없음
- 각 직원이 개인 근태를 입력하면 **전체 공유**되어야 함
- 휴가/OT 등 기존 결재 시스템에서 **승인된 항목은 자동 반영**
- 내 대시보드에 이미 출퇴근 위젯 + 월간 캘린더가 있어 **입력은 대시보드가 자연스러움**

### 핵심 목표

1. 전사원 근태 현황을 부서별 주간 캘린더로 한눈에 **조회**
2. 개인 근태 입력은 **내 대시보드**에서 수행 (출퇴근 시 근무 유형 선택)
3. 승인된 휴가/OT가 캘린더에 자동 반영
4. 모든 로그인 사용자가 전사 근태 조회 가능

## 2. 사용자 스토리 (User Stories)

### US-1: 전사 근태 캘린더 조회
- **As a** 로그인 사용자
- **I want to** 자원관리 > 근태현황 탭에서 전사원의 주간 근태를 볼 수 있다
- **So that** 팀원/타부서 일정을 파악하고 업무 조율이 가능하다
- **AC**: 부서별 섹션, 직원별 행, 요일별 컬럼으로 근태 블록 표시

### US-2: 주간 네비게이션
- **As a** 사용자
- **I want to** 이전 주 / 다음 주 / 오늘로 이동할 수 있다
- **So that** 과거 및 미래 일정도 확인할 수 있다
- **AC**: 좌우 화살표로 주 이동, "오늘" 버튼으로 현재 주 복귀

### US-3: 내 대시보드에서 근태 입력
- **As a** 로그인 사용자
- **I want to** 내 대시보드의 출퇴근 위젯에서 오늘의 근무 유형을 선택할 수 있다
- **So that** 출근과 동시에 외근/교육/출장 등을 한 번에 등록한다
- **AC**: 출퇴근 위젯에 근무 유형 선택 드롭다운 추가
- **AC**: 별도 "근태 추가" 버튼으로 과거/미래 날짜에도 근태 등록 가능

### US-4: 승인된 휴가/OT 자동 표시
- **As a** 사용자
- **I want to** 결재 승인된 휴가와 OT가 캘린더에 자동으로 나타난다
- **So that** 수동 입력 없이 결재 시스템과 연동된다
- **AC**: LeaveRequest(APPROVED) → 해당 날짜에 휴가 블록 자동 표시
- **AC**: OvertimeRequest(APPROVED) → 해당 날짜에 OT 블록 표시

### US-5: 부서별 접기/펼치기
- **As a** 사용자
- **I want to** 부서별 섹션을 접거나 펼칠 수 있다
- **So that** 관심 부서만 집중해서 볼 수 있다
- **AC**: 부서명 클릭 시 토글, 접힌 상태는 세션 유지

## 4. 데이터 모델 (Data Model)

### 4.1 신규 모델: WorkScheduleEntry (개인 근태 항목)

```prisma
model WorkScheduleEntry {
  id          String      @id @default(cuid())
  userId      String
  date        DateTime    @db.Date
  entryType   EntryType   // WORK, FIELD, TRAINING, BUSINESS_TRIP, HALF_AM, HALF_PM, ANNUAL, SICK, SPECIAL, OT
  label       String?     // 사용자 메모 (예: "강남 현장", "안전교육")
  sourceType  EntrySource @default(MANUAL)  // MANUAL | LEAVE_APPROVED | OT_APPROVED
  sourceId    String?     // LeaveRequest.id or OvertimeRequest.id 등 원본 참조
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt

  @@unique([userId, date, entryType, sourceId])
  @@index([userId, date])
  @@index([date])
  @@index([sourceId])
  @@map("work_schedule_entries")
  @@schema("attendance")
}
```

## 5. API 설계 (API Design)

### 5.1 근태현황 조회 API (근태현황 페이지용)

```
GET /api/v1/work-schedule?start=2026-04-07&end=2026-04-13
```

- 인증 필요 (JWT), 권한 제한 없음 (전체 공개)
- 응답: 부서별 그룹 + 직원별 엔트리 목록

### 5.2 개인 근태 추가 API (내 대시보드용)

```
POST /api/v1/work-schedule
Body: { "date": "2026-04-07", "entryType": "FIELD", "label": "강남 현장" }
```

- 본인만 추가 가능 (req.userId 자동 적용)

### 5.3 개인 근태 삭제 API (내 대시보드용)

```
DELETE /api/v1/work-schedule/:id
```

- 본인이 수동 입력한 항목만 삭제 가능 (sourceType === "MANUAL")

### 5.4 전사원+부서 목록 API (신규 내부 API)

```
GET /internal/users/all-with-departments
```

## 6. UI 설계 (UI Design)

### 6.1 자원관리 > 근태현황 (조회 전용)

```
┌─────────────────────────────────────────────────────┐
│  ◀  2026.04.07 ~ 04.13 (15주차)  ▶   [오늘]        │
├─────────────────────────────────────────────────────┤
│ ▼ 경영진 (3명)                                      │
│  ┌────────┬──────┬──────┬──────┬──────┬──────┬──────┐
│  │ 이름   │ 월   │ 화   │ 수   │ 목   │ 금   │ 토/일│
│  │ 김대표 │ 출근 │ 출근 │ 외근 │ 출근 │ 출근 │      │
│  │ 이총괄 │ 출근 │ 교육 │ 교육 │ 출근 │ 연차 │      │
│  └────────┴──────┴──────┴──────┴──────┴──────┴──────┘
│ ▼ 기술팀 (5명)                                      │
│ ► 영업팀 (접힘)                                      │
└─────────────────────────────────────────────────────┘
```

- 클릭/추가/삭제 없음 — 순수 조회
- 부서별 접기/펼치기만 인터랙션

### 6.2 내 대시보드 (입력)

기존 출퇴근 위젯(CheckInWidget)에 **근무 유형 선택** 기능 추가:
- 출근 버튼 옆에 드롭다운: 사무실(기본) / 외근 / 교육 / 출장
- "근태 추가" 버튼: 과거/미래 날짜 + 유형 선택 모달
- "내 근태" 리스트: 본인 이번 주 근태 요약 + 삭제 가능

## 9. 기술적 결정 (Technical Decisions)

| 항목 | 결정 | 이유 |
|------|------|------|
| 데이터 저장 | 별도 WorkScheduleEntry 테이블 | AttendanceRecord는 출퇴근 FSM 전용 |
| 조회 권한 | 전체 공개 (로그인만) | "각자 근태를 입력하면 모든 이들이 공유" |
| 입력 위치 | 내 대시보드 | 기존 출퇴근 위젯과 자연스러운 UX 통합 |
| 근태현황 | 조회 전용 | 입력/편집은 대시보드 담당, 책임 분리 |
| 자동 반영 | 승인 시점에 동기적 생성 | 이벤트 기반보다 단순, MVP 적합 |
| 뷰 단위 | 주간 (월~일) | 레퍼런스 스크린샷 기준 |
