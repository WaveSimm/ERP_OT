## 1. 개요

회사의 공용자산(차량·시설)을 일자/시간 단위로 예약하는 시스템. 자원관리 페이지에 "공용자산 예약" 탭으로 통합되어 OPERATOR 이상이 직접 예약하고 충돌을 자동 차단한다.

### 1.1 배경

- **현재 상태**: 공용자산(차량·시설)은 `/admin/equipment-resources`에서 ADMIN이 마스터만 등록. 누가 언제 쓰는지 추적·예약 불가.
- **이전 결정 흐름**:
  - 2026-05-04: 자원-모델-분리 PDCA Phase 1 — `EquipmentResource` 모델 신설
  - 2026-05-05 오후: 공용자산 마스터 페이지를 `/admin/equipment-resources`로 이동, EQUIPMENT 타입 폐기 (시설/차량만), 프로젝트 미연계로 격리
  - 2026-05-05 후반: 사용자 명시 — "이제 공용자산을 사용할 방법이 있어야겠어"
- **수요**:
  - 회의실 충돌 방지 (같은 시간 두 팀 사용 불가)
  - 차량 출장 일정 등록·조회
  - 정기 회의 반복 예약
  - 누가 언제 무엇을 쓰는지 시각 확인

### 1.2 핵심 가치

1. **충돌 방지**: 같은 자원 동시 예약 자동 차단
2. **시간 정밀도**: 30분 단위로 회의·출장 단위 표현 가능
3. **반복 지원**: 정기 회의 등 매일/매주/매월 등록 한 번
4. **회사달력 인지**: 휴일·주말 시각 표시 (예약 차단 X, 사용자 자율)
5. **자원관리 통합**: 직원·외부인력 부하와 같은 화면에서 공용자산도 함께 운영

### 1.3 관련 문서

- `자원-모델-분리.plan.md` (모델 신설 배경)
- `회사달력.plan.md` v1.2 (휴일 시각 통합 패턴)

## 3. 핵심 도메인 모델

### 3.1 EquipmentReservation

```
EquipmentReservation
├── id
├── resourceId          → EquipmentResource.id (차량/시설)
├── userId              → 예약자 (auth_users.id)
├── title               예: "분기 임원회의", "고객사 방문 출장"
├── description?        선택 — 상세 메모, 동승자 등
├── startAt             datetime (UTC 저장, KST 표시)
├── endAt               datetime
├── isAllDay            true면 시간 무시, 일자 단위 예약
├── recurrence?         반복 패턴 (null이면 단일 예약)
├── recurrenceParentId? 반복 시리즈의 첫 인스턴스 id (예외 처리용)
├── status              CONFIRMED / CANCELED
├── createdAt, updatedAt
```

### 3.2 반복 패턴 (recurrence)

```
{
  "freq": "DAILY" | "WEEKLY" | "MONTHLY",
  "interval": 1,                    // 매 N일/주/월 (기본 1)
  "byWeekday"?: ["MON","WED",...]   // WEEKLY 한정
  "until"?: "2026-12-31",           // 종료일 (또는 count)
  "count"?: 10                      // 횟수
}
```

JSON 컬럼으로 저장. 1차에서는 단순 형식만 사용 (RRULE 표준 채택은 2차 검토).

### 3.3 인스턴스 전개 vs 가상 인스턴스

- **저장**: 반복은 **단일 row + recurrence JSON**으로 저장 (DB 부담 ↓)
- **조회**: 캘린더 표시 시점에 서비스 레이어에서 **조회 기간 내 인스턴스 가상 전개** (충돌 검사·표시 모두 전개된 인스턴스 기준)
- **부분 수정/취소**: 한 인스턴스만 변경 시 **예외 row 추가** (`recurrenceParentId` 동일 + `recurrence=null` + 변경 내용)

## 5. 범위

### 5.1 In Scope (1차)

- [ ] FR-01 EquipmentReservation 모델 + 마이그레이션 (project schema)
- [ ] FR-02 `GET /api/v1/equipment-reservations?from&to&resourceId&userId` — 기간·자원·예약자 필터 조회 (반복 인스턴스 전개)
- [ ] FR-03 `GET /api/v1/equipment-reservations/:id` — 단건 조회
- [ ] FR-04 `POST /api/v1/equipment-reservations` — 등록 (충돌 검사)
- [ ] FR-05 `PATCH /api/v1/equipment-reservations/:id` — 수정 (본인 또는 ADMIN/MANAGER)
- [ ] FR-06 `DELETE /api/v1/equipment-reservations/:id` — 취소 (`status=CANCELED` soft) (본인 또는 ADMIN/MANAGER)
- [ ] FR-07 충돌 검사 로직 — 같은 resourceId + 시간 겹침 + status≠CANCELED → 차단 (반복 시리즈는 모든 인스턴스에 대해 검사)
- [ ] FR-08 반복 예약 인스턴스 전개 (서비스 레이어, 조회 시점)
- [ ] FR-09 반복 시리즈 부분 수정/취소 (예외 row 패턴)
- [ ] FR-10 자원관리 페이지 신규 탭 "공용자산 예약" — 직원현황과 외부 자원 사이
- [ ] FR-11 캘린더 뷰: 자원별 가로 타임라인 (월간/주간 토글)
- [ ] FR-12 예약 추가/수정 모달: 자원 / 종일·시간 / 시작·종료 / 반복 / 사유 / 비고
- [ ] FR-13 시간 단위 30분 정밀도 (`<select>` 또는 `<TimeInput step={1800}>`)
- [ ] FR-14 회사달력 휴일·주말 시각 표시 (예약 차단 X) — `useHolidaysMap` 재활용

- [ ] FR-16 내 예약 목록 (탭 내 사이드 또는 별도 섹션)
- [ ] FR-17 충돌 시 명확한 에러: "5/10 14:00~16:00 대회의실은 이미 예약됨 (예약자: 김OO, 사유: ...)"

|---|---|---|---|

| 시간 단위 | 1분 / 15분 / 30분 / 1시간 | **30분** | 사용자 결정 |
| 충돌 정책 | 동시 1예약 차단 / 수량 풀 | **동시 1예약 차단** | 사용자 결정 |

| 다른 사람 수정 | 본인만 / ADMIN+MANAGER / 누구나 | **본인 + ADMIN/MANAGER** | 사용자 결정 |
| Soft delete | 즉시 삭제 / status CANCELED | **status CANCELED** | 이력 추적, 감사 |
