# ERP·OT 플랫폼 소개 PPT — 구성과 페이지별 목적

> 발표 자료 `ERP-OT-플랫폼-소개.pptx`(16:9, 20슬라이드)의 구성 정리.
> 각 슬라이드의 "목적"은 PPT 발표자 노트에도 동일하게 기재되어 있음.
> 생성 스크립트: `build_ppt.py` (python-pptx, 수정 후 재실행 시 덱 갱신).

| 묶음 | 슬라이드 | 의도 |
|------|----------|------|
| ① 도입 | 1~5 | 무엇·왜·효과 — 관심과 공감 형성 |
| ② 전경 | 6 | 전체 구성 지도로 위치감 |
| ③ 기능 | 7~12 | 모듈별 상세(핵심 본문) |
| ④ 지식검색 | 13~14 | 차별화 포인트 OT-Brain |
| ⑤ 기반 | 15~17 | 기술·인프라·보안 신뢰 |
| ⑥ 마무리 | 18~20 | 현황·로드맵·행동 유도 |

## 페이지별 목적

| # | 제목 | 페이지 목적 |
|---|------|-------------|
| 1 | 표지 | 발표 주제와 대상(전 임직원), 자료 성격(사내 소개)을 제시 |
| 2 | 발표 구성과 페이지별 목적 | 전체 흐름과 각 묶음 목적을 미리 제시, 청중이 흐름 예측 |
| 3 | 한눈에 보기 | 시스템 정체성을 한 문장으로 각인, 세부 전 '전체상' 제공 |
| 4 | 도입 배경과 목적 | 현재 불편(분산·중복·검색불가)을 짚고 해결 목표로 연결 |
| 5 | 도입 효과 | 영역별 구체적 이득 — '나에게 무슨 도움인가' |
| 6 | 전체 구성 지도 | 시스템 레이어(업무/지식/공통/기술)를 지도로 |
| 7 | 기능 ① 프로젝트 관리 | 핵심 모듈 구조(프로젝트-작업-세그먼트)와 자동 집계 강점 |
| 8 | 기능 ② 자원·장비 | 인력·장비 운용과 '과부하 가시화' 가치 |
| 9 | 기능 ③ 수리(AS) ★실운영 | 검증된 모듈 강조 → 신뢰. 성공 사례 |
| 10 | 기능 ④ 회계 | 하위 12기능을 '마스터-흐름'으로 압축 |
| 11 | 기능 ⑤ 결재·게시판 | 결재·협업까지 포함함을 전달 |
| 12 | 기능 ⑥ 근태·관리자 | 관리 체계까지 갖췄음을 전달 |
| 13 | OT-Brain ① | NAS 통합검색 규모·원리 — '묻힌 자료를 깨운다' |
| 14 | OT-Brain ② | '바로 열어보는' 경험 + 비용0·사내보관 강점 |
| 15 | 기술·아키텍처 | 검증된 현대 스택 + 체계적 품질관리로 신뢰 |
| 16 | 인프라 | 외부 비용 없이 사내 1대로 운영되는 자립 인프라 |
| 17 | 보안 | 데이터 보호 우려 선제 해소 — '밖으로 안 나간다' |
| 18 | 운영 현황·성숙도 | 솔직한 현재 위치 — 검증(AS) vs 확장 중 구분 |
| 19 | 향후 로드맵 | 방향 제시로 기대감, 피드백 반영 안내 |
| 20 | 마무리 | 접속 방법·피드백 채널 안내 → 사용 유도 |

## 발표 운영 팁

- **대상별 강조점**: 경영진 → 5(효과)·17(보안)·18(현황) / 실무자 → 7~12(기능)·13~14(검색) / 신규입사자 → 3·6·20
- **소요 시간**: 슬라이드당 1~1.5분 기준 약 25~30분 (질의응답 별도)
- **데모 연계**: 9(수리 실운영)·13~14(NAS 검색)는 실제 화면 시연 권장
- **수정**: 내용 변경은 `build_ppt.py` 편집 후 재실행, 상세 근거는 `ERP-OT-플랫폼-소개.md` 참조
