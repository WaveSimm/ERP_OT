## 1. 한눈에 보기

**오션테크 ERP·OT 통합 플랫폼**은 회사의 프로젝트, 인력·장비 자원, 수리(AS), 구매·재고·회계, 전자결재, 사내 협업, 그리고 수십 년간 축적된 NAS 자료 검색까지 **하나의 웹 시스템에서 처리**하는 자체 구축 통합 업무 플랫폼입니다.

| 구분 | 내용 |
|------|------|
| **무엇** | 해양 관측·계측 사업(부이·관측소·선박)을 위한 맞춤형 통합 업무 시스템 |
| **누가** | 오션테크 전 임직원(부서별 권한 차등) |
| **어디서** | 사내 웹 브라우저 접속(설치 불필요) |
| **왜** | 분산된 엑셀·수기 업무를 통합하고, 프로젝트·자원·회계를 한 흐름으로 연결 |
| **특징** | 해양 사업 도메인 특화 · 외부 SaaS 미사용(사내 자체 운영) · 데이터 사내 보관 |

## 2. 도입 배경과 목적

### 2.1 기존의 문제
- 프로젝트 일정·인력 배정·장비 운용이 **개별 엑셀·담당자 머릿속**에 분산
- 수리(AS), 발주, 재고, 경비정산이 **서로 단절** → 같은 정보를 여러 번 입력
- 수십 년 NAS 자료(도면·보고서·계약서)가 **폴더에 묻혀 검색 불가**
- 결재·승인이 **종이·메신저**로 흩어져 이력 추적 어려움

### 2.2 목표
1. **단일 진실 원천(Single Source of Truth)**: 프로젝트·자원·자산을 한 곳에서 관리
2. **업무 연결**: 프로젝트 → 자원 배정 → 장비 운용 → 수리 → 발주·정산 → 결재를 하나의 흐름으로
3. **지식 자산화**: NAS 수십 년 자료를 통합 검색 가능한 지식 베이스로
4. **사내 운영**: 외부 클라우드 의존 없이 사내 인프라에서 안전하게 운영

## 5. 기능 모듈 상세

### 5.1 🗂 내 대시보드 / 🎯 지휘센터
- **내 대시보드**: 개인에게 배정된 작업·일정·결재 대기·알림을 모아 보는 개인 홈
- **지휘센터**: 전사 프로젝트 진척·자원 가동·이슈를 한 화면에 종합한 경영·관리자 뷰

### 5.2 📋 프로젝트 관리
해양 사업(부이 유지보수·관측소·선박 작업 등) 프로젝트의 일정과 자원을 관리하는 핵심 모듈.
- **계층 구조**: 프로젝트 → 작업(Task, 논리 컨테이너) → 작업 세그먼트(실제 일정 + 자원 배정)
- **자원 배정**: 세그먼트별로 인력·장비를 %(분담) 또는 시간/일 단위로 배정
- **진도율 자동 집계**: 하위 작업 진척이 상위 프로젝트 진도율로 자동 반영(집계 캐시)
- **마일스톤**: 작업에 시점 표시(플래그) 가능
- **템플릿**: 반복 사업 유형을 템플릿화(일정 오프셋 방식)하여 신규 프로젝트 빠르게 생성
- **그룹핑**: 부서·고객사·커스텀 기준으로 프로젝트 묶음(N:M) 관리

### 5.3 👥 자원 관리
- 사람(직원)을 단일 자원 모델로 관리(인증 계정과 통합)
- 프로젝트·일정에 배정된 인력의 **가용성·부하율** 시각화
- **피크/과배정 경고**: 특정 기간 과배정 인력 식별
- 휴가·근태와 연동하여 실제 가용 인력 반영

### 5.4 🔧 장비 관리
- 장비를 **독립 자산 풀**로 관리(센서·계측기 등)
- **체크아웃/반납 순환**: 장비를 프로젝트에 대여·회수하는 흐름
- **장비 마스터**: 모델명·제조사·SKU·재고 통일 표기
- 일정(스케줄)·센서·통계 하위 화면 제공

### 5.5 🛠 수리(AS) 관리  ★현재 실운영 중
회사에서 가장 먼저 정식 운영에 들어간 모듈.
- **접수 → 수리 → 부품 사용 → 정산** 이력 일원화
- 고객사·공급사·부품 마스터 연계
- 통계 화면으로 수리 추이·부품 소진 파악
- 엑셀로 관리하던 기존 수리 데이터를 마이그레이션하여 이관 완료

### 5.6 📦 회계 (구매·재고·발주·정산)
구매-재고-결재를 연결하는 통합 회계 모듈. 하위 12개 기능:

| 하위 기능 | 설명 |
|-----------|------|
| 고객사(customers) | 고객·발주처 마스터(정규화·병합 규칙 적용) |
| 공급사(suppliers) | 매입처 마스터 |
| 품목(products) | 품목·SKU(ProductVariant) 마스터 |
| 재고(inventory) | 재고 수량·이동 관리 |
| 창고/위치(locations) | 재고 보관 위치 |
| 발주(orders) | 발주서 생성·결재·송금 워크플로우 |
| 입고(inbound) | 입고 요청·검수 |
| 번들(bundles) | 묶음 출하(BundleShipment) |
| 계약(contracts) | 계약 관리 |
| 정산(settlements) | 발주·매입 정산 |
| 경비(expenses) | 부서 경비 접수(재무팀 연계) |
| 감사(audits) | 변경 이력 추적 |
- **발주 워크플로우**: 결재라인·회계정산·발주송금·번호체계 포함. 부분결제 자동 분할 지원
- **재고 SKU 모델**: 품목 변형(ProductVariant)·입고요청·번들출하 체계

### 5.7 📝 전자결재
- **결재 양식**: 경비정산·지출결의서 등(EXPENSE 양식으로 일원화)
- **결재선**: 부서·직급 기반 결재라인 설정(관리자에서 정의)
- 상신 → 승인/반려 → 완료 흐름과 이력 추적
- 경비정산은 ERP 회계와 통합(매뉴얼 제공)

### 5.8 📋 게시판 & 협업
- **카테고리 게시판**: 공지·자료·기능요청 등 카테고리별 글
- **기능 요구 게시판**: 릴리즈 피드백·개선 요청 관리(전용 관리 페이지)
- **작업 비고**: 작업 단위 메모·협업
- **회사 달력**: 한국 공휴일 자동 갱신(KASI 연동) + 사내 일정

### 5.9 근태·인사 (attendance)
- **근태 현황**: 출퇴근·근무 일정(WorkScheduleEntry 별도 관리)
- 휴가·휴일근무 신청 시 근태에 자동 동기화
- 인력 현황·부하율 계산과 연계

### 5.10 ⚙ 관리자 기능
| 기능 | 설명 |
|------|------|
| 사용자(users) | 계정·직급·권한 관리 |
| 결재라인(approval-lines) | 부서·직급별 결재선 정의 |
| 회사달력(calendar) | 공휴일·사내 일정 관리 |
| 장비자원(equipment-resources) | 장비 마스터 관리 |
| 기능요청(feature-requests) | 사내 개선 요청 트래킹 |
| 활동로그(activity-logs) | 감사·접근 이력 |

## 6. OT-Brain — NAS 통합 지식 검색

수십 년간 축적된 회사 NAS 자료를 **검색 가능한 지식 자산**으로 만드는 별도 지식 플랫폼. ERP 게시판의 **🔎 NAS 통합검색**으로 접속.

### 6.1 규모
- **약 804만 파일 / 30.4TB** 스캔 완료(도면·보고서·계약서·카탈로그·사진 등)
- 파일명·폴더 경로는 **전량 즉시 검색**, 본문(텍스트)은 단계적 추출 중

### 6.2 검색 방식 (한국어·영어 모두)
- **키워드 검색**: 파일명·폴더 경로 부분일치 + 단어경계 정밀도(한국어 교착어 대응)
- **의미 검색(AI)**: 다국어 임베딩(bge-m3)으로 뜻이 비슷한 문서까지 검색
- **중요도 가중(IDF)**: 흔한 단어(매뉴얼·사진)는 낮추고 구체적 주제어(모델명·선박명)는 높여 정확도 향상
- **재순위(Reranker)**: GPU 기반으로 상위 결과 정밀 재정렬

### 6.3 파일 열기 (2026-06-11 신규)
- 검색 결과에서 **파일명 클릭** → PDF·이미지·텍스트는 브라우저 미리보기, 그 외(엑셀·한글·도면)는 다운로드
- 폴더 경로 복사 기능도 제공(탐색기에서 바로 열기)

### 6.4 운영 방식
- **비용 0 운영**: 로컬 AI(임베딩·검색·답변) 사용, 외부 전송 없음
- 본문 추출은 **야간 자동 처리**(업무시간엔 검색 우선, GPU 경합 회피)

> ※ OT-Brain은 향후 ERP와 양방향 연동되는 **오션테크 통합 지식 플랫폼**으로 확장 예정.

## 8. 기술 스택과 아키텍처

### 8.1 기술 스택
| 계층 | 기술 |
|------|------|
| 프론트엔드 | Next.js 14+ (App Router), TypeScript, Tailwind CSS, TanStack Query, Zustand |
| 백엔드 | Node.js 20 LTS, Fastify, TypeScript, Prisma(ORM), Zod(검증) |

| 인프라 | Docker Compose (사내 온프레미스) · Turborepo 모노레포 |

### 8.2 아키텍처 원칙
- **모노레포**: 프론트(apps) · 백엔드(services) · 공유 패키지(packages)를 한 저장소에서 관리
- **클린 아키텍처(4계층)**: API → 애플리케이션 → 도메인 ← 인프라
- **마이크로서비스**: 인증·사용자·프로젝트·장비·결재·경비 등 서비스 분리(서비스별 DB 스키마)
- **문서 우선·PDCA**: 설계 문서 → 구현 → 검증(Gap 분석) → 보고 주기로 품질 관리

## 9. 인프라 구성

- **약 16개 컨테이너**가 사내 서버 1대(Docker)에서 운영
  - PostgreSQL 16(pgvector) · Redis 7 · RabbitMQ 3.13
  - 8개 마이크로서비스(auth·user·project·equipment·approval·expense 등)
