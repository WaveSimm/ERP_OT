# 영향분석 현재상태분석 재구성 (Report)

**일자**: 2026-06-23 | **대상**: services/project(impact.service), apps/web(ImpactPanel)

## 배경
"현재 상태분석"이 What-If와 사실상 중복이었음.

## 구현
- impact.service 재작성: buildContext/propagate/projectEndChange + `computeActualDelay`(지연+미완료 실측), `analyzeCurrentState(projectId)`가 실제 지연 태스크 목록 반환. What-If는 가정용으로 분리.
- 라우트 `GET /:projectId/impact`→analyzeCurrentState(파라미터 없음). ImpactPanel: 모드탭(현재상태/What-If), delayedTasks 목록, 불필요 UI 제거.

## 검증
- 실제 지연 데이터 기반 현재상태 표시 확인.
