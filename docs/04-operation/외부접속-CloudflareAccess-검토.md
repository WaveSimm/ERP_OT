# 외부 접속 검토 — Cloudflare Tunnel + Access (③안)

> 작성: 2026-07-06
> 상태: **검토안** (도입 결정 전 — 미결정 사항은 7장)
> 배경 논의: 지사(부산)↔본사(경기)는 site-to-site VPN 구성 완료.
> 남은 문제 = **현장 직원의 스마트폰·노트북 수시 접속**.

## 1. 목표와 비목표

**목표**
- 현장(외부망)에서 스마트폰·노트북 **브라우저만으로** ERP 웹 접속
- VPN 앱 토글 등 사용 마찰 제거 — "안 쓰게 되는 보안"이 되지 않게
- 공유기 포트 개방 **0개** 유지 (인바운드 공격면 추가 없음)

**비목표 (이 안에서 다루지 않음)**
- SSH·DB·NAS 등 비웹 접근 → 개발자는 기존 사내 VPN 유지
- 사내망 전체 개방 — 열리는 것은 ERP 웹 딱 하나

## 2. 구성도

```
[현장 폰/노트북]                      [본사 서버 .41]
     │ https://erp.<도메인>              │
     ▼                                   │ cloudflared 데몬 (아웃바운드 터널)
┌─────────────────────┐   터널(서버가    │   └→ localhost:3000 (erp-ot-web)
│ Cloudflare 엣지      │◄── 먼저 연결) ──┘
│  - TLS(HTTPS) 종단   │
│  - Access 인증 게이트 │      * 인바운드 포트 개방 없음
│    (이메일 OTP/SSO)  │      * 방화벽 인바운드 규칙 추가 없음
└─────────────────────┘
```

동작: 서버의 `cloudflared`가 Cloudflare로 **아웃바운드** 연결을 유지 →
외부 사용자는 `erp.<도메인>` 접속 → Cloudflare Access가 **신원 인증 후**에만
터널을 통해 web(3000)으로 전달. 인증 실패 시 서버에 트래픽이 도달조차 안 함.

## 3. 인증 정책(안)

| 항목 | 안 |
|---|---|
| 1차 인증 (Access) | **이메일 One-time PIN** — 허용 이메일 목록(직원 명단) 등록. 회사 구글 워크스페이스가 있으면 Google SSO로 업그레이드 |
| 2차 인증 (ERP) | 기존 ERP 로그인 그대로 (id/pw) — 이중 게이트 구조 |
| Access 세션 | 24h(안) — 매일 아침 PIN 1회, 이후 하루 무마찰 |
| 접속 대상 제한 | `erp.<도메인>` 하나만. admin(3010)·preview(3009)는 미노출 |
| 국가 제한 | 한국만 허용 (Access 정책 1줄) — 해외 스캐닝 원천 차단 |

기기 분실 시: Access 콘솔에서 해당 사용자 세션 즉시 폐기 + ERP 비밀번호 변경.
(ERP refreshToken 7d는 모바일 고려 시 3d 단축 검토 — 별도 결정)

## 4. 도입 절차 (예상 소요: 반나절)

1. 도메인 준비 — 회사 도메인을 Cloudflare에 등록(네임서버 이전) 또는 서브도메인 위임
2. 서버에 `cloudflared` 설치 + 터널 생성 (`erp.<도메인>` → `http://localhost:3000`)
3. Access 애플리케이션 생성 + 정책(직원 이메일 목록, 한국 한정, 세션 24h)
4. ERP측 보완 (아래 5장) 적용
5. 파일럿: 현장 직원 2~3명 폰으로 1주 시험 → 전사 안내

## 5. ERP측 선행/병행 조치

| 조치 | 이유 | 상태 |
|---|---|---|
| 쿠키 `secure` 플래그 활성 | 이제 HTTPS 접속이므로 필수 (현재 NODE_ENV 조건부 — 보안 미결 배치 B와 동일 항목) | 필요 |
| IDOR 잔여 점검 | 외부 접점 생기기 전 필수 (배치 B) | 필요 |
| CVE 스캔 | 〃 (배치 B) | 필요 |
| 모바일 화면 점검 | 현장 주사용 기능(근태·작업비고·결재)이 폰 화면에서 사용 가능한지 | 필요 |
| API `Cache-Control: no-store` | 외부 PC/폰 브라우저 캐시 잔존 최소화 | 권장 |
| 접속 실패 알림 연동 | Access 로그 → 기존 ops 알림 체계 | 권장 |

## 6. 비용·의존성

- **비용 0원**: Cloudflare Free 플랜 + Access 무료 티어(50명) — 현 인원 규모 커버.
  도메인이 없다면 도메인 구입비(연 ~2만원)만 발생.
- **외부 의존**: Cloudflare 장애 시 외부 접속 불가 (사내망·지사 VPN 경로는 무영향).
  ERP 자체는 사내에서 정상 — 외부 접속만 끊기는 격리된 장애 범위.
- **개인정보 경유**: 트래픽이 Cloudflare를 경유(TLS 종단). 근태·경비 등 개인정보가
  지나가므로 내부 검토 필요 시 명시. (Cloudflare는 ISMS/SOC2 등 인증 보유)

## 7. 미결정 사항 (도입 결정 시 확정 필요)

- [ ] 사용할 도메인 (기존 회사 도메인 유무, Cloudflare 이전 가능 여부)
- [ ] 직원 인증 이메일 — 개인 이메일 허용 vs 회사 계정 통일
- [ ] Access 세션 길이(24h안) / ERP refreshToken 단축(7d→3d) 여부
- [ ] 파일럿 대상 인원
- [ ] admin 패널(3010)의 외부 접속 필요 여부 (기본: 미노출 유지)

## 8. 대안 요약 (기각 사유)

| 안 | 기각/보류 사유 |
|---|---|
| ① 로드워리어 VPN 프로필 | 현장에서 VPN 앱 토글 마찰 → 실사용률 낮고 우회(카톡 공유 등) 유발 |
| ② Tailscale | 유력한 차선. 다만 전 직원 폰에 앱 설치·계정 관리 필요. 비웹 접근이 필요해지면 재검토 |
| 직접 공개(포트포워딩+프록시) | 인바운드 공격면 생성, 인증서·프록시 직접 운영 부담. 배치 B 완료 전 불가 |

**개발자 노트북은 이 안과 무관하게 기존 사내 VPN 유지** (SSH·DB 접근 필요).
