## 1. 기관명 정규화 규칙

### 1.1 접두사 제거 (자동 적용)

| 제거 대상 | 예시 변환 |
|---|---|
| `㈜` (유니코드) | `㈜오션` → `오션` |
| `(주)` | `(주)해양정보기술` → `해양정보기술` |
| `주식회사` | `주식회사 하이브` → `하이브` |

**SQL 템플릿**:
```sql
UPDATE equipment.customers
SET name = TRIM(regexp_replace(name, '^(㈜|\(주\)|주식회사)\s*', ''))
WHERE name ~ '^(㈜|\(주\)|주식회사)';
```

→ 담당자(`customer_contacts.name`)에도 동일 패턴 적용 가능 (현재는 해당 없음).

### 1.3 알려진 중복·병합 규칙

엑셀 이관 시 발견되는 **동일 기관의 다른 표기들**. 이관 파서에서 자동 매핑.

|---|---|
| 인하대학교 | 인하대 |
| 오션재고 / 오션테크(데모) / 과학기지 | 오션테크 |
| ㈜해원문화재연구소 / 해원문화재연구소 | 해원문화재연구소 |
| 남동해 수산연구소 (공백 O) | 남동해수산연구소 (공백 X) |
| 제주수산 | 제주해양수산연구원 |
| 조사원 | 국립해양조사원 (KHOA) |
| SM오션 | 에스엠오션 (SM OCEAN) |
| 과학기지 | 오션테크 |
| CNC오션 / CNC OCENA(오타) | 씨앤씨오션 (CNC OCEAN) |

### 1.4 별개 유지 (유사하지만 서로 다른 기관)

혼동하기 쉬우므로 이관 시 자동 병합하지 **말 것**.

- **부산대학교** vs **부산대학교산학협력단** (본교 vs 산학협력단 법인)
- **한국해양대학교** vs **한국해양수산연구원** (대학 vs 연구원)
- **전남대** vs **목포해양대** 등 지역별 대학 (개별)

## 2. 담당자(Contact) 정규화 규칙

### 2.1 중복 제거 (동일 기관 내 동일 이름)

정규화된 이름이 같으면 중복. 우선순위:
1. `cc_` 접두사(기존 데이터) > `migcc_` 접두사(이관 데이터)
2. 동률이면 `createdAt` 빠른 것 유지

**SQL 템플릿**:
```sql
WITH ranked AS (
  SELECT id,
    ROW_NUMBER() OVER (
      PARTITION BY "customerId", lower(regexp_replace(name, '[\s\-_.·,\(\)]', '', 'g'))
      ORDER BY CASE WHEN id LIKE 'migcc_%' THEN 2 ELSE 1 END, "createdAt" ASC
    ) AS rk
  FROM equipment.customer_contacts
)
DELETE FROM equipment.customer_contacts WHERE id IN (SELECT id FROM ranked WHERE rk > 1);
```

### 2.2 직책 접미사 분리

`이름 직책` 또는 `이름직책님` → `이름` + `position=직책`

**인식 직책 목록**:
```
주임, 대리, 과장, 부장, 차장, 실장, 팀장, 이사, 상무, 전무, 사장, 대표, 회장,
박사, 교수, 연구원, 기술원, 기술사, 주무관, 사무관, 서기관
```

**SQL 템플릿**:
```sql
UPDATE equipment.customer_contacts
SET
  name = TRIM(regexp_replace(name, '\s*(주임|대리|과장|...)님?\s*$', '')),
  position = CASE
    WHEN COALESCE(position, '') = '' THEN
      regexp_replace(
        COALESCE(substring(name from '(주임|대리|...)님?\s*$'), ''),
        '님\s*$', '')
    ELSE position
  END
WHERE name ~ '(주임|대리|...)님?\s*$' AND length(name) <= 15;
```

### 2.3 노이즈 담당자 식별·제거

명백한 삭제 대상 유형:
- **메모·설명문**: 예) "수리비과다로 장비 OT반납(제조사운송비 OT에서 지불)"
- **회사명만 기재**: 예) "올포랜드" (회사명 자체)
- **외부인**: 예) "원광대 정신택교수님" (다른 기관 소속인 교수)
- **명사·역할**: 예) "재고" (개인 이름 아님)
- **전화번호만**: 예) "010-6603-0203"

식별 SQL:
```sql
SELECT cc.name, c.name AS org
FROM equipment.customer_contacts cc
JOIN equipment.customers c ON c.id = cc."customerId"
WHERE
  cc.name ~ '^[\d\-\s\(\)]+$'           -- 숫자/전화번호
  OR cc.name LIKE '%교수%'               -- 외부인 교수
  OR cc.name ~ '(주식회사|㈜|기술단|연구소|회사|산업|공사)$'  -- 회사명
  OR cc.name IN ('재고', '올포랜드');    -- 명사
```

### 2.4 이름 정제 (긴 수식 trim)

`이름 직책/학위/(회사명)` 형식 → `이름`만.

수동 처리 필요 (자동화 어려움). 예시:
- `이현흥 대표이사(오션선박기술)` → `이현흥`
- `박창욱 대표 / 해양기술사(오셔닉)` → `박창욱`
- `김동헌 주임 010-5584-7019` → `김동헌` + `phone=010-5584-7019`

### 2.5 유지 원칙

- **짧은 이름(2글자)**: 실제 짧은 한국 이름일 수 있음 → **유지**
- **외국인 이름**: 긴 외국어 이름도 유지
- **애매한 경우 삭제 지양** — 데이터 손실 리스크가 높음

## 3. 이관 파서 적용 순서 (권장)

수리완료(1,084건) 또는 차후 엑셀 이관 시:

### Phase 1: 기관(Customer) 처리
1. 엑셀에서 기관명 추출 (unique)
2. **1.1** 접두사 제거 정규화 적용
3. **1.3** 병합 규칙 조회 → canonical 기관명으로 매핑
4. 매핑 실패 기관 → `matching_검토용.xlsx` 출력 → 사용자 확정
5. 신규 기관만 DB `customers`에 INSERT
6. **1.2** 영문 약자인 경우 정식명 병기 형태로 저장

### Phase 2: 담당자(Contact) 처리
1. 엑셀에서 (기관, 담당자명) pair 추출
2. **2.3** 노이즈 필터 (자동 skip)
3. **2.4** 이름 정제 시도 (정규식 가능한 것만, 나머지는 notes로)
4. **2.2** 직책 접미사 분리 자동 적용
5. DB `customer_contacts`에 INSERT
6. 이관 완료 후 **2.1** 중복 제거 실행

### Phase 3: 사후 검증
1. 중복 그룹 재검사: 없어야 함
2. 노이즈 패턴 재검사: 없어야 함
3. Ad-hoc 사용자 리뷰: 애매한 기관명 쌍 확인
