## 4. 인프라·운영 (Top 이슈)

### 🚨 Critical (8건)

1. **실제 API 키 평문 저장** — `.env`에 `ANTHROPIC_API_KEY=sk-ant-api03-DJc…`, `CLOVA_OCR_SECRET_KEY=ZGx0…` (gitignore는 되어있으나 로테이션·스크러빙 정책 없음)
2. **시크릿 fallback** — `JWT_ACCESS_SECRET:-change_me_in_production`, `INTERNAL_API_TOKEN:-dev_internal_token_min32chars_change`
3. **DB/Redis/RabbitMQ 동일 기본 비밀번호** — `erp_password`/`redis_password`
4. **포트 0.0.0.0 바인딩** — 5432·6379·5672·15672·9090 + 백엔드 3001-3007 모두 외부 노출
5. **헬스체크 부재** — 14개 컨테이너 중 `ocr-engine`만 정의, `depends_on`은 대부분 start-only
6. **PostgreSQL SPOF** — 단일 인스턴스, 7개 서비스 + OCR 스키마 혼재, replica/Patroni 없음
7. **WAL/PITR 부재** — 최대 데이터 손실 24h
8. **오프사이트 백업 없음** — 47개 .sql.gz 모두 같은 호스트 `./backups/`에 존재

### ⚠️ High

- 로그 순환(log rotation) 설정 없음 → 호스트 디스크 점유 위험
- Reverse proxy (Nginx/Traefik) 부재 → TLS 종료 미해결
- 리소스 제한 없음 (ocr-engine 4g만) → OOM 전파 위험
- `container_name:` 고정 → `docker compose up --scale` 불가
- **볼륨 백업 누락**: `file_storage` · `approval_uploads` · `ocr_uploads` · `rabbitmq_data` · `redis_data`
- `RABBITMQ_PASS` vs `RABBITMQ_PASSWORD` env 키 drift (현재 우연히 같은 값)

### 이관 준비도

`docs/04-operation/개발환경-이전-가이드.md`에 **미포함** 항목:
- API 키 로테이션 절차

- TLS/Reverse Proxy 설정
