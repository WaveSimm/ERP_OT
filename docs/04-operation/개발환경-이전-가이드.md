# 개발 환경 이전 가이드

**목적**: 다른 PC에서 ERP-OT 프로젝트를 이어서 개발하는 방법
**작성일**: 2026-03-19

---

## 전체 흐름

```
현재 PC                          새 PC
─────────────────                ─────────────────────────────
1. git commit                    4. 필수 소프트웨어 설치
2. GitHub repo 생성              5. git clone
3. git push                      6. .env 설정
                                 7. docker compose up --build
                                 8. (선택) DB 데이터 복원
```

---

## 현재 PC 작업

### 1. 첫 커밋 생성

```bash
cd E:/claude/ERP_OT

git commit -m "Initial commit: ERP-OT platform"
```

### 2. GitHub 원격 저장소 생성

1. https://github.com/new 접속
2. Repository name: `erp-ot-platform`
3. **Private** 선택 (회사 코드)
4. "Initialize this repository" 체크 해제 (빈 repo)
5. Create repository 클릭

### 3. 원격 연결 및 Push

```bash
git remote add origin https://github.com/<본인계정>/erp-ot-platform.git
git branch -M main
git push -u origin main
```

> 이후 개발 내용은 작업 완료 후 `git add . && git commit -m "..." && git push` 로 동기화

---

## 새 PC 작업

### 4. 필수 소프트웨어 설치

| 소프트웨어 | 권장 버전 | 설치 경로 |
|-----------|---------|---------|
| Docker Desktop | 최신 | https://www.docker.com/products/docker-desktop |
| Node.js | 20 LTS | https://nodejs.org |
| pnpm | 9.x | `npm install -g pnpm@9` |
| Git | 최신 | https://git-scm.com |

설치 확인:
```bash
docker --version
node --version    # v20.x.x
pnpm --version    # 9.x.x
git --version
```

### 5. 코드 Clone

```bash
git clone https://github.com/<본인계정>/erp-ot-platform.git
cd erp-ot-platform
```

### 6. 환경변수 설정

```bash
cp .env.example .env
```

`.env` 파일을 열어 아래 항목 확인/수정:

```env
# 로컬 개발이면 기본값 그대로 사용 가능
JWT_ACCESS_SECRET=your_access_secret_here    # 임의 문자열로 변경 권장
JWT_REFRESH_SECRET=your_refresh_secret_here  # 임의 문자열로 변경 권장

# 초기 관리자 계정 (변경 원할 경우)
ADMIN_EMAIL=admin@erp-ot.local
ADMIN_INITIAL_PASSWORD=Admin1234!
```

> JWT Secret은 로컬 개발 시 기본값 그대로 써도 동작하지만,
> 인터넷 오픈 환경에서는 반드시 32자 이상 랜덤 문자열로 변경하세요.

### 7. 전체 서비스 실행

```bash
docker compose up -d --build
```

최초 실행 시 시간이 걸립니다 (이미지 빌드 + DB 초기화 자동 수행).

실행 상태 확인:
```bash
docker compose ps
```

로그 확인 (DB 초기화 완료 확인):
```bash
docker compose logs -f auth-service
# 정상: "Admin user created: admin@erp-ot.local" → "auth-service running on port 3001"

docker compose logs -f project-service
# 정상: "DB schema ready. Starting service..."
```

### 8. 접속 확인

브라우저에서 http://localhost:3000 접속 후 로그인:

- 이메일: `admin@erp-ot.local`
- 비밀번호: `Admin1234!` (또는 .env에서 설정한 값)

## (선택) 기존 데이터 이전

코드만 옮기면 빈 DB로 시작합니다.
기존 PC의 프로젝트/태스크/템플릿 데이터를 함께 옮기려면 아래 절차를 따르세요.

### 현재 PC — 데이터 백업

```bash
docker exec erp-ot-postgres pg_dump -U erp_user erp_ot > erp_ot_backup.sql
```

`erp_ot_backup.sql` 파일을 USB나 클라우드로 새 PC로 복사합니다.

### 새 PC — 데이터 복원

새 PC에서 `docker compose up -d --build` 가 완료된 후 실행:

```bash
docker exec -i erp-ot-postgres psql -U erp_user erp_ot < erp_ot_backup.sql
```

> 복원 후 서비스를 재시작할 필요는 없습니다.

## 이후 동기화 방법

두 PC 사이에서 코드를 계속 동기화하는 일반적인 워크플로우:

```bash
# 작업 시작 전 (새 내용 받기)
git pull

# 작업 완료 후 (변경 내용 올리기)
git add .
git commit -m "작업 내용 요약"
git push
```

Docker 컨테이너는 코드 변경 시 재빌드가 필요합니다:

```bash
# 특정 서비스만 재빌드
docker compose build auth-service
docker compose up -d auth-service

# 전체 재빌드
docker compose up -d --build
```

## 문제 해결

### 포트가 이미 사용 중
```bash
# 사용 중인 포트 확인
netstat -ano | findstr :3000

# Docker 컨테이너 전체 중지
docker compose down
```

### DB 연결 실패 (auth-service/project-service가 재시작 반복)
PostgreSQL이 아직 준비되지 않은 것입니다. `entrypoint.sh`가 자동으로 재시도하므로 1~2분 기다리면 됩니다.

```bash
docker compose logs postgres   # DB 준비 상태 확인
```

### 이미지 빌드 캐시 문제
```bash
docker compose build --no-cache
docker compose up -d
```
