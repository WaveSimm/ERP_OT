# 보안관리대장 (Security Management Register)

> **목적**: ERP·OT 플랫폼(사내 온프렘)의 보안 점검 결과·조치 현황·진행 기록을 지속 관리하는 살아있는 문서.
> **갱신 규칙**: 점검/조치/사고가 있을 때마다 ① 취약점 대장 상태 갱신 ② 하단 「진행 기록」에 날짜별 추가.
> **기준선**: KISA 「주요정보통신기반시설 기술적 취약점 분석·평가 상세가이드」(U-항목) + OWASP(`docs/04-operation/2026-05-02-OWASP-매핑.md`).
> **대상**: ERP_OT(`github.com/WaveSimm/ERP_OT`), ot-brain(`github.com/WaveSimm/ot-brain`), 호스트 `oceantech`(192.168.0.41, Ubuntu 26.04, 온프렘).

---

## 1. 보안 아키텍처 현황 (노출면)

- **외부(0.0.0.0) 리슨**: `22/SSH`, `3000/web` 단 2개 (2026-06-13 배치 A로 축소).
- 그 외 인프라·백엔드(postgres·redis·rabbitmq·ollama·ocr-engine·ocr-test-ui·서비스 3001~3008)는 **127.0.0.1 바인딩** → 외부 도달 불가. 컨테이너 간 통신은 docker 네트워크.
- ⚠️ **남은 핵심 노출**: SSH(비번 인증 ON)·web(평문 HTTP, LAN). → VPN + 방화벽 + 리버스프록시(TLS)로 잠그는 것이 다음 목표.
- 참고(외부 조언, 사내): 외부 포트 개방 금지·VPN 사용·SSH는 개발자 IP만·서버 네트워크 C클래스(서브넷) 분리.

## 3. 양호 항목 (검증됨, 조치 불요)

- **JWT 코어**: HS256(알고리즘 혼동 차단), 시크릿 32자+ Zod 검증·폴백 없음, Access 1h/Refresh 7d, 리프레시 sha256 저장+회전+재사용탐지, bcrypt 12.
- **SQL Injection 없음**: 전부 Prisma 태그드 템플릿/`$N` 바인딩(knowledge-api 동적 SQL도 토큰은 바인딩, hex 가드).
- **게시판 XSS 없음**: react-markdown + `rehypeSanitize`, `rehype-raw` 미사용, `dangerouslySetInnerHTML` 없음.
- **경로탐색 차단**: `path.basename`+정규화+`..`/절대경로 거부, 저장명은 UUID.
- **업로드**: 크기/타입 allowlist 적용.
- **커맨드 인젝션·SSRF 없음**: child_process 없음, fetch는 env 내부 URL만.
- **redis 인증 있음**(`--requirepass`), `.env` 미커밋(.gitignore), Helmet/HSTS/보안헤더 적용.
- **KISA-U 리눅스**: UID0=root만, /etc/passwd(644)·shadow(640), telnet/ftp/r-서비스 없음, world-writable 없음, PermitEmptyPasswords no.
