## mix 시나리오 URL 정정 (2026-05-02 발견)

`scripts/load-test/scenario-b-mixed.js`에 존재하지 않는 엔드포인트 4건:

| 잘못된 URL | 정정 |
|---|---|
| `/api/v1/posts?categoryCode=notice&limit=20` | `/api/v1/posts/feed?categoryCode=notice&limit=20` |
| `/api/v1/me/tasks?limit=5` | `/api/v1/me/kanban` (응답 구조: columns.IN_PROGRESS[].taskId) |
| `/api/v1/me/tasks?limit=20` | `/api/v1/me/kanban` |
| `/api/v1/dashboard/personal` | `/api/v1/dashboard` |

**영향**:
- 4월 28일 baseline의 mix error 32~34%는 시나리오 결함이 만든 허위 신호
- 정정 후 60 VU mix는 ~4% timeout만 — 실제 시스템은 60 VU 평상 부하를 안정 처리 중
- 90 VU 스트레스에서도 ~4% — 임계점 가까운 상태가 아님
- **부하테스트 PDCA Match Rate 96% 재산정 필요** — gap-detector가 URL 동작 검증을 안 함
- worklog write block은 loadtest 사용자에 segment_assignments가 0건이라 실제 POST 미발생 (임베딩 부하 측정 누락)

## 주요 관찰

### 1. 검색 burst 60VU 회귀 (Round 6) — 유일한 진짜 회귀
- p95 search 모든 scope 70~79% 악화 (487→859ms 등)
- **원인 추정**: 임베딩 데이터 누적 + HNSW 인덱스 비대화
- 30 VU 정상(Round 3)는 +11~38%로 견딜 만함
- **별도 PDCA 후보** (`/pdca plan 검색-회귀-조사`)

### 2. mix 시나리오는 깨끗한 baseline 확보
- 60 VU: p95 140ms, error 4.0% (timeout만)
- 90 VU: p95 179ms, error 3.9% (timeout만)
- **모든 endpoint 200 OK**, login 100% 성공 유지
- 보안 패치 후 비교 기준으로 신뢰 가능

### 3. burst 시나리오 — 60VU 개선/90VU 악화
- 60 VU(Round 1): home -21%, login -29% 개선되었으나 error +140% (1.20→2.88%)
- 90 VU(Round 4): home +33%, error 개선
- **원인 추정**: 백그라운드 hook (집계/알림/임베딩) ↔ dev 모드 컴파일 경합

## 결론

**baseline 갱신 완료. 진짜 신호는 검색 burst 회귀 1건.**

- mix 시나리오: ~4% timeout만 — 안정
- search 시나리오: 60 VU에서 70~79% 회귀 — 별도 조사 필요
- burst 시나리오: 60/90 모두 임계점 근처 — 모니터링 강화 필요

보안 패치 진행 시 baseline 비교 가능 항목:
- mix p95 (현재 140/179ms) → 패치 후 +20% 이상이면 alert
- mix error rate (현재 ~4%) → 패치 후 5%+ 악화 시 alert
- burst home p95 (60VU 768ms / 90VU 1.79s) → +30% 악화 시 alert
