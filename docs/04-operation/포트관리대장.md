# 포트관리대장 (Port Inventory)

> **목적**: ERP_OT·ot-brain·호스트(192.168.0.41)에서 사용하는 모든 포트의 노출 범위·용도를 한눈에 관리. 신규 서비스 추가/방화벽 검토 시 본 대장을 갱신.
> **작성**: 2026-06-20 (실행 컨테이너 + `ss -tln` 실측 기준).
> **노출 표기**: 🔒 `127.0.0.1`(로컬 전용, LAN도 차단) / ⚠️ `0.0.0.0`(LAN 노출) / 🧩 컨테이너 내부(호스트 매핑 없음).

---

## 1. 노출 원칙

- **신규 서비스는 기본 `127.0.0.1` 바인딩** — 외부 필요 시 리버스프록시 경유(직접 0.0.0.0 금지).
- **외부(인터넷) 차단은 네트워크 경계 보안장비**가 담당(서버는 사내 LAN `192.168.0.0/16` 내부).
- 호스트 방화벽(ufw)은 경계장비 + SSH 키인증 + 127.0.0.1 바인딩의 다층방어로 **완화 수용**(V-09) — 별도 호스트 방화벽 미적용.

## 4. 호스트 프로세스 (compose 외)

| 포트 | 용도 | 노출 | 보안 |
|------|------|------|------|
| 22 | SSH | ⚠️ 0.0.0.0 | **키 인증 전용**(V-06, 비번 차단), root 차단(V-07) |
| 8080 | 사진첩 작성 페이지 | ⚠️ 0.0.0.0 | 직원 LAN 사용 |
| 3389·3390 | 원격제어(팀뷰어류) | ⚠️ 0.0.0.0 | 사용 중. 도구 자체 인증 + 경계장비 |

---

## 5. 외부(LAN) 노출 요약 — 실측 4종

`ss -tln`에서 0.0.0.0/* 로 잡힌 것:
- **22** SSH(키인증) · **3000** web(직원) · **8080** 사진첩(직원) · **3389/3390** 원격제어(사용중)

그 외 모든 백엔드·DB·MQ·otbrain은 `127.0.0.1` → LAN 차단.

## 6. 갱신 규칙

- 신규 docker 서비스: `ports`에 `127.0.0.1:` 접두 필수(외부 필요 시에만 0.0.0.0 + 리버스프록시).
- 호스트 신규 리슨 추가/제거 시 본 대장 갱신.
- 관련: `보안관리대장.md`(V-03 포트 바인딩·V-09 방화벽·V-05 TLS).
