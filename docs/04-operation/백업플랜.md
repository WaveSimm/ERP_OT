# ERP / ot-brain 백업 플랜

**작성 2026-06-17** · 대상: 운영 서버(`192.168.0.41`) ERP_OT + ot-brain

> 원칙: **Git = 코드(로직)**, **NAS = 데이터(상태)**. 둘 다 있어야 완전 복구. 한 곳만으론 불완전.

---

## 1. 자산별 백업처

| 자산 | 백업처 | 비고 |
|------|--------|------|
| 코드 (ERP_OT·ot-brain) | **Git(GitHub)** | 커밋/push 시. 오프사이트 ✅ |
| DB `erp_ot`(~3MB)·`otbrain`(~17GB 덤프) | **NAS** `\\192.168.0.220\erp-backup` | 운영 데이터·검색 청크 |
| 업로드 첨부 (auth·approval·expense·ocr) | **NAS** | 파일 |
| 시크릿 `.env` 2개 | **NAS** | 복구 필수 |
| Ollama/OCR 모델 | 백업 안 함 | 재다운로드 |
| NAS 원본 문서 | 백업 안 함 | 이미 NAS 원본 |
| `file_storage` 볼륨 | 백업 안 함 | 현재 비어있음 |

**복구 공식**: 코드 = `git clone`, 데이터 = NAS에서 복원.

## 2. NAS 백업 자동화 (서버 주도, 구현됨)

- **계정**: `otbrain-scan` (전용·최소권한 — erp-backup 쓰기 가능, 문서 원본 oceantech엔 쓰기 차단)
- **자격증명**: 서버 `~/.nas-backup-cred` (chmod 600, **git 제외**)
- **방식**: 서버 user crontab → `erp-nas-backup.sh` (sudo 불필요, docker+smbclient 컨테이너로 업로드)
- **스크립트**: `scripts/backup/erp-nas-backup.sh` (오케스트레이터) + `scripts/backup/nas-uploader.sh` (컨테이너 업로드/보존정리)
- **스테이징**: `~/.nasbackup_stage` (업로드 후 삭제)

| 대상 | 주기(crontab, UTC) | NAS 위치 | 보존 |
|------|------|------|------|
| erp_ot DB(full) + 업로드 4종 + .env | 매일 `0 0 * * *` (KST 09:00) | `erp-backup/daily/` | 30일 |
| otbrain DB(full 17GB) | 주1회 `0 5 * * 0` (일 KST 14:00) | `erp-backup/weekly/` | 28일 |

> ⏰ EXIF 야간배치(UTC 11:00–23:00 = KST 20:00–08:00)와 **시간대 회피**해서 NAS 경합 방지.
> 파일명에 `_YYYYMMDD` 포함 → 보존기간 경과분 자동 정리.

### 수동 실행
```bash
ssh oceantech@192.168.0.41
sh ~/erp-nas-backup.sh daily     # 즉시 일일 백업
sh ~/erp-nas-backup.sh weekly    # 즉시 otbrain 백업
tail -f ~/nas-backup.log         # 로그
```

### 상태 확인 (NAS 내용)
```bash
docker run --rm -v ~/.nas-backup-cred:/cred:ro alpine sh -c \
 'apk add -q samba-client; smbclient //192.168.0.220/erp-backup -A /cred -c "cd daily; ls"'
```
※ Windows 탐색기에서 안 보이면: 공유 "네트워크 숨김" 또는 계정 뷰 차이일 수 있음 → 위 smbclient ls가 정본.

## 3. 로컬 백업 (기존, 빠른 롤백용)

- `erp-ot-backup` 컨테이너: 매일 02:00, ERP DB+업로드 → `~/ERP_OT/backups/` (서버 로컬), 30일.
- ⚠️ **같은 서버 디스크**라 DR(재해복구) 정본 아님. **정본은 NAS.** otbrain은 여기서 빠짐(NAS가 유일).

## 4. 복구 절차 (포맷/재설치 시)

1. 새 OS → Docker → `git clone` (ERP_OT·ot-brain) — 코드
2. **NAS에서** 최신 `daily/`+`weekly/` 내려받아: `pg_restore`(erp_ot, otbrain) + 업로드 tar 해제 + `.env` 복원
3. 모델 재다운로드(`ollama pull bge-m3` 등) → 가동
4. 상세: `사내서버-이관-가이드-Ubuntu-2026-06-12.md` 6·7부
   - ⚠️ otbrain 복원은 pgvector 확장 필요 → compose의 pgvector postgres 이미지로 띄우면 자동 처리

## 5. 장기 개선 (재설치 후 권장)

- **WAL 아카이빙(PITR)**: otbrain 진짜 증분(매주 full 17GB 반복 제거)
- **Synology Btrfs 스냅샷**: NAS측 버전 보관 저비용
- `db-backup` 컨테이너를 ERP+otbrain 통합 + NAS 직접 타깃으로 일원화
