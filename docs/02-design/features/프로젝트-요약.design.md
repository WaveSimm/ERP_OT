# 프로젝트-요약 설계

> 작성일: 2026-06-24
> 대상: `services/project` (3003) + `apps/web`
> 관련: 프로젝트-관리, 자원-기여도-진척률

## 1. 백엔드 — `GET /api/v1/projects/:projectId/summary`

### 응답 스키마
```ts
{
  id, name, status, description,
  overallProgress: number,
  startDate: string|null, endDate: string|null,   // effectiveStart/End → YYYY-MM-DD
  createdBy, creatorName: string|null,
  ownerId, ownerName: string|null,
  createdAt: string,
  counts: { person: number, external: number, equipment: number, departments: number },
  departments: { name: string, count: number }[],   // PERSON 참여자 부서별 인원 (sortOrder 정렬)
  participants: {
    resourceId: string,
    type: "PERSON"|"EXTERNAL"|"EQUIPMENT",
    name: string,
    departmentName?: string|null,   // PERSON만
    company?: string|null,          // EXTERNAL만
    taskCount: number,              // 참여 distinct 태스크 수
    segmentCount: number,           // 배정된 세그먼트 수
    avgContribution: number,        // 평균 분담율
    avgProgress: number,            // 평균 본인 진척률
  }[]
}
```

### 집계 로직 (project.service.getProjectSummary)
1. project 로드(없으면 404). effectiveStart/End, status, overallProgress(기존 계산), description, createdBy, ownerId, createdAt.
2. 프로젝트의 task→segment→assignment 로드:
   `task.findMany({ where:{projectId}, include:{ segments:{ include:{ assignments:true } } } })`
3. assignment들을 자원키(personUserId | externalPersonId | equipmentResourceId)별로 그룹.
   각 자원: segmentCount, distinct taskCount, avg(contributionWeight), avg(progressPercent).
4. 이름/부서 해석:
   - PERSON: auth `GET /internal/users/all-with-departments` 1회 → id→{name, departmentName, departmentSortOrder} 맵. (createdBy/ownerId 이름도 같은 맵에서 해석)
   - EXTERNAL: `externalPerson.findMany({ where:{ id: in [...] } })` → name, company.
   - EQUIPMENT: `equipmentResource.findMany` → name.
5. departments: PERSON 참여자를 departmentName으로 group → [{name, count}] (sortOrder 오름차순, 미지정은 "부서 미지정").
6. counts 집계.

### 라우트
`services/project/src/api/routes/project.routes.ts` 에 `fastify.get("/:projectId/summary", ...)` 추가. 권한: 로그인 사용자(조회). 캐시: 없음(요청 시 집계) — 추후 필요 시 cache 추가.

## 2. 프론트 — ProjectSummaryDrawer

### API 클라이언트
`projectApi.getSummary(id)` → `GET /projects/:id/summary`.

### 컴포넌트 `apps/web/src/components/ProjectSummaryDrawer.tsx`
- 우측 슬라이드 드로어(또는 중앙 모달). TaskDrawer 스타일 차용.
- 섹션:
  1. **헤더**: 프로젝트명 + 상태 배지 + 닫기
  2. **기본 정보**: 진도율 바, 기간(시작~종료), 작성자, 소유자, 생성일, 설명
  3. **참여 종류 요약**: 직원 N · 외부 N · 장비 N · 부서 N (칩)
  4. **참여 부서**: 부서별 인원 막대/리스트
  5. **참여 인원**: 표 — 이름 · 부서/회사 · 태스크수 · 평균투입% · 평균진척% (종류 아이콘). 길면 스크롤.
- 조회 전용(편집 없음).

### 진입점
`apps/web/src/app/projects/page.tsx` 프로젝트 행에 **요약 버튼(ℹ️/"요약")** 추가 → `setSummaryProjectId(p.id)` → 드로어 오픈. 행 클릭(상세 이동)과 충돌 없게 `stopPropagation`.

## 3. 엣지케이스
- 참여자 0명: "배정된 자원이 없습니다" 표시.
- auth 호출 실패: 이름/부서 빈 값 graceful(자원ID/“-”), 나머지 정상 렌더.
- 외부/장비: 부서 컬럼 "-"(또는 회사명/장비표시).
- avg 계산 시 0 나눗셈 방지.

## 4. 미적용(향후)
- 요약 캐시, 참여자 클릭 시 개인 자원 현황 드릴다운, 부하 히트맵 링크.
