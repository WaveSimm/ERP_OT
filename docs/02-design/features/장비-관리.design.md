### v1.3 변경 내역 (2026-04-06)
- **EquipmentComponent 모델 추가**: 장비 구성요소(엔진, GPS, 모뎀 등) 관리
- **Deployment.equipmentId → optional (String?)**: 센서 단독 프로젝트 투입 지원
- **센서 상세 페이지 장비와 동일 수준으로 격상**:
  - 전체 이력 탭 (구분 필터 드롭다운)
  - 정비이력 등록 폼
  - 운영일정 탭 (프로젝트 투입 + 기타 일정)

- **전체 이력 탭**: 정비이력 + 운영일정 통합 조회, 투입이력 탭 제거
- **일정 충돌 감지 이중화**: ScheduleService + DeploymentService 양쪽에서 검사
- **기타 일정등록**: 타입 드롭다운(교정/교육/대기), 정비는 정비이력에서 관리

### 1.2 서비스 간 통신

```
[Client] ──HTTP→ equipment-service:3005
                     │
                     ├── auth-service:3001     (JWT 검증, X-Internal-Token)
                     ├── project-service:3003  (프로젝트 정보 조회, 내부 API)
                     └── RabbitMQ:5672         (이벤트 비동기 통신)

이벤트 흐름:
  project-service → "project.schedule.changed" → equipment-service (일정 동기화)
  equipment-service → "equipment.deployed"      → project-service  (투입 알림)
  equipment-service → "equipment.returned"      → project-service  (반납 알림)
  equipment-service → "maintenance.due"         → notification     (정비 예정 알림)
```

### 1.3 project-service 연동 설계

**Resource 모델 확장** (project-service 측):
```
Resource 테이블에 externalId 필드 추가
  - type = EQUIPMENT → externalId = equipment-service의 Equipment.id
  - type = PERSON    → externalId = null (기존 userId 사용)
```

**연동 흐름**:
1. 프로젝트에서 장비 자원 배정 시 → equipment-service에 일정 등록 API 호출
2. equipment-service에서 직접 투입 구성 생성 시 → project-service에 알림
3. 장비 일정 충돌 시 → project-service 측에 경고 반환

## 2. DB 스키마 설계 (Prisma Schema)

```prisma
// services/equipment/prisma/schema.prisma

generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["multiSchema"]
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
  schemas  = ["equipment"]
}

// ─────────────────────────────────────────────
// Enums
// ─────────────────────────────────────────────

enum AssetStatus {
  AVAILABLE        // 가용 (창고 대기)
  IN_OPERATION     // 운용중 (프로젝트 투입)
  DEPLOYED         // 센서: 장비에 장착되어 투입중
  IN_MAINTENANCE   // 정비중
  BROKEN           // 고장
  RETIRED          // 퇴역

  @@schema("equipment")
}

enum MaintenanceType {
  PREVENTIVE   // 예방 정비
  CORRECTIVE   // 수리 정비
  CALIBRATION  // 교정
  UPGRADE      // 업그레이드

  @@schema("equipment")
}

enum ScheduleType {
  PROJECT      // 프로젝트 투입
  MAINTENANCE  // 정비
  CALIBRATION  // 교정
  TRAINING     // 교육
  STANDBY      // 대기

  @@schema("equipment")
}

enum DeploymentStatus {
  PLANNED    // 계획됨
  ACTIVE     // 투입 진행중
  COMPLETED  // 완료 (반납됨)
  CANCELLED  // 취소

  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 1. 카테고리 (장비 종류 + 센서 종류 통합)
// ─────────────────────────────────────────────

model Category {
  id          String   @id @default(cuid())
  name        String                         // "USV", "AUV", "MBES", "ADCP" 등
  type        String                         // "EQUIPMENT" | "SENSOR"
  description String?
  sortOrder   Int      @default(0)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  equipment   Equipment[]
  sensors     Sensor[]

  @@unique([name, type])
  @@map("categories")
  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 2. 장비 (시스템 단위: USV, AUV 등)
// ─────────────────────────────────────────────

model Equipment {
  id           String      @id @default(cuid())
  categoryId   String
  name         String                          // "USV-Alpha", "AUV-Bravo"
  serialNumber String      @unique
  manufacturer String?                         // 제조사
  model        String?                         // 모델명
  acquiredAt   DateTime?                       // 취득일
  status       AssetStatus @default(AVAILABLE)
  description  String?
  imageUrl     String?                         // 대표 이미지
  metadata     Json?                           // 추가 스펙 정보 (무게, 크기, 최대 잠수 깊이 등)
  createdBy    String
  createdAt    DateTime    @default(now())
  updatedAt    DateTime    @updatedAt

  category            Category              @relation(fields: [categoryId], references: [id])
  components          EquipmentComponent[]
  maintenanceRecords  MaintenanceRecord[]
  schedules           AssetSchedule[]
  sensorCompatibility SensorCompatibility[]
  deployments         Deployment[]

  @@map("equipment")
  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 2-1. 장비 구성요소 (엔진, GPS, 통신모뎀 등)
// ─────────────────────────────────────────────

model EquipmentComponent {
  id          String   @id @default(cuid())
  equipmentId String
  name        String                          // "메인 엔진", "GPS 모듈", "통신 모뎀"
  spec        String?                         // "Yamaha 115HP", "Trimble R12i"
  notes       String?                         // 비고
  sortOrder   Int      @default(0)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  equipment Equipment @relation(fields: [equipmentId], references: [id], onDelete: Cascade)

  @@index([equipmentId])
  @@map("equipment_components")
  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 3. 센서 (독립 자산 풀)
// ─────────────────────────────────────────────

model Sensor {
  id               String      @id @default(cuid())
  categoryId       String
  name             String                          // "ADCP-001", "MBES-R2Sonic-2020"
  serialNumber     String      @unique
  manufacturer     String?
  model            String?
  acquiredAt       DateTime?
  status           AssetStatus @default(AVAILABLE)
  description      String?
  // 교정 관련
  calibrationIntervalDays Int?                     // 교정 주기 (일)
  lastCalibratedAt DateTime?                       // 최근 교정일
  nextCalibrationDue DateTime?                     // 다음 교정 예정일 (자동 계산)
  // 현재 위치 추적
  currentLocation  String?     @default("창고")    // "창고", "정비센터" 등
  currentEquipmentId String?                       // 현재 장착된 장비 ID (null이면 미장착)
  currentDeploymentId String?                      // 현재 투입 구성 ID
  metadata         Json?
  createdBy        String
  createdAt        DateTime    @default(now())
  updatedAt        DateTime    @updatedAt

  category             Category  @relation(fields: [categoryId], references: [id])
  maintenanceRecords   MaintenanceRecord[]
  schedules            AssetSchedule[]
  sensorCompatibility  SensorCompatibility[]
  deploymentSensors    DeploymentSensor[]

  @@map("sensors")
  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 4. 장비-센서 호환성 (N:M)
// ─────────────────────────────────────────────

model SensorCompatibility {
  id           String   @id @default(cuid())
  equipmentId  String
  sensorId     String
  notes        String?                           // "슬롯 A에 장착", "펌웨어 v2.0 이상 필요"
  createdAt    DateTime @default(now())

  equipment Equipment @relation(fields: [equipmentId], references: [id], onDelete: Cascade)
  sensor    Sensor    @relation(fields: [sensorId], references: [id], onDelete: Cascade)

  @@unique([equipmentId, sensorId])
  @@map("sensor_compatibility")
  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 5. 정비 이력 (장비 + 센서 공용, 다형성)
// ─────────────────────────────────────────────

model MaintenanceRecord {
  id             String          @id @default(cuid())
  // 다형성: 장비 또는 센서 중 하나만 참조
  equipmentId    String?
  sensorId       String?
  type           MaintenanceType
  title          String
  description    String?
  performedBy    String?                         // 수행자 이름 또는 userId
  performedAt    DateTime                        // 정비 수행일
  completedAt    DateTime?                       // 정비 완료일
  cost           Decimal?        @db.Decimal(12, 2)
  durationHours  Float?                          // 소요 시간
  replacedParts  String?                         // 교체 부품 설명
  attachmentUrls Json?                           // 첨부파일 URL 배열
  notes          String?
  createdBy      String
  createdAt      DateTime        @default(now())
  updatedAt      DateTime        @updatedAt

  equipment Equipment? @relation(fields: [equipmentId], references: [id], onDelete: Cascade)
  sensor    Sensor?    @relation(fields: [sensorId], references: [id], onDelete: Cascade)

  @@index([equipmentId])
  @@index([sensorId])
  @@map("maintenance_records")
  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 6. 운영 일정 (장비 + 센서 공용)
// ─────────────────────────────────────────────

model AssetSchedule {
  id            String       @id @default(cuid())
  // 다형성: 장비 또는 센서 중 하나만 참조
  equipmentId   String?
  sensorId      String?
  type          ScheduleType
  title         String
  description   String?
  startDate     DateTime
  endDate       DateTime
  projectId     String?                          // 프로젝트 투입인 경우
  projectName   String?                          // 비정규화: 빠른 표시용
  deploymentId  String?                          // 연결된 투입 구성 ID
  createdBy     String
  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt

  equipment Equipment? @relation(fields: [equipmentId], references: [id], onDelete: Cascade)
  sensor    Sensor?    @relation(fields: [sensorId], references: [id], onDelete: Cascade)

  @@index([equipmentId, startDate, endDate])
  @@index([sensorId, startDate, endDate])
  @@index([projectId])
  @@map("asset_schedules")
  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 7. 투입 구성 (Deployment Configuration)
// ─────────────────────────────────────────────

model Deployment {
  id            String           @id @default(cuid())
  equipmentId   String?                          // ← v1.3: optional (센서 단독 투입 지원)
  projectId     String                           // project-service 프로젝트 ID
  projectName   String                           // 비정규화
  taskId        String?                          // project-service 태스크 ID (참조)
  taskName      String?                          // 비정규화
  status        DeploymentStatus @default(PLANNED)
  startDate     DateTime
  endDate       DateTime?
  configParams  Json?                            // 장비 전체 설정 파라미터
  notes         String?
  createdBy     String
  createdAt     DateTime         @default(now())
  updatedAt     DateTime         @updatedAt

  equipment Equipment?         @relation(fields: [equipmentId], references: [id])
  sensors   DeploymentSensor[]

  @@index([equipmentId])
  @@index([projectId])
  @@index([taskId])
  @@map("deployments")
  @@schema("equipment")
}

// 투입 시 장착된 센서 (체크아웃 기록)
model DeploymentSensor {
  id            String    @id @default(cuid())
  deploymentId  String
  sensorId      String
  configParams  Json?                            // 센서별 설정 파라미터
  checkedOutAt  DateTime  @default(now())        // 체크아웃 시각
  returnedAt    DateTime?                        // 반납 시각 (null이면 아직 투입중)
  notes         String?

  deployment Deployment @relation(fields: [deploymentId], references: [id], onDelete: Cascade)
  sensor     Sensor     @relation(fields: [sensorId], references: [id])

  @@index([deploymentId])
  @@index([sensorId])
  @@map("deployment_sensors")
  @@schema("equipment")
}

// ─────────────────────────────────────────────
// 8. 투입 구성 템플릿
// ─────────────────────────────────────────────

model DeploymentTemplate {
  id            String   @id @default(cuid())
  name          String                           // "표준 수심측량 구성"
  description   String?
  categoryId    String?                          // 대상 장비 카테고리
  sensorConfig  Json                             // [{sensorCategoryId, configParams}]
  isPublic      Boolean  @default(false)         // 조직 공용 여부
  createdBy     String
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  @@map("deployment_templates")
  @@schema("equipment")
}
```

## 3. API 설계 (상세)

### 3.1 장비 API

| Method | Path | Description | Auth |
|--------|------|-------------|------|
| GET | `/api/v1/equipment` | 장비 목록 (필터: categoryId, status, search) | USER |
| GET | `/api/v1/equipment/:id` | 장비 상세 | USER |
| POST | `/api/v1/equipment` | 장비 등록 | ADMIN |
| PUT | `/api/v1/equipment/:id` | 장비 수정 | ADMIN |
| PATCH | `/api/v1/equipment/:id/status` | 장비 상태 변경 | ADMIN |
| DELETE | `/api/v1/equipment/:id` | 장비 삭제 (soft: RETIRED) | ADMIN |
| GET | `/api/v1/equipment/:id/components` | 장비 구성요소 목록 | USER |
| POST | `/api/v1/equipment/:id/components` | 구성요소 추가 | ADMIN |
| PUT | `/api/v1/equipment/components/:compId` | 구성요소 수정 | ADMIN |
| DELETE | `/api/v1/equipment/components/:compId` | 구성요소 삭제 | ADMIN |

```

```

```

   b. DeploymentSensor.returnedAt = now()
   c. Sensor.status = AVAILABLE, currentEquipmentId = null, currentDeploymentId = null
   d. Sensor.currentLocation = "창고"
   e. AssetSchedule 업데이트 (endDate 확정)
2. 트랜잭션 커밋
3. RabbitMQ 이벤트 발행: "equipment.returned"
```

## 4. 핵심 비즈니스 로직

### 4.1 일정 충돌 감지 (이중화)

충돌 감지는 **순수 날짜 겹침 검사**로 단순화 — 모든 일정 타입 간 겹침을 충돌로 판정.

**ScheduleService** (수동 일정 등록 시):
```typescript
// schedule.service.ts — checkConflicts()
private async checkConflicts(
  equipmentId: string | null, sensorId: string | null,
  startDate: string, endDate: string, excludeId?: string,
) {
  const where: any = { startDate: { lt: end }, endDate: { gt: start } };
  if (equipmentId) where.equipmentId = equipmentId;
  if (sensorId) where.sensorId = sensorId;
  if (excludeId) where.id = { not: excludeId };
  return this.prisma.assetSchedule.findMany({ where });
}
```

**DeploymentService** (프로젝트 투입 시):
```typescript
// deployment.service.ts — create() 내부
// Step 2: 장비 일정 충돌 검사 (equipmentId 존재 시)
if (data.equipmentId) {
  const conflicts = await this.prisma.assetSchedule.findMany({
    where: { equipmentId: data.equipmentId, startDate: { lt: endDt }, endDate: { gt: startDt } },
  });
  if (conflicts.length > 0) throw new Error(`장비 일정 충돌: ${names}`);
}

// Step 3: 각 센서별 일정 충돌 검사
for (const s of data.sensors) {
  const conflicts = await this.prisma.assetSchedule.findMany({
    where: { sensorId: s.sensorId, startDate: { lt: endDt }, endDate: { gt: startDt } },
  });
  if (conflicts.length > 0) throw new Error(`센서 일정 충돌: ${name}`);
}
```

> **설계 결정**: NON_CONFLICT_PAIRS(타입별 충돌 예외) 제거. 모든 겹침은 충돌. 단순 명확.

### 4.2 센서 상태 전이 (FSM)

```
                    ┌──────────┐
          등록 ────→│ AVAILABLE │←────── 반납 (return)
                    └────┬─────┘        정비 완료
                         │
           ┌─────────────┼─────────────┐
           ▼             ▼             ▼
    ┌──────────┐  ┌──────────────┐  ┌────────┐
    │ DEPLOYED │  │IN_MAINTENANCE│  │ BROKEN │
    └──────────┘  └──────────────┘  └────────┘
     (체크아웃)      (정비 접수)     (고장 보고)
                                        │
                                        ▼
                                   ┌─────────┐
                                   │ RETIRED │
                                   └─────────┘
                                    (퇴역 처리)
```

**유효한 전이**:
- AVAILABLE → DEPLOYED (체크아웃)
- AVAILABLE → IN_MAINTENANCE (정비 접수)
- AVAILABLE → BROKEN (고장 보고)
- DEPLOYED → AVAILABLE (반납)
- DEPLOYED → BROKEN (투입 중 고장)
- IN_MAINTENANCE → AVAILABLE (정비 완료)
- IN_MAINTENANCE → BROKEN (정비 중 추가 고장 발견)
- BROKEN → IN_MAINTENANCE (수리 접수)
- BROKEN → RETIRED (폐기)
- \* → RETIRED (어떤 상태에서든 퇴역 가능)

### 4.3 교정 만료 검사 (Cron)

```typescript
// 매일 09:00 실행
async checkCalibrationDue() {
  const sensors = await this.sensorRepo.findCalibrationDueSoon(30); // D-30 이내

  for (const sensor of sensors) {
    const daysRemaining = differenceInDays(sensor.nextCalibrationDue, new Date());

    if (daysRemaining <= 0) {
      // 만료됨 — 긴급 알림
      await this.notify('CALIBRATION_EXPIRED', sensor);
    } else if (daysRemaining <= 7) {
      // D-7 경고
      await this.notify('CALIBRATION_DUE_SOON', sensor, { daysRemaining });
    } else if (daysRemaining <= 30) {
      // D-30 사전 알림
      await this.notify('CALIBRATION_APPROACHING', sensor, { daysRemaining });
    }
  }
}
```
