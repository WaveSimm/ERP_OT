## 1. 개요

### 1.1 설계 목표

1. **단순 모델 + 풍부 표현**: 단일 row + JSON recurrence로 단발/반복 모두 표현
2. **충돌 자동 차단**: 같은 자원 시간 겹침 등록 시 transactional 검사로 차단
3. **시간대 일관**: UTC 저장, KST 표시 — 회사달력 패턴 재활용
4. **휴일 인지·자율**: 회사달력 휴일은 시각만 표시, 예약 자체는 무제한
5. **권한 명확**: VIEWER 조회 / OPERATOR 본인 / ADMIN·MANAGER 모든 예약
6. **자원 격리**: EquipmentResource 삭제 시 RESTRICT (활성 예약 있으면 차단)

### 1.2 설계 원칙

- **Single Row Recurrence**: 반복 시리즈는 1 row + JSON. 캘린더 표시 시점에 가상 전개
- **Exception via Sibling Row**: 부분 취소/수정은 `recurrenceParentId` 동일한 별도 row (recurrence=null)
- **Soft Delete**: 즉시 DELETE 안 함. `status=CANCELED` 갱신 (이력 보존)
- **Server-side Conflict Check**: UI 검증은 UX, 충돌 차단은 서버 트랜잭션이 SSOT
- **Holidays Visual Only**: 휴일 데이터는 회사달력 internal API에서 가져와 시각만 사용

### 2.2 데이터 흐름 — 단발 예약 등록

```
OPERATOR → POST /api/v1/equipment-reservations
  ├─ 1. authenticate hook + requireRole("OPERATOR","MANAGER","ADMIN")
  ├─ 2. Zod 검증 (resourceId, startAt < endAt, recurrence 형식, ...)
  ├─ 3. EquipmentResource 존재·isActive 확인
  ├─ 4. transaction:
  │     ├─ a. findConflicts(resourceId, startAt, endAt, excludeId=null)
  │     ├─ b. 충돌 있으면 throw RESERVATION_CONFLICT
  │     └─ c. INSERT (status=CONFIRMED)
  └─ 5. 201 → 객체 반환
```

### 2.3 데이터 흐름 — 반복 예약 등록

```
OPERATOR → POST + recurrence: { freq:"WEEKLY", byWeekday:["MON","WED"], until:"2026-12-31" }
  ├─ 1~3. 동
  ├─ 4. transaction:
  │     ├─ a. expand(recurrence, [startAt, until])  → [instance1, instance2, ...]
  │     ├─ b. for each instance: findConflicts (resourceId, instance.startAt, instance.endAt)
  │     ├─ c. 하나라도 충돌이면 throw RESERVATION_CONFLICT (충돌 인스턴스 일자 응답에 포함)
  │     └─ d. INSERT 단일 row (recurrence JSON)
  └─ 5. 201
```

### 2.4 데이터 흐름 — 조회 (반복 가상 전개)

```
GET /equipment-reservations?from=2026-05-01&to=2026-05-31&resourceId=...
  ├─ 1. authenticate
  ├─ 2. SELECT WHERE status=CONFIRMED AND (
  │       (recurrence IS NULL AND endAt >= from AND startAt <= to)
  │     OR (recurrence IS NOT NULL AND startAt <= to AND (until IS NULL OR until >= from))
  │     )
  ├─ 3. 단발 row → 그대로 반환
  ├─ 4. 반복 row → expand(rec, [from, to]) → 인스턴스 배열
  ├─ 5. 같은 recurrenceParentId의 예외 row 적용 (인스턴스 덮어쓰기/삭제)
  └─ 6. 200 → flat 인스턴스 배열
```

## 3. 데이터 모델

### 3.1 Prisma 스키마 (services/project/prisma/schema.prisma 추가분)

```prisma
model EquipmentReservation {
  id                  String              @id @default(cuid())
  resourceId          String              @map("resource_id")
  userId              String              @map("user_id")           // auth_users.id
  title               String                                         // 사유
  description         String?
  startAt             DateTime            @map("start_at")           // UTC
  endAt               DateTime            @map("end_at")
  isAllDay            Boolean             @default(false) @map("is_all_day")

  // 반복 (null이면 단발)
  recurrence          Json?                                          // {freq, interval?, byWeekday?, until?, count?}
  recurrenceParentId  String?             @map("recurrence_parent_id")  // 시리즈 첫 인스턴스 id (예외 row)

  status              ReservationStatus   @default(CONFIRMED)        // CONFIRMED | CANCELED
  cancelReason        String?             @map("cancel_reason")
  canceledAt          DateTime?           @map("canceled_at")
  canceledBy          String?             @map("canceled_by")

  createdAt           DateTime            @default(now()) @map("created_at")
  updatedAt           DateTime            @updatedAt @map("updated_at")

  resource            EquipmentResource   @relation(fields: [resourceId], references: [id], onDelete: Restrict)

  @@index([resourceId, startAt])              // 충돌 검사 핫패스
  @@index([userId, startAt(sort: Desc)])      // 내 예약 목록
  @@index([status])                            // 활성 필터
  @@index([recurrenceParentId])               // 예외 row 조회
  @@map("equipment_reservations")
  @@schema("project")
}

enum ReservationStatus {
  CONFIRMED
  CANCELED

  @@schema("project")
}
```

### 3.3 핵심 인덱스

### 3.4 Zod 검증

```ts
// recurrence JSON
const recurrenceSchema = z.object({
  freq: z.enum(["DAILY", "WEEKLY", "MONTHLY"]),
  interval: z.number().int().min(1).max(52).optional(),       // 매 N (기본 1)
  byWeekday: z.array(z.enum(["MON","TUE","WED","THU","FRI","SAT","SUN"])).optional(),  // WEEKLY 전용
  until: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(),  // 종료일
  count: z.number().int().min(1).max(366).optional(),         // 횟수 (until과 둘 중 하나만)
}).refine(
  (r) => !!r.until !== !!r.count || (!r.until && !r.count),    // until/count 동시 X (둘 다 없으면 무한 — 정책: count 또는 until 필수)
  { message: "반복은 종료일 또는 횟수 중 하나가 필요합니다." },
);

// create
const createSchema = z.object({
  resourceId: z.string().cuid(),
  title: z.string().min(1).max(200),
  description: z.string().max(1000).nullable().optional(),
  startAt: z.string().datetime(),                              // ISO 8601 (UTC)
  endAt: z.string().datetime(),
  isAllDay: z.boolean().default(false),
  recurrence: recurrenceSchema.nullable().optional(),
}).refine(
  (d) => new Date(d.startAt) < new Date(d.endAt),
  { message: "종료시각은 시작시각보다 늦어야 합니다.", path: ["endAt"] },
).refine(
  (d) => {
    // 30분 정밀도 검증
    const start = new Date(d.startAt);
    const end = new Date(d.endAt);
    return start.getUTCMinutes() % 30 === 0 && end.getUTCMinutes() % 30 === 0
        && start.getUTCSeconds() === 0 && end.getUTCSeconds() === 0;
  },
  { message: "시간은 30분 단위여야 합니다." },
);
```

### 4.2 GET `/equipment-reservations` 상세

**Query Params**:
| 파라미터 | 타입 | 기본 | 설명 |
|---|---|---|---|
| `from` | YYYY-MM-DD | 당월 1일 | 조회 시작 |
| `to` | YYYY-MM-DD | 당월 말일 | 조회 끝 |
| `resourceId` | cuid | — | 자원 필터 (옵션) |
| `userId` | string | — | 예약자 필터 (옵션) |

**Response (200)** — 인스턴스 평탄화 배열:
```json
[
  {
    "id": "ckxx...",                       // 단발이면 own id, 반복 인스턴스면 parent id
    "instanceKey": "ckxx...:2026-05-10",   // 반복 시리즈 안에서 유니크 (parent id + 시작일)
    "resourceId": "...",
    "resourceName": "대회의실",
    "userId": "...",
    "userName": "김OO",
    "title": "분기 임원회의",
    "description": null,
    "startAt": "2026-05-10T05:00:00.000Z",  // UTC
    "endAt": "2026-05-10T07:00:00.000Z",
    "isAllDay": false,
    "isRecurring": true,
    "isException": false,                  // 시리즈 예외로 변경된 인스턴스
    "recurrenceSummary": "매주 월 (~2026-12-31)",  // 표시용
    "status": "CONFIRMED"
  }
]
```

정렬: `startAt asc, resourceId asc`.

### 4.3 POST `/equipment-reservations` 상세

**Request**:
```json
{
  "resourceId": "...",
  "title": "분기 임원회의",
  "description": "Q2 검토",
  "startAt": "2026-05-10T05:00:00.000Z",
  "endAt": "2026-05-10T07:00:00.000Z",
  "isAllDay": false,
  "recurrence": {
    "freq": "WEEKLY",
    "byWeekday": ["MON"],
    "until": "2026-12-31"
  }
}
```

**Response (201)**: 생성된 row.

**충돌 응답 (409)**:
```json
{
  "error": {
    "code": "RESERVATION_CONFLICT",
    "message": "5/10 14:00~16:00 대회의실은 이미 예약되어 있습니다.",
    "details": {
      "conflicts": [
        {
          "instanceKey": "ck...:2026-05-10",
          "startAt": "2026-05-10T05:00:00.000Z",
          "endAt": "2026-05-10T07:00:00.000Z",
          "title": "타 회의",
          "userName": "이OO"
        }
      ]
    }
  }
}
```

### 4.4 PATCH `/equipment-reservations/:id`

**Query**:
| 파라미터 | 값 | 설명 |
|---|---|---|
| `scope` | `instance` (기본) / `series` | 반복 시리즈에서 인스턴스 한 회만 vs 전체 |

**시리즈 부분 수정 흐름** (`scope=instance`):
1. parent row를 그대로 두고
2. 새 row 생성 (recurrence=null, recurrenceParentId=parent, isException=암묵)
3. 충돌 검사

**시리즈 전체 수정** (`scope=series`):
1. parent row 갱신 (충돌 검사 — 갱신된 모든 인스턴스)
2. 기존 예외 row는 유지

### 4.5 DELETE `/equipment-reservations/:id` (취소)

- `scope=instance`: 그 인스턴스만 취소 row(`status=CANCELED`) 추가 (recurrenceParentId)
- `scope=series`: parent row의 `status=CANCELED` (모든 인스턴스 사라짐) — 예외 row도 일괄 CANCELED
- soft delete — 이력 보존

```
  자원관리 → 공용자산 예약 탭 → [+ 예약 추가]
  → 자원 선택 + 날짜·시간 + 사유 → 저장
  → 충돌 시 에러 (예약자·시간·사유 노출)
  → 정상 등록 → 캘린더 즉시 반영

[B] 반복 예약
  추가 모달 → ☑ 반복 → 빈도/요일/종료 → 저장
  → 모든 인스턴스 충돌 검사 → 충돌 1건이라도 있으면 에러 (어느 인스턴스인지 안내)
  → 정상 등록 → 캘린더에 모든 인스턴스 표시

[C] 부분 취소
  반복 인스턴스 클릭 → 상세 popover → [취소]
  → "이 인스턴스만 / 시리즈 전체" 선택
  → 본인이거나 ADMIN/MANAGER만 가능
  → status=CANCELED + 이력

[D] 다른 사람 예약 수정 (관리자)
  ADMIN/MANAGER가 캘린더 셀 클릭 → 수정 가능
  → "관리자 권한으로 변경" 표시 (감사 로그)

[E] 충돌 응답 화면
  사용자: "5/10 14:00~16:00 대회의실"
  서버: { code: RESERVATION_CONFLICT, conflicts: [...] }
  UI: 모달 상단에 빨간 안내
      "이미 예약: 5/10 14:00~16:00 / 김OO / Q1 회고"
```

## 7. 에러 처리

### 7.1 에러 코드

| HTTP | Code | 메시지 | 원인 |
|---|---|---|---|
| 400 | INVALID_INPUT | 입력값 오류 | Zod 위반 |
| 400 | INVALID_TIME_RANGE | 종료시각은 시작시각보다 늦어야 합니다 | startAt >= endAt |
| 400 | INVALID_TIME_PRECISION | 시간은 30분 단위여야 합니다 | 분 % 30 ≠ 0 |
| 400 | INVALID_RECURRENCE | 반복 패턴이 올바르지 않습니다 | until/count 동시 또는 byWeekday 잘못 |
| 401 | UNAUTHENTICATED | 인증 필요 | 토큰 없음 |
| 403 | RESERVATION_FORBIDDEN | 본인 예약만 변경할 수 있습니다 | OPERATOR가 타인 예약 PATCH/DELETE |
| 404 | RESOURCE_NOT_FOUND | 자원을 찾을 수 없습니다 | resourceId 오류 |
| 404 | RESERVATION_NOT_FOUND | 예약을 찾을 수 없습니다 | id 오류 |
| 409 | RESERVATION_CONFLICT | 시간 겹치는 예약이 있습니다 | 충돌 검사 |
| 422 | RESOURCE_INACTIVE | 비활성 자원입니다 | EquipmentResource.isActive=false |
| 500 | INTERNAL_ERROR | 서버 오류 | 예외 |

### 7.2 응답 포맷

기존 SSOT 따름:
```json
{
  "error": {
    "code": "RESERVATION_CONFLICT",
    "message": "...",
    "details": { "conflicts": [...] }
  }
}
```

## 8. 보안

- [x] **인증**: 모든 엔드포인트 `authenticate` hook
- [x] **권한**: 등록은 OPERATOR↑, 수정·취소는 본인 또는 ADMIN/MANAGER
- [x] **CSRF**: state-changing 메서드는 X-CSRF-Token 헤더 검증 (Layer 5)
- [x] **XSS**: title/description plain text, HTML 입력 sanitize (DTO 단계)
- [x] **SQL Injection**: Prisma ORM
- [x] **권한 우회 방지**: 서비스 레이어가 단일 진입점에서 본인/관리자 체크 (route preHandler 단독 의존 금지)
- [x] **레이스 컨디션**: 충돌 검사 + INSERT를 단일 transaction에 묶음 (READ COMMITTED 충분, SERIALIZABLE 미요구 — 같은 자원 동시 등록은 매우 드문 빈도)
- [x] **민감 정보**: 응답에 다른 사람의 description은 그대로 노출 (예약 사유는 회사 공유 정보로 간주). 추후 정책 변경 시 가림 처리

## 12. 마이그레이션 / 롤백

### 12.1 마이그레이션 (요약 SQL)

```sql
-- 1. enum
CREATE TYPE project."ReservationStatus" AS ENUM ('CONFIRMED', 'CANCELED');

-- 2. 테이블
CREATE TABLE project.equipment_reservations (
    "id" TEXT NOT NULL PRIMARY KEY,
    "resource_id" TEXT NOT NULL,
    "user_id" TEXT NOT NULL,
    "title" TEXT NOT NULL,
    "description" TEXT,
    "start_at" TIMESTAMP(3) NOT NULL,
    "end_at" TIMESTAMP(3) NOT NULL,
    "is_all_day" BOOLEAN NOT NULL DEFAULT false,
    "recurrence" JSONB,
    "recurrence_parent_id" TEXT,
    "status" project."ReservationStatus" NOT NULL DEFAULT 'CONFIRMED',
    "cancel_reason" TEXT,
    "canceled_at" TIMESTAMP(3),
    "canceled_by" TEXT,
    "created_at" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "updated_at" TIMESTAMP(3) NOT NULL,

    CONSTRAINT "equipment_reservations_resource_fkey" FOREIGN KEY ("resource_id")
      REFERENCES project.equipment_resources(id) ON UPDATE CASCADE ON DELETE RESTRICT
);

-- 3. 인덱스
CREATE INDEX "equipment_reservations_resource_start_idx"
  ON project.equipment_reservations("resource_id", "start_at");
CREATE INDEX "equipment_reservations_user_start_idx"
  ON project.equipment_reservations("user_id", "start_at" DESC);
CREATE INDEX "equipment_reservations_status_idx"
  ON project.equipment_reservations("status");
CREATE INDEX "equipment_reservations_recurrence_parent_idx"
  ON project.equipment_reservations("recurrence_parent_id");
```
