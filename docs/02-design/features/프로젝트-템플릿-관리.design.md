# 프로젝트-템플릿-관리 설계

> 작성일: 2026-06-24
> 대상: `services/project` (3003) + `apps/web`

## 1. 백엔드

### 1.1 버그 수정 — 템플릿 인스턴스화 XOR 위반
`template.service.ts instantiate()`가 `segmentAssignment.create` 시 legacy `resourceId`만 채워 DB의 `segment_assignments_xor_check`(personUserId/externalPersonId/equipmentResourceId 중 하나 NOT NULL) 위반 → 500.
**수정**: `resourceId`를 장비(EquipmentResource)→외부(ExternalPerson)→직원(personUserId 폴백) 순으로 판별해 알맞은 polymorphic 컬럼을 함께 채움.

### 1.2 기존 API 재사용 (변경 없음)
- `GET /templates`, `GET /templates/:id`
- `POST /projects/:id/save-as-template` (저장)
- `POST /templates/:id/instantiate` (적용=새 프로젝트)
- `PATCH /templates/:id` (수정), `DELETE /templates/:id` (삭제) — 프론트 클라이언트에 노출만 추가

## 2. 프론트 (`apps/web`)

### 2.1 API 클라이언트 (`core-domains.ts` templateApi)
- 추가: `update(id, {name?,category?,...})` → PATCH, `delete(id)` → DELETE

### 2.2 프로젝트 목록 (`projects/page.tsx`)
- 툴바: `검색 · 💾 템플릿저장 · ⚙️ 템플릿관리 · 📁 새폴더 · + 새 프로젝트`
  - 폴더 라벨 "폴더"→"새폴더"
- **템플릿저장 모달**: 원본 프로젝트 select + 이름 + 카테고리 + 자원배정 포함 → `saveAsTemplate`
- **새 프로젝트 모달**: 상단에 "📋 템플릿으로 만들기" → `TemplateWizard`(projectId 미사용, instantiate로 새 프로젝트 생성). 아래는 빈 프로젝트 폼.
- **템플릿관리 모달**(`TemplateManagerModal.tsx`): 목록 표(이름·카테고리 인라인수정 / 태스크수·사용수 / 추천★ 토글 / 생성일 / 삭제🗑)

### 2.3 상세 화면 (`projects/[id]/page.tsx`)
- 템플릿 적용/저장 버튼 + TemplateWizard 렌더 + 저장 모달 + 관련 state/import **제거**.

## 3. 권한
- 저장/관리 버튼은 `isManager`(ADMIN/MANAGER)만 노출. 백엔드 라우트도 동일 role 가드.

## 4. 엣지케이스
- 템플릿 0개: 관리 모달 "저장된 템플릿이 없습니다".
- 인라인 수정 빈값: 무시 후 재로드.
- 삭제: confirm 후 진행.
