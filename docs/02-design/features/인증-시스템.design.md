## 1. 아키텍처 개요

```
[Browser]
    │
    ├─ POST /api/auth/login          → Next.js route (auth-service 프록시)
    ├─ POST /api/auth/refresh        → Next.js route (auth-service 프록시)
    ├─ GET/POST /api/v1/auth/*       → auth-service:3001
    ├─ GET/POST /api/v1/users/*      → auth-service:3001
    └─ 그 외 /api/v1/*              → project-service:3003 (기존)
```

### 토큰 흐름
```
로그인 요청 (email + password)
    │
    ▼
auth-service: bcrypt 검증
    │
    ├─ Access Token  (JWT, 8시간, Authorization: Bearer)
    └─ Refresh Token (JWT, 7일, httpOnly 쿠키 + DB 저장)

Access Token 만료 시:
    POST /api/auth/refresh (쿠키의 refresh token)
    → 새 Access Token 발급 + Refresh Token 로테이션
```

## 2. 데이터 모델

### 기존 Prisma 스키마 활용 (services/auth/prisma/schema.prisma)

```prisma
model User {
  id           String    @id @default(cuid())
  email        String    @unique
  name         String                          // ← 추가 필요
  passwordHash String    @map("password_hash")
  role         UserRole  @default(OPERATOR)
  isActive     Boolean   @default(true)
  lastLoginAt  DateTime?
  createdAt    DateTime  @default(now())
  updatedAt    DateTime  @updatedAt

  refreshTokens RefreshToken[]

  @@map("auth_users")
}

model RefreshToken {
  id        String   @id @default(cuid())
  token     String   @unique                  // JWT string (해시 저장 권장)
  userId    String
  expiresAt DateTime
  createdAt DateTime @default(now())

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId])
  @@map("auth_refresh_tokens")
}

enum UserRole {
  ADMIN
  MANAGER
  OPERATOR
  VIEWER
}
```

> **변경사항**: `User`에 `name String` 필드 추가 필요

## 4. API 명세

### 인증 엔드포인트

#### POST /api/v1/auth/login
```typescript
// Request
{ email: string; password: string }

// Response 200
{
  accessToken: string;
  user: { id: string; email: string; name: string; role: UserRole };
}
// Set-Cookie: refreshToken=<jwt>; HttpOnly; Path=/api/auth/refresh; SameSite=Strict

// Error 401: 이메일/비밀번호 불일치
// Error 403: 비활성화된 계정
```

#### POST /api/v1/auth/refresh
```typescript
// Cookie: refreshToken=<jwt>
// Response 200
{ accessToken: string }
// Set-Cookie: refreshToken=<new_jwt>; HttpOnly (토큰 로테이션)

// Error 401: 유효하지 않은 refresh token
```

#### POST /api/v1/auth/logout
```typescript
// Authorization: Bearer <accessToken>
// DB에서 refresh token 삭제
// Set-Cookie: refreshToken=; Max-Age=0
// Response 204
```

#### GET /api/v1/auth/me
```typescript
// Authorization: Bearer <accessToken>
// Response 200
{ id: string; email: string; name: string; role: UserRole; lastLoginAt: string | null }
```

#### PATCH /api/v1/auth/me/password
```typescript
// Request
{ currentPassword: string; newPassword: string }
// Response 204

// Error 400: 현재 비밀번호 불일치
// Error 422: 새 비밀번호 8자 미만
```

## 5. 핵심 로직

### auth.service.ts

```typescript
class AuthService {
  async login(email, password) {
    const user = await userRepo.findByEmail(email);
    if (!user || !user.isActive) throw Unauthorized();

    const valid = await bcrypt.compare(password, user.passwordHash);
    if (!valid) throw Unauthorized();

    const accessToken = jwt.sign(
      { sub: user.id, role: user.role, name: user.name },
      ACCESS_SECRET, { expiresIn: "8h" }
    );
    const refreshToken = jwt.sign(
      { sub: user.id },
      REFRESH_SECRET, { expiresIn: "7d" }
    );

    // DB에 refresh token 저장 (기존 토큰 삭제 후 신규 저장)
    await prisma.refreshToken.deleteMany({ where: { userId: user.id } });
    await prisma.refreshToken.create({
      data: { token: refreshToken, userId: user.id,
              expiresAt: new Date(Date.now() + 7 * 86400000) }
    });

    await userRepo.updateLastLogin(user.id);
    return { accessToken, refreshToken, user };
  }

  async refresh(refreshToken) {
    const payload = jwt.verify(refreshToken, REFRESH_SECRET);
    const stored = await prisma.refreshToken.findUnique({ where: { token: refreshToken } });
    if (!stored || stored.expiresAt < new Date()) throw Unauthorized();

    // 토큰 로테이션
    const newAccessToken = jwt.sign({ sub: payload.sub, ... }, ACCESS_SECRET, { expiresIn: "8h" });
    const newRefreshToken = jwt.sign({ sub: payload.sub }, REFRESH_SECRET, { expiresIn: "7d" });

    await prisma.refreshToken.update({
      where: { token: refreshToken },
      data: { token: newRefreshToken, expiresAt: new Date(Date.now() + 7 * 86400000) }
    });

    return { accessToken: newAccessToken, refreshToken: newRefreshToken };
  }
}
```

## 6. Next.js 프록시 라우팅 수정

### apps/web/src/app/api/v1/[...path]/route.ts 수정

```typescript
const AUTH_SERVICE_URL = process.env.AUTH_SERVICE_URL ?? "http://localhost:3001";
const PROJECT_SERVICE_URL = process.env.API_BASE_URL ?? "http://localhost:3003";

function getTargetUrl(path: string[], search: string) {
  const prefix = path[0];
  if (prefix === "auth" || prefix === "users") {
    return `${AUTH_SERVICE_URL}/api/v1/${path.join("/")}${search}`;
  }
  return `${PROJECT_SERVICE_URL}/api/v1/${path.join("/")}${search}`;
}
```

### apps/web/src/app/api/auth/login/route.ts 교체

```typescript
// auth-service 실제 호출
const AUTH_SERVICE_URL = process.env.AUTH_SERVICE_URL ?? "http://localhost:3001";

export async function POST(req: NextRequest) {
  const body = await req.json();
  const res = await fetch(`${AUTH_SERVICE_URL}/api/v1/auth/login`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(body),
  });
  const data = await res.json();
  const response = NextResponse.json(data, { status: res.status });

  // refresh token 쿠키 전달
  const setCookie = res.headers.get("set-cookie");
  if (setCookie) response.headers.set("set-cookie", setCookie);

  return response;
}
```

## 9. Resource.userId 마이그레이션

현재: `userId = "user-심윤송"` (임의 문자열)
변경 후: `userId = "cm..."` (실제 User DB ID)

### 처리 방식
1. 사용자 계정 생성 시 → 관리자가 자원 관리 페이지에서 "계정 연결" 버튼으로 수동 매핑
2. (현재 이미 구현된 `handleOpenUserIdModal` 활용)
3. userId 형식이 `user-`로 시작하면 → 자원 관리 페이지에서 경고 표시
