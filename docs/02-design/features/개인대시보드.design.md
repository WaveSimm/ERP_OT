## 3. 출퇴근 상태 머신 (FSM)

```
                   ┌─────────────────────────────────────────────────┐
                   │            출퇴근 상태 머신                       │
                   └─────────────────────────────────────────────────┘

NOT_STARTED ──[checkIn]──► CHECKED_IN ──[breakOut]──► ON_BREAK
    │                          │                          │
    │                          │ [checkOut]               │ [breakIn]
    │                          ▼                          ▼
    │                     CHECKED_OUT              CHECKED_IN
    │                                             (checkState = RESUMED)
    │                                                  │
    │                                                  │ [checkOut]
    │                                                  ▼
    │                                             CHECKED_OUT
    │
    └──[자정 경과]──► ABSENT (Cron Job 자동 처리)
    └──[휴가일]   ──► LEAVE  (LeaveRequest APPROVED 기준)
    └──[공휴일]   ──► HOLIDAY (PublicHoliday 기준)

유효하지 않은 전이 (에러 반환):
  CHECKED_OUT → checkIn  (당일 재출근 불가, 수동 수정만 Admin 가능)
  NOT_STARTED → breakOut (출근 전 외출 불가)
  ON_BREAK    → checkOut (복귀 후 퇴근 필요)
  ON_BREAK    → breakOut (이중 외출 불가)
```

```typescript
// domain/state-machine/attendance.fsm.ts

type CheckState = 'NOT_STARTED' | 'CHECKED_IN' | 'ON_BREAK' | 'CHECKED_OUT';
type CheckAction = 'CHECK_IN' | 'BREAK_OUT' | 'BREAK_IN' | 'CHECK_OUT';

const VALID_TRANSITIONS: Record<CheckState, CheckAction[]> = {
  NOT_STARTED: ['CHECK_IN'],
  CHECKED_IN:  ['BREAK_OUT', 'CHECK_OUT'],
  ON_BREAK:    ['BREAK_IN'],
  CHECKED_OUT: [],              // 당일 재출근 불가
};

function transition(current: CheckState, action: CheckAction): CheckState {
  if (!VALID_TRANSITIONS[current].includes(action)) {
    throw new InvalidStateTransitionError(
      `상태 '${current}'에서 '${action}' 동작은 허용되지 않습니다.`
    );
  }
  const next: Record<string, CheckState> = {
    'NOT_STARTED:CHECK_IN':  'CHECKED_IN',
    'CHECKED_IN:BREAK_OUT':  'ON_BREAK',
    'ON_BREAK:BREAK_IN':     'CHECKED_IN',   // checkState는 RESUMED로 별도 저장
    'CHECKED_IN:CHECK_OUT':  'CHECKED_OUT',
  };
  return next[`${current}:${action}`];
}
```

## 4. 연차 계산 로직

### 4.1 부여 기준별 계산

```typescript
// application/leave.service.ts

class LeaveCalculator {
  /**
   * 입사일 기준 (HIRE_DATE): 매 입사일 기념일에 갱신
   * 회계연도 기준 (FISCAL_YEAR): 1월 1일 일괄 부여
   */
  calculateAnnualLeave(
    hireDate: Date,
    targetYear: number,
    policy: AttendancePolicy
  ): number {
    if (policy.leavePolicy === 'FISCAL_YEAR') {
      return this.calculateFiscalYear(hireDate, targetYear, policy);
    }
    return this.calculateHireDate(hireDate, targetYear, policy);
  }

  // 회계연도 기준: 매년 1/1 일괄 부여
  // 입사 첫해: 근속 개월 수 비례 (11개월 미만은 월 1일)
  private calculateFiscalYear(hireDate: Date, year: number, policy: AttendancePolicy): number {
    const yearsServed = year - hireDate.getFullYear();
    if (yearsServed <= 0) {
      // 입사 첫해: 입사월부터 12월까지 월 1일 (최대 11일)
      const monthsLeft = 12 - hireDate.getMonth(); // 0-indexed
      return Math.min(monthsLeft, 11);
    }
    if (yearsServed === 1) return policy.annualLeaveBase; // 15일 (기본)
    // 3년차부터 2년마다 +1일, 최대 25일
    const bonus = Math.floor((yearsServed - 1) / 2);
    return Math.min(policy.annualLeaveBase + bonus, 25);
  }

  // 입사일 기준: 입사 1년 미만은 매월 1일 선부여
  private calculateHireDate(hireDate: Date, year: number, policy: AttendancePolicy): number {
    const yearsServed = differenceInYears(new Date(year, 11, 31), hireDate);
    if (yearsServed < 1) {
      const monthsServed = differenceInMonths(new Date(year, 11, 31), hireDate);
      return Math.min(monthsServed, 11); // 최대 11일
    }
    const bonus = Math.floor((yearsServed - 1) / 2);
    return Math.min(policy.annualLeaveBase + bonus, 25);
  }

  // 반차 계산: HALF_AM / HALF_PM = 0.5일
  calculateLeaveDays(type: LeaveType, startDate: Date, endDate: Date): number {
    if (type === 'HALF_AM' || type === 'HALF_PM') return 0.5;
    // 연차/병가/특별휴가: 영업일 기준 (주말·공휴일 제외)
    return this.countBusinessDays(startDate, endDate);
  }

  private countBusinessDays(start: Date, end: Date): number {
    let count = 0;
    let cur = new Date(start);
    while (cur <= end) {
      const dayOfWeek = cur.getDay();
      const isWeekend = dayOfWeek === 0 || dayOfWeek === 6;
      const isHoliday = this.holidaySet.has(cur.toISOString().split('T')[0]);
      if (!isWeekend && !isHoliday) count++;
      cur.setDate(cur.getDate() + 1);
    }
    return count;
  }
}
```

### 4.2 잔여 연차 계산

```typescript
interface LeaveBalanceSummary {
  year: number;
  totalDays: number;          // 부여 일수
  adjustedDays: number;       // 관리자 조정
  usedDays: number;           // 사용 확정
  pendingDays: number;        // 승인 대기
  remainingDays: number;      // 실질 잔여 = total + adjusted - used - pending
  effectiveRemaining: number; // 신청 가능 잔여 = total + adjusted - used - pending
}

// 잔여 연차 초과 시 경고 (신청 시)
function validateLeaveBalance(
  balance: LeaveBalance,
  requestDays: number
): void {
  const effective = balance.totalDays + balance.adjustedDays
    - balance.usedDays - balance.pendingDays;
  if (requestDays > effective) {
    throw new ValidationError(
      `신청 일수(${requestDays}일)가 잔여 연차(${effective}일)를 초과합니다.`
    );
  }
}
```

## 6. 알림 통합 아키텍처

### 6.1 설계 방식: 단일 Notification 테이블 + Redis Pub/Sub

```
[알림 발행 흐름]

project-service                attendance-service
  │ (멘션, 마감임박, Stale)        │ (휴가승인, OT승인)
  │                               │
  ▼                               ▼
Redis Pub/Sub (channel: "notifications")
  │
  ▼
[WebSocket Gateway] (project-service 내 통합 또는 별도 gateway)
  │
  ├─ INSERT → attendance.notifications (영구 저장)
  │
  └─ socket.emit('notification', payload) → 클라이언트

[클라이언트]
  - 단일 WebSocket 연결 (project-service 게이트웨이)
  - 모든 서비스 알림 수신
  - Polling fallback: GET /api/v1/notifications/unread-count (30초마다)
```

### 6.3 attendance-service → Redis 발행 예시

```typescript
// application/notification.service.ts (attendance-service)

class NotificationService {
  async notifyLeaveResult(
    request: LeaveRequest,
    approved: boolean,
    rejectReason?: string
  ): Promise<void> {
    const payload: NotificationPayload = {
      userId: request.userId,
      type: approved ? 'LEAVE_APPROVED' : 'LEAVE_REJECTED',
      source: 'attendance',
      priority: approved ? 2 : 1,
      title: approved ? '휴가 승인' : '휴가 반려',
      body: approved
        ? `${request.startDate} ~ ${request.endDate} 휴가가 승인되었습니다.`
        : `휴가 신청이 반려되었습니다. 사유: ${rejectReason}`,
      linkUrl: '/me/attendance',
      metadata: { leaveRequestId: request.id },
    };

    // Redis Pub/Sub 발행
    await this.redis.publish('notifications', JSON.stringify(payload));

    // DB 저장 (영구 보관)
    await this.notificationRepo.create(payload);
  }
}
```
