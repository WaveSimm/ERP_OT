## 2. 데이터 모델

*(Plan v1.1과 동일 — 이미 구현 완료)*

```prisma
enum EntryType { WORK FIELD TRAINING BUSINESS_TRIP HALF_AM HALF_PM ANNUAL SICK SPECIAL OT }
enum EntrySource { MANUAL LEAVE_APPROVED OT_APPROVED }

model WorkScheduleEntry {
  id          String      @id @default(cuid())
  userId      String
  date        DateTime    @db.Date
  entryType   EntryType
  label       String?
  sourceType  EntrySource @default(MANUAL)
  sourceId    String?
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt

  @@unique([userId, date, entryType, sourceId])
  @@index([userId, date])
  @@index([date])
  @@index([sourceId])
  @@map("work_schedule_entries")
  @@schema("attendance")
}
```

### 5.2 내 대시보드 — 출퇴근 위젯 확장

**파일**: `apps/web/src/components/AttendanceView.tsx` (기존 파일 수정)

**변경 사항**:

1. **CheckInWidget 근무 유형 드롭다운 추가**:
   - 출근 버튼 누를 때 workType 외에 entryType도 선택
   - 기본값: WORK (사무실 출근)
   - 선택지: WORK / FIELD / TRAINING / BUSINESS_TRIP
   - 출근 시 자동으로 WorkScheduleEntry POST 호출

2. **"근태 추가" 버튼 추가**:
   - 출퇴근 위젯 하단에 배치
   - 클릭 시 모달: 날짜 선택 + 유형 드롭다운 + 메모
   - 과거/미래 날짜에 근태 등록 가능

3. **"내 이번 주 근태" 리스트 추가**:
   - 이번 주 내 WorkScheduleEntry 목록 표시
   - MANUAL 항목만 삭제 가능 (X 버튼)
   - 자동 생성 항목(휴가/OT)은 출처 표시만

### 5.4 API Client

**파일**: `apps/web/src/lib/api.ts`

```typescript
export const attendanceOverviewApi = {
  getWeekly: (start: string, end: string) =>
    request<any>(`/work-schedule?start=${start}&end=${end}`),
  createEntry: (data: { date: string; entryType: string; label?: string }) =>
    request<any>("/work-schedule", { method: "POST", body: JSON.stringify(data) }),
  deleteEntry: (id: string) =>
    request<void>(`/work-schedule/${id}`, { method: "DELETE" }),
};
```

## 8. 기술적 결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 근태현황 페이지 | 조회 전용 | 입력은 대시보드에서 — UX 책임 분리 |
| 입력 위치 | 내 대시보드 CheckInWidget 확장 | 기존 출퇴근 위젯과 자연스러운 통합 |
| API 변경 | 없음 | POST/DELETE 엔드포인트 유지, 호출 위치만 변경 |
| 근태 유형 선택 | 출퇴근 시 함께 선택 | 별도 단계 없이 원클릭 등록 |
