# 구매/회계 업무 플로우 및 ERP 구현 현황

> **작성일**: 2026-04-12
> **대상**: 사내 공유 (영업팀, 재무팀, 관리팀)
> **목적**: 현재 ERP 시스템의 구매/회계 모듈 구현 현황 및 향후 계획 공유

---

## 1. 업무 영역 구분

| 영역 | 담당 | 설명 |
|------|------|------|
| **해외 구매** | 영업팀 + 재무팀 | 해외 제조사 발주 → 선적 → 통관 → 입고 |
| **국내 구매** | 전 사원 | 지출결의서 → 부서 결재 → 재무팀 접수 → 입고 |
| **수입원가정산** | 재무팀 | 수입면장 기반 원가 산출 (송금액 + 관세/운송비) |
| **재고관리** | 전 부서 | 재고 현황, 입출고, 창고 이동, 추가비용 추적 |
| **전자결재** | 전 사원 | 지출결의서, 기안서, 발주서 등 18종 양식 |

---

## 2. 해외 구매 흐름 (영업팀 + 재무팀)

```
[1] 계약 등록 (영업팀)
 │  계약번호(#YY-NN), 사업명, 고객사 입력
 │
 ▼
[2] 해외 발주 생성 (영업팀)
 │  계약 선택 → 제조사, 품목, 통화(EUR/GBP/USD), 금액 입력
 │
 ▼
[3] 발주 확정 + 송금 (영업팀 + 재무팀)
 │  해외 제조사에 발주 전달, invoice/OA 번호 입력, 송금 처리
 │
 ▼
[4] 제작 트래킹 (영업팀)
 │  제조사와 소통하며 진행율(0~100%) 주기적 업데이트
 │  담당자 대시보드에서 전체 발주 현황 확인
 │
 ▼
[5] 선적 → 통관 (영업팀 + 재무팀)
 │  선적일 입력, 통관일/통관담당자 입력
 │  관세업체로부터 수입면장(수입신고필증) 수령
 │
 ▼
[6] 입고 + 수입원가정산 (영업팀 + 재무팀)
 │  입고일/위치 입력 → 재고번호 자동 부여
 │  재무팀이 수입면장 기반으로 원가정산 작성
 │  (송금액 + 관세/통관/운송비 = 수입원가)
 │
 ▼
[7] 납품/출고 (영업팀)
    고객사 납품 시 재고에서 출고 처리
```

## 3. 국내 구매 흐름 (전 사원 → 결재 → 재무팀)

```
[1] 지출결의서 작성 (전 사원)
 │  건명, 목적, 품목 내역, 증빙 첨부
 │
 ▼
[2] 부서 결재 (결재라인)
 │  부서장/대표이사 승인 (1~3단 결재)
 │
 ▼
[3] 재무팀 접수
 │  승인된 지출결의서가 재무팀 대시보드에 자동 표시
 │
 ▼
[4] 재고 대상 판정 (재무팀)
 │  재고관리 대상 여부 결정
 │  ├─ 재고 대상 → 입고 대기
 │  └─ 재고 미대상(소모품 등) → 완료
 │
 ▼
[5] 입고 + 재고번호 부여 (재고 대상인 경우)
    재고번호 자동 부여, 재고 트랜잭션 기록
```

## 4. 수입원가정산 구조

```
수입원가정산서 1건
 ├── 기본정보: 신고번호, 공급자, 신고일, 계약번호
 │
 ├── 송금 내역 (복수 건)
 │   └── 송금일, 외화금액, 환율, 원화금액, invoice번호
 │
 ├── 관세/통관 비용
 │   └── 관세, 국외운반비, 국내운반비, 통관수수료, 창고보관료, 취급수수료
 │
 ├── 부대비용 (선택)
 │   └── 특정 품목에 귀속되는 추가 비용
 │
 └── 품목 명세
     └── 모델/규격별 수량, 단가, 금액 → 재고번호 연결

수입원가 = 송금액 합계 + 관세/통관비 합계 + 부대비용
공급가액 = 수입원가 / 1.1 (VAT 역산)
```

## 5. 재고관리 흐름 (전 부서)

```
재고번호 체계: E{5자리}_{순번} (예: E01622_1)

분류: 미착품 / 상품 / 원재료 / 전기상품 / 전기원재료
상태: 재고 / 출고 / 수리중

입출고 유형:
 ├── 입고(구매): 해외 발주 입고 또는 국내 구매 입고
 ├── 이동(창고): 창고 간 이동
 ├── 출고(판매): 고객사 납품
 └── 반납: 출고 후 회수

추가비용 추적 (TCO):
 ├── 초기 원가 (수입원가정산 또는 구매가)
 ├── + A/S, 업그레이드, 부품추가, 모듈추가, 수리, 기타
 └── = 총소유비용(TCO) 자동 계산
```

## 6. 데이터 연결 구조

```
계약 (#YY-NN)
 │
 ├──→ 해외 발주 (제조사, 품목, 통화, 금액)
 │     │
 │     └──→ 발주 품목 (장비 마스터 참조)
 │
 ├──→ 수입원가정산 (신고번호, 송금, 관세)
 │     │
 │     └──→ 정산 품목 ──→ 재고 (재고번호)
 │
 └──→ 재고 상세에서 계약번호 확인 가능
       (재고 → 정산서 → 계약 체인)
```

| 사유 | 건수 | 조치 방안 |

## 8. 시스템 구현 단계별 현황

| 단계 | 내용 | 상태 | 주요 화면 |
|:----:|------|:----:|-----------|
| Phase 1 | 장비마스터 + 계약 + 해외발주 + 트래킹 | **완료** | 발주 목록/상세, 계약 관리 |
| Phase 2 | 전자결재 프레임워크 + 파일 첨부 | **완료** | 결재 작성/승인/반려 |
| Phase 3 | 재고 시스템 + 추가비용 추적 | **완료** | 재고 목록/상세, 입출고 |
| Phase 4 | 지출결의서 (국내 구매) | **완료** | 지출결의서 → 재무팀 |
| Phase 5 | 수입원가정산 | **완료** | 원가정산 작성/조회 |
| Phase 6 | 문서 발행 (PDF/엑셀) | 예정 | 발주서, 정산서, 결재문서 출력 |
| Phase 7 | 재고 실사 | **완료** | 실사 계획/현장 입력 |
| Phase 8 | 근태 결재 마이그레이션 | 예정 | 휴가/OT → 결재 프레임워크 통합 |
| Phase 9 | 고도화 | 예정 | 통장잔고, TCO분석, 일괄서명 |

## 9. 향후 작업 계획

### 즉시 필요 (데이터 보강)

1. **과거 연도 계약 파일 수집 및 등록**
   - 각 부서에서 2022~2024년 계약 리스트 제공 요청
   - 등록 완료 시 정산서-계약 자동 매칭 재실행
   - 목표: 연결률 67% → 90% 이상

2. **발주-정산 연결** (order_id 매핑)
   - invoice 번호 기반 자동 매칭

### 기능 구현 예정

3. **문서 발행** (Phase 6)
   - 발주서, 원가정산서, 거래명세서 PDF/엑셀 출력
   - 결재 문서 PDF (결재란 포함)

4. **근태 결재 통합** (Phase 8)
   - 휴가/OT 신청을 전자결재 프레임워크로 통합

5. **고도화** (Phase 9)
   - 통장잔고 관리, 추가필요금액 자동 계산
   - TCO 분석 보고서
   - 다건 일괄 결재
