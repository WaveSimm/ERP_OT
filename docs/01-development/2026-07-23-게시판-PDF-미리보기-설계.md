## 1. 현상 분석 (원인 규명 완료)

- 게시판은 PDF를 본문에 임베드하는 기능이 **원래 없음**.
  - 업로더(`apps/web/src/components/board/AttachmentUploader.tsx:44`): `isInline = file.type.startsWith("image/")` — **이미지만** 본문 마크다운 삽입, PDF는 항상 첨부 목록.
  - 렌더러(`PostMarkdownView.tsx`): ReactMarkdown + `rehypeSanitize` — iframe/embed 태그는 어차피 제거됨.
  - DB 확인: PDF 첨부 8건 전부 `is_inline=false`, 본문 참조 0건.
- "본문에 PDF가 보이는 글"의 정체 = **작성자가 PDF 페이지를 캡처(이미지)해서 본문에 붙여넣은 것** (예: "차세대 사내업무 시스템 안내" 글 = 인라인 이미지 4장 + PDF 원본 첨부).
- 규정류 5건(취업규칙·복리후생·여비규정 등)은 본문 7~26자(사실상 빈 글) + PDF 첨부만 → 다운로드해야만 내용 확인 가능.

## 2. 확정 결정사항

| # | 결정 | 비고 |
|---|---|---|
| 1 | **PDF 첨부는 게시글 상세에서 iframe 인라인 미리보기** (브라우저 내장 뷰어) | 데스크톱 한정. 수십~수백 페이지도 뷰어가 보이는 페이지만 렌더링해서 문제없음 |
| 2 | **모바일은 iframe 대신 "PDF 열기/다운로드" 카드로 분기** | 안드로이드 크롬은 iframe PDF 미렌더(빈 박스/다운로드로 빠짐), iOS 사파리는 첫 페이지만 표시 — 2026년 현재도 동일. 빈 박스 노출 방지가 목적 |
| 3 | **페이지→이미지 변환(pdftoppm)은 사용하지 않음** | 수십 페이지 문서에서 10~25MB 이미지 폭탄. 기각 |
| 4 | **WYSIWYG 에디터 도입하지 않음** | 마크다운 유지. "내용과 표현의 분리" 원칙 — 전자결재도 폼 기반+고정 서식으로 갈 예정이라 에디터 불필요. 게시판은 렌더링 품질에 투자 |
| 5 | **PDF 텍스트 추출은 화면 표시용이 아니라 검색 인덱스용** | 별도 과제. 텍스트 추출은 사진·표·레이아웃이 소실되므로 "원본처럼 보여주기"에는 부적합 |
| 6 | (후순위) 모바일에서 "폰에서 바로 보고 싶다" 피드백이 실제로 나오면 self-hosted pdf.js 뷰어를 해당 화면 lazy load로 검토 | 선제 구현 금지 |

## 3. 구현 가이드

### 백엔드: 수정 불필요 ✅

`services/auth/src/api/routes/attachment.routes.ts:65` — 이미 PDF를 inline으로 서빙 중:

```ts
const INLINE_SAFE = new Set(["image/jpeg", "image/png", "image/gif", "image/webp", "application/pdf"]);
const disposition = INLINE_SAFE.has(mimeType) ? "inline" : "attachment";
```

인증은 쿠키 기반이라 iframe 同一출처 GET에 그대로 실림(현재 `<a href>` 다운로드가 동작하는 것과 동일 경로).

### 프론트: `apps/web/src/components/board/PostAttachmentList.tsx` 한 파일

- `visible.filter(a => a.mimeType === "application/pdf")` 로 PDF만 골라 첨부 목록 위에 미리보기 블록 렌더.
- `<details open>` + `<summary>`(파일명 + "새 탭에서 크게 보기" 링크) + `<iframe src={attachmentApi.downloadUrl(a.id)} className="w-full h-[80vh]">` — 접기 가능, JS 추가 0.
- **모바일 분기**: 화면 폭 기준(예: `useMediaQuery` 또는 Tailwind `hidden md:block`)으로 iframe은 md 이상에서만, 미만에서는 기존 다운로드 행만 노출. UA 스니핑 불필요.
- 기존 첨부 목록(다운로드 행)은 그대로 유지 — PDF도 목록에 남김(다운로드 용도).
- 사용처는 게시글 상세 1곳: `apps/web/src/app/board/[catCode]/[boardCode]/[postId]/page.tsx:180`.
