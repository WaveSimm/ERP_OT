# 모바일 전용 뷰 (현장 작업일지) — 설계 문서

## 배경 / 목적
현장에서 휴대폰으로 접속했을 때, PC용 전체 화면 대신 **꼭 필요한 기능만** 담은 경량 화면을 제공한다.
핵심 용도: 현장 작업자가 자신에게 배정된 태스크를 열어 **작업일지**를 작성/수정하는 것.

## 범위 (확정)
- 모바일 상단 메뉴는 **글자 메뉴 3개**만: `근태` · `칸반` · `작업 목록`
  - (PC용 전체 네비게이션·프로젝트·전사대시보드·경비 등은 모바일에서 노출하지 않음)
- '프로젝트' 별도 메뉴 없음 — `작업 목록`이 이미 프로젝트별로 묶여 표시되므로 불필요.
- `작업 목록`(MyTasksView)은 모바일에서 컬럼을 **태스크명만** 남기고 나머지 6개
  (`상태`·`시작일`·`종료일`·`내 투입`·`분담`·`내 진척률`)를 숨겨 짤림 없이 표시.
- 태스크를 탭하면 기존 `TaskDrawer` 상세가 열려 **모든 정보 표시 + 작업일지 작성**(스크롤).

## 진입 방식
- `middleware.ts`에서 **User-Agent가 모바일**인 문서(GET, text/html) 요청을 `/m` 으로 리다이렉트.
- 탈출구: `viewMode=desktop` 쿠키가 있으면 리다이렉트 우회(모바일 화면의 "PC" 버튼이 세팅).
- `/m`, `/login`, `/api`, 정적파일(_next·확장자)은 리다이렉트 제외.

## 재사용 전략 (기존 데스크톱 코드 최소 변경)
- `me/dashboard/page.tsx`의 `DashboardPage`에 `mobile?: boolean` prop 추가.
  - `mobile`이면 탭을 `근태·칸반·작업목록`으로 필터, 숨은 탭 선택 시 칸반으로 보정, 헤더/패딩 축소.
  - `<MyTasksView mobile />` 로 컬럼 축소 모드 전달.
- `MyTasksView`에 `mobile?: boolean` prop 추가 — 그리드 `1fr` + 6개 컬럼 셀 미렌더.
- `TaskDrawer` 루트 `w-[520px]` → `w-full sm:w-[520px]` (폰 전체폭, 데스크톱 무변화).
- 신규: `app/m/layout.tsx`(모바일 크롬), `app/m/page.tsx`(`<DashboardPage mobile />`).

## 영향 / 안전성
- 데스크톱(≥sm) 동작 불변 — 모든 변경은 `mobile` 플래그·`sm:` 브레이크포인트로 게이팅.
- 운영(3000)은 pull 후 docker 재빌드 시에만 반영되므로, 각 커밋은 빌드 가능한 상태를 유지.
