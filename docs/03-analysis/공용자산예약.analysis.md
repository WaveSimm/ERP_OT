## 4. Gap List

### 🔴 Major (1)

#### G1. PATCH scope=instance for recurring series — 1차 보류
- **위치**: `services/project/src/application/equipment-reservation.service.ts:254-260`
- **현재 동작**: 반복 시리즈에서 `PATCH /:id?scope=instance` 호출 시 400 `UNSUPPORTED_PARTIAL_UPDATE` ("1차에서 지원하지 않습니다")
- **Design §4.4**: "별도 row + recurrenceParentId" 패턴 명세
- **완화책**:
  - 에러 메시지에 워크어라운드 명시 (`cancel(scope=instance)` + 새 단발 등록)
  - Modal에서 반복 시리즈 단일 인스턴스 편집 차단 (`ReservationModal.tsx:354-359`)
  - **cancel-instance**는 정상 구현 (service:330-360)
- **영향**: 부분 수정 UX 불편, 기능적 회피 가능
- **권고**: §13 미해결 이슈 #8 ("RRULE EXDATE/THISANDFUTURE")로 이미 2차 등재됨 → 의도된 보류, **Report 전 추가 작업 불필요**

### 🟡 Minor (5)

#### G2. SQL conflict-detection 패턴 차이
- **위치**: `equipment-reservation.service.ts:384-432`
- **Design §4.3 / §11.2 Do 1 #5**: `findConflicts` SQL `WHERE NOT (endAt <= $start OR startAt >= $end)`
- **실제**: 자원의 CONFIRMED 행 전부 로드 후 메모리 `overlaps()`로 비교 (recurrence-expander.ts:156-158, 엄격 `<`로 인접 시간대 충돌 아님 정상 처리)
- **service.ts:380 주석**: "평균 ~수십 건 수준이라 충분"
- **영향**: 자원당 100+ 예약 누적 시 잠재적 성능 저하; 인덱스 `(resourceId, startAt)` 활용 안 됨
- **권고**: 현 규모에서는 acceptable. 부하 증가 시 SQL 최적화 (2차)

#### G3. PATCH/DELETE 기본 scope inverted
- **위치**: `equipment-reservation.routes.ts:64, 80` — `scope ?? "series"`
- **Design §4.4**: 기본값 `instance`
- **완화**: FE가 항상 명시적으로 scope 전달 (`api.ts:423` update default `"series"`, cancel은 호출 시점 명시)
- **영향**: 외부 API 사용자가 scope 누락 시 design과 다른 동작
- **권고**: 5분 작업 — route default를 `instance`로 변경 OR design을 `series`로 정정

#### G4. Prisma `@map` directive 누락
- **위치**: `services/project/prisma/schema.prisma:389-409` (모든 필드)
- **migration.sql 결과**: `"resourceId"`, `"userId"`, `"startAt"` (camelCase 컬럼)
- **Design §3.1**: `@map("resource_id")` 등 명세
- **CLAUDE.md 규칙**: "DB tables: snake_case"
- **참고**: 프로젝트 기존 스키마도 동일 패턴 (camelCase columns) — 프로젝트 전반 deviation, 본 기능 단독 이슈 아님
- **권고**: 변경 안 함 (전사 리팩토링 필요)

#### G5. Day-view 휴일/주말 시각 표시 누락
- **위치**: `apps/web/src/components/equipment-reservation/ReservationDayView.tsx`
- **Week view**: `bg-red-50/40` (휴일) + `bg-gray-50/50` (주말) 적용 (Calendar.tsx:131)
- **Day view**: 휴일/주말 배경 무 — `useHolidaysMap`은 Container에서 호출되나 DayView에 holidayName prop 미전달
- **영향**: Cosmetic, week 뷰로 전환 가능
- **권고**: 15분 작업 — Container → DayView로 holidayName 전파 + week-view 클래스 미러링

#### G6. localToIso 브라우저 로컬 타임존 의존
- **위치**: `apps/web/src/components/equipment-reservation/ReservationModal.tsx:65-71`
- **현재**: `new Date(y, mo-1, d, h, m).toISOString()` — KST 머신에서만 정확한 UTC 산출
- **Design §1.1**: "UTC 저장, KST 표시"
- **영향**: 다른 timezone 디바이스에서 시간 drift
- **권고**: 로컬 전용 운영 단계(CLAUDE.md)에서는 acceptable; 외부 노출 시 명시적 KST 변환 추가

### G7. 테스트 파일 0건
- **검색**: `services/project/src/**/*reservation*.test.ts`, `*reservation*.spec.ts`, `recurrence*.spec.ts` — **0 results**
- **Design §9**: Unit (recurrence-expander), Integration (CRUD/permissions/conflict/series), Manual (UI) — 18 케이스
- **§11.1**: recurrence-expander spec "Phase 9 권장"
- **영향**: 회귀 위험. 권한/충돌/반복취소 시나리오 검증 0건
- **권고**: 최소 `recurrence-expander.spec.ts` 추가 (DAILY/WEEKLY byWeekday/MONTHLY/until/count, overlaps adjacent vs partial) — ~1h, +5pt
