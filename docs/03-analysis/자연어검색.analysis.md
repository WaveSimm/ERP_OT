## Gaps

### Yellow / Minor

**G-1. 백필 스크립트 형태 차이**
- Design §11.1: TS 2개 (`backfill-embeddings-auth.ts`, `backfill-embeddings-project.ts`)
- 실제: 단일 `scripts/backfill-embeddings.mjs` (`posts|worklogs|all` arg, docker exec psql)
- Severity: Low — 기능적으로 동일, 더 단순함. **Design v1.1에서 패턴 변경 반영 권장**.

**G-2. EMBEDDING_TIMEOUT_MS 불일치**
- Design §13: `EMBEDDING_TIMEOUT_MS=5000`
- `docker-compose.yml`: `EMBEDDING_TIMEOUT_MS: "30000"` (Cold-start 대응)
- Severity: Medium. **Design v1.1에서 30000으로 갱신**하거나 5000 → 30000 변경 사유 명시.

**G-3. MAX_INPUT_CHARS 불일치**
- 서비스 코드 (`embedding.service.ts`): 5000
- 백필 스크립트 (`backfill-embeddings.mjs`): 4000
- Severity: Low — 4000/5000 차이는 1024-d bge-m3 품질에 무시할 영향. **5000으로 통일 권장**.

**G-4. Zod 검증 위치**
- Design: `q.min(2)` 단일 위치 검증
- 실제: Zod는 `min(1)`, 서비스 레이어에서 `q.trim().length < 2` 재검증 (동일 400 + `QUERY_TOO_SHORT`)
- Severity: Trivial. 동작 동일.

### Green / Bonus

**B-1. WorkLog 권한 scope 사전 필터링 (Design 대비 강화)**
- Design §4.3: 검색 후 결과 필터
- 실제: OPERATOR의 경우 SQL `WHERE`에 `projectIds = ANY(...)` 사전 적용 → 정확도+성능 모두 개선
- Severity: Improvement — Design §4.3에 "OPERATOR 사전 필터" 명시 권장.

**B-2. ADMIN/MANAGER vs OPERATOR scope 분리**
- Design: 단일 필터
- 실제: ADMIN/MANAGER는 전체, OPERATOR는 본인 참여 + 본인 작성 프로젝트만
- Severity: Improvement.

**B-3. Partial failure 처리**
- `Promise.allSettled` 사용 → project-service down 시 게시판만 부분 결과 반환
- Severity: 설계 의도 충족 (강건함).
