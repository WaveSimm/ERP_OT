## Gaps

### Yellow / Documentation Drift

**Y1. `SEARCH_KEYWORD_MATCH_BONUS` 8번째 환경변수 추가 (Design 미반영)**
- Design ref: §5.1, §10.3 — 7개로 명시
- 실제: 8개 (`SEARCH_KEYWORD_MATCH_BONUS=0.20` 추가)
- 사유: bge-m3 점수 분포 좁음 + 작은 데이터셋(7+5건)에서 keyword × 0.4 가산만으로는 정확 매칭 글이 cutoff 통과 어려움. ILIKE 매칭 시 절대 보너스 직접 가산.
- 영향: Plan 의도("정확 매칭 우선 노출") 충족, 가중치 합 검증·권한 필터 영향 없음
- **Severity**: Yellow (구현이 Design을 확장)

**Y2. `SEARCH_MIN_SCORE` 기본값 0.45 → 0.35**
- Design ref: §5.1, §4.4, §5.3 — `0.45`
- 실제 코드 default: `0.40`, docker-compose: `0.35`
- 사유: bge-m3 코사인 점수가 0.4~0.6에 모임 → 이론치 0.78 도달 불가
- 부수 이슈: 코드 default(0.40)와 운영 default(0.35) 불일치
- **Severity**: Yellow (env 파라미터로 운영 조정 가능)

**Y3. 점수 합산식 Design 미반영**
- Design 식: `embed × W_e + max(exact, trgm) × W_k`
- 실제 식: `embed × W_e + max(exact, trgm) × W_k + (exact > 0 ? KEYWORD_MATCH_BONUS : 0)`
- **Severity**: Yellow — Y1과 같은 origin
