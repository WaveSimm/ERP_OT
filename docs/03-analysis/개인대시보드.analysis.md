## 설계 초과 구현 (Bonus)

| 항목 | 설명 |
|------|------|
| PENDING_2ND / PENDING_3RD 상태 | 2/3차 결재 흐름 (설계는 1차까지) |
| canTransition() 유틸 함수 | FSM 유효성 사전 확인용 |
| 잔여 연차 초과 시 즉시 예외 | 서버 레벨 검증 (설계는 경고만) |
| Redis 채널 user별 분리 | `notification:${userId}` (설계는 공통 채널) |
| MyTasksView 드래그 정렬 | 프로젝트 순서 localStorage 유지 |
| MyTasksView 인라인 편집 | 시작일/종료일/진행률/배정률 직접 수정 |
| AttendanceSummaryBar 대시보드 통합 | 설계는 컴포넌트 분리였으나 인라인 구현 |

## 미구현 / 갭 항목

| 갭 | 심각도 | 설명 |
|----|--------|------|
| GET /me/team-members/:userId/kanban | Low | Manager 전용 팀원 칸반 API. team 페이지에서 사용 예정이나 API 미구현 |
| TeamMemberKanban UI | Low | 위 API 없으므로 UI도 미구현. team 페이지 탭 추가 필요 |
| MyProjectList 독립 UI | Low | /me/projects API는 구현됨. 대시보드에 "참여 프로젝트" 탭/카드 없음 |
