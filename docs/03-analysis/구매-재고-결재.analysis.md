## 7. 결론

- **Match Rate 88%** — Report 가능 기준(90%) 직전. Medium Gap 중 M1·M3 두 건은 frontend 작업 1~2시간으로 해소 가능 → 90%+ 도달 현실적
- **핵심 도메인은 견고**: SKU 3계층, InboundRequest 통합 큐, InventoryItemLocation 분산, BomItem/BundleShipment, sibling-asset 추적까지 모두 schema·service·routes·UI에 일관 적용
- **사용자 결정 필요 항목**:
  1. **M5 (Bundle B안 문서화)** — Design §19.1.6를 ProductMaster(itemType=BUNDLE)+BomItem(parentMasterId) 기준으로 재작성할지 여부
  2. **M4 (표준가 제거 강도)** — `referencePrice` 컬럼을 deprecated로 둘지, 완전 제거할지
  3. **L2 (마이그 apply 시점)** — 변경 영향 큰 작업이라 별도 검토안+승인 흐름 필요
- **잔여 액션 (우선순위)**:
  1. **즉시 (Match Rate +2~3%)**: M1 토글 UI 추가, M3 안내 문구 정리
  2. **단기 (Match Rate +1~2%)**: M2 발주 가용 산입 또는 백로그 처리, M5 Design 정정
  3. **중기**: L2 마이그 apply (사용자 명시 승인 후), L4/L5 legacy cleanup
