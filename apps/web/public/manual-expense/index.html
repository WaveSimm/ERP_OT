1. 개요

경비정산은 직원이 회사 일로 사용한 비용 중 개인카드·현금으로 결제한 금액을 환급받는 워크플로우입니다.

⚠ 법인카드는 경비정산 대상이 아닙니다. 법인카드는 회사가 직접 결제하므로 환급(송금)이 필요 없습니다.
법인카드 사용분은 지출결의서 양식으로 직접 작성하세요 (§11 참고).
경비정산 시스템은 개인 비용(개인카드·현금)을 회사가 환급하기 위한 도구입니다.
  • 입력: 개인카드 명세서 파일(Excel/HTML) import, 수동 거래 입력(현금 결제), 영수증 사진 업로드
  • 분류: 사업(계약)·구분·상세내역 입력 (카테고리 도메인은 v1.4부터 폐기 — 사업/구분으로 통합)
  • 매칭: 영수증과 거래를 묶어 증빙 확보 (수동 매칭)
  • 정산: 거래를 정산묶음으로 모아서 결재 상신 (정산 묶음 제목은 사용자가 직접 입력)
  • 결재: EXPENSE(지출결의서) 양식 사용, 결제수단 "개인정산"으로 표시 — EXPENSE_CLAIM 별도 양식은 폐기되어 모든 결재가 EXPENSE 단일 양식
  • 재무 처리: 결재 완료 즉시 자동으로 회계 > 재무 접수 모듈에 등록 → 재무팀이 본인 계좌로 송금 → 정산완료

2. 메뉴 구조

경비정산은 내 대시보드 > 경비 탭에서 모든 기능 사용 가능합니다.

접근 경로
내 대시보드 → 상단 탭 "경비" 클릭
→ 요약 카드들 (정산대상 / 정산완료 / 제외 / 전체 등)
→ 빠른 작업 버튼: 정산내역관리, 정산결재, 카드 관리, 매뉴얼
→ 최근 정산 (아무 빠른 작업도 선택 안 했을 때만 표시)
빠른 작업역할
📋 정산내역관리거래 목록 — 사업(계약)·구분·상세내역 분류, 정산분류 할당, 영수증 업로드·매칭·분할 모두 여기서 처리
📦 정산결재정산묶음 목록 + 상세 + 결재 상신·취소
💳 카드 관리본인 개인카드 등록 (명세서 import 시 사용. 법인카드 X)
📘 매뉴얼이 문서 (새 탭)
⚠ v1.4 변경: 카테고리 도메인 폐기. 별도의 "🏷 카테고리" 관리 페이지가 없어졌고, 카테고리 셀렉트 컬럼 대신 사업(계약) · 구분 · 상세내역 3개 컬럼으로 분류합니다. 기존 PERSONAL 카테고리에 해당하던 거래는 정산분류 셀렉트의 "제외" 옵션으로 처리하세요.
ℹ 영수증 관련 모든 작업은 정산내역관리 페이지 안에서 처리됩니다 (별도 페이지 없음).

3. 업무 전체 흐름

  1. 거래 등록: 카드 명세서 파일을 import하거나 현금 거래를 수동 입력
  2. 사업·구분·상세내역 입력: 거래마다 사업(계약)·구분·상세내역을 분류 (카테고리 도메인 없음)
  3. 영수증 업로드 + 매칭: 정산내역관리 상단 "+ 영수증 업로드"로 다중 업로드 또는 거래 행의 + 버튼으로 단일 업로드. 모달에서 거래에 직접 매칭
  4. 정산분류 할당: 같은 정산 묶음에 포함될 거래들을 정산분류 셀렉트로 묶음 ("+ 새 정산"으로 새 묶음 생성 시 제목은 사용자가 직접 입력). 정산 대상이 아닌 거래는 "제외" 옵션 선택
  5. 결재 상신: 정산결재에서 정산묶음 선택 → 프로젝트(옵션)·본문(옵션)·결재선 입력 → "결재 상신"
  6. 결재 진행: 결재중 → 결재완료 (결재자 승인 시 자동으로 회계 > 재무 접수에 등록됨)
  7. 재무 처리: 재무팀이 회계 > 재무 접수에서 송금 처리 → 정산완료(입금)

4. 거래 등록

4-1. 개인카드 명세서 import (자동)

📋 정산내역관리 → 우측 상단 "📥 카드 명세서 import"
  1. 카드사 사이트에서 개인카드 명세서 Excel(.xls/.xlsx) 또는 HTML 파일 다운로드
  2. 파일 선택 → 자동 파싱(신한카드/국민카드/현대카드 등) → 거래들이 미내역분류 상태로 추가
  3. 같은 거래가 중복 등록되지 않도록 승인번호 기준 자동 skip

지원 카드사: 신한카드, 국민카드, 현대카드 (다른 카드사는 수동 입력 또는 영수증 업로드 사용)

⚠ 법인카드 명세서는 올리지 마세요. 법인카드 사용분이 섞이면 회사가 이미 결제한 금액을 다시 환급하는 오류가 발생합니다.
법인카드는 전자결재 → 새 결재 → 지출결의서 양식으로 별도 처리하세요.
명세서에 사적·법인카드 거래가 섞여 들어왔다면 해당 거래는 "제외" 상태로 변경하세요.

4-2. 수동 입력 (현금)

📋 정산내역관리 → 우측 상단 "+ 수동 입력"

거래일·가맹점·금액·결제수단을 직접 입력해 거래 생성. 명세서에 없는 현금 결제, 외부 결제 등에 사용. 결제수단 드롭다운에서 "현금" 선택.

4-3. 영수증 모달에서 자동 생성

영수증 업로드 시 모달에서 "➕ 내역 추가" 클릭하면 영수증 정보(가맹점/금액/날짜)로 거래 자동 생성 + 매칭. 카드 명세서에 없는 영수증을 빠르게 정리 가능.

5. 영수증 업로드 + 매칭 + 분할

5-1. 업로드 방법 2가지

다중 업로드: 📋 정산내역관리 → 우측 상단 "+ 영수증 업로드" → 파일 다중 선택 → 모달이 순차 큐로 열림
단일 업로드: 거래 행의 영수증 컬럼 + 버튼 → 해당 거래에 바로 매칭됨
  1. 이미지(JPG/PNG/WebP) 또는 PDF 업로드
  2. OCR 자동 시작 (CLOVA OCR) — 보통 1~3초
  3. OCR 완료 후 가맹점·금액·날짜 자동 추출
  4. 다중 업로드 시 영수증 모달이 자동 큐로 순차 열림

5-2. 영수증 모달에서 매칭 (수동)

모달 좌측: 영수증 이미지(줌 가능). 우측: 추출 정보 + 거래 매칭.

  • 추출 정보 수정: OCR 잘못 인식한 경우 직접 수정
  • 결제수단 자동 추정: 영수증 텍스트에서 카드번호 끝 4자리·카드사명·"현금" 인식 (등록된 본인 개인카드 매칭)
  • 매칭할 거래 선택: 미매칭 거래 목록에서 직접 고름 (자동 매칭 후보 X — 사용자 책임으로 정확 매칭)
  • "이 거래에 매칭" 클릭 → 즉시 확정. 영수증의 추출 정보가 거래 정보로 동기화
ℹ 자동 매칭 후보는 비활성화 상태입니다. 모든 매칭은 사용자가 명시적으로 선택.

5-3. 두 버튼 — "💾 저장" vs "➕ 내역 추가"

버튼언제 쓰나
💾 저장OCR 추출 정보만 수정 (영수증 메타데이터 업데이트). 거래 생성·매칭 X
➕ 내역 추가카드 명세서에 없는 영수증 — 결제수단 선택 + 정보 입력 → 거래 자동 생성 + 매칭

5-4. 영수증 분할

모달 하단 "분할" 버튼: 영수증 1장에 여러 거래가 함께 있을 때(여러 영수증 사진을 합쳐 찍었을 때 등) N개 영역으로 분할해 각각 매칭 가능.

  • 각 영역을 드래그로 선택 → 분할 실행
  • 분할된 영수증마다 별도 OCR + 매칭 모달 자동 큐 처리
  • 원본 영수증은 분할 완료 시 자동 삭제 (필요하면 본인 폴더에서 다시 업로드)

6. 정산내역관리

거래 목록 페이지. 모든 거래의 카테고리·상세내역·정산묶음 할당을 관리. 영수증 업로드·매칭·분할도 모두 이 페이지에서.

6-1. 행별 인라인 편집

컬럼편집
거래일시 / 가맹점 / 결제수단 / 금액거래 자체 정보 (명세서 import 또는 수동 입력 시 결정)
사업(계약)검색 드롭다운 — 진행 중인 계약 목록에서 선택. 미설정 가능
구분자유 텍스트 — 식대/교통비/접대비 등 기존 카테고리 역할 (v1.4부터 사업 안 분류로 사용)
상세내역자유 텍스트 — 결재 문서·정산묶음에 노출되는 상세 설명
영수증📎 보기 / + 업로드 (없음)
정산분류드롭다운 4옵션 — 미설정 / 기존 DRAFT 정산묶음 / + 새 정산 (직접 제목 입력) / 제외
상태PENDING (미분류) / CATEGORIZED (분류완료) / EXCLUDED (제외) / SETTLED (정산묶음 포함)

6-2. 일괄 작업 (상단 바)

체크박스로 거래 1건 이상 선택 → 상단 일괄 작업바에서 사업(계약)·구분·상세내역·정산분류 한 번에 적용

Tip: Shift+클릭으로 범위 선택. 정산완료/전체 탭에서는 진행 단계 범례(결재중 → 결재완료 → 정산완료)가 우측에 표시됩니다.

6-3. 정산분류 "+ 새 정산"

여러 거래를 한 정산묶음으로 묶을 때:

  1. 거래들을 체크 → 정산분류 셀렉트에서 "+ 새 정산" 선택 → 일괄 적용
  2. 제목 입력 prompt가 뜸 — 사용자가 직접 정산묶음 제목 입력 (예: "2026-05 식대 정산", "차세대로거 출장비"). 자동 제목 생성은 v1.6.3 (5/16) 부터 폐기됨
  3. 선택한 거래 모두 새 정산묶음에 할당됨. 제목은 정산결재 상세 화면에서 ✏️ 클릭해 수동 수정 가능 (DRAFT/REJECTED/SUBMITTED 단계까지)

6-4. 정산분류 "제외"

회사 환급 대상이 아닌 거래(사적 결제 등)를 "제외"로 설정:

  • 정산분류 셀렉트에서 "제외" 선택
  • 거래 상태가 EXCLUDED로 변경 + 정산묶음 할당 자동 해제
  • 이전 PERSONAL 카테고리 자동 처리를 대체 (명시적 선택 방식)

6-5. 선택 삭제

일괄 작업바의 "선택 삭제": 선택한 거래 영구 삭제 (매칭·정산 할당도 해제). 카드 명세서 재import 시 다시 복원됨.

7. 정산결재

7-1. 정산묶음 목록

📦 정산결재 → 정산묶음 행 클릭 → 상세 화면 (인라인)

7-2. 정산묶음 상세에서 결재 상신 정보 입력

DRAFT/REJECTED 상태에서 결재 상신 직전에 다음 정보 입력 (모두 옵션):

  • 프로젝트: 검색 드롭다운 + 직접 입력 가능 — 결재 문서의 "프로젝트" 필드
  • 본문: 결재 문서에 함께 들어갈 자유 텍스트 (지출 사유 등)
  • 결재선: 부서·이름 셀렉트로 결재자 추가 (역할: 결재/합의). "부서 기본선 불러오기" 버튼으로 본인의 기본 결재라인 한번에 로드
ℹ 결재선을 비우면 부서 기본 결재라인이 자동 적용됩니다.

7-3. 결재 상신 — 바로 결재 진행

  • 📤 결재 상신: DRAFT 또는 REJECTED에서 클릭
  • 지출결의서 결재 문서 자동 생성 + 임시저장 거치지 않고 바로 결재 대기 상태로 진행
  • 거래 정보가 자동 변환되어 결재 문서 항목에 등재됨 (가맹점+카테고리+메모 → "내역", 결제수단 "개인정산")
  • 매칭된 영수증의 파일명은 "증빙" 컬럼에 표시 + 결재 문서 첨부로 자동 업로드
  • 영수증이 없는 거래가 있으면 상신 차단됩니다 (가맹점 N건 외 N건 메시지 표시)

7-4. 결재 취소·삭제

  • ↩ 결재 취소: 결재중(SUBMITTED) 상태에서 가능. 결재 문서 회수 + 정산묶음 DRAFT 복귀
  • 삭제: DRAFT 또는 REJECTED 상태에서만 가능
  • 📥 Excel 다운로드: 정산묶음 전체를 엑셀로 받기
  • 📄 결재 문서 보기: 결재 시스템 화면으로 이동

7-5. 결재 양식 — 지출결의서 통합

ℹ 경비정산은 지출결의서 양식으로 결재됩니다.
• 결제수단: "개인정산"으로 자동 표시 (일반 지출결의서는 "법인카드" 기본)
• 거래 항목별로 영수증 다운로드 링크가 결재 문서에 포함됨

8. 재무 처리 (재무팀 전용)

결재가 완료되면 자동으로 회계 메뉴 > 재무 접수 페이지에 등록됩니다. 재무팀이 송금 처리하면 기안자 화면에 "정산완료"로 자동 반영.

접근 경로: 상단 메뉴 회계 > 재무 접수 (재무팀·관리자만 권한)

8-1. 재무 접수 화면

  • 결재 승인된 모든 지출 건이 자동 등록 (지출결의서 + 개인정산 모두)
  • 종류 배지 표시: 업체 송금(일반 지출결의서) / 개인경비 송금(경비정산)
  • 송금 컬럼: 송금 처리 완료 시 ✓ 날짜, 미처리 시 "미처리"
  • 행 클릭 → 상세 모달 (좌: 정보/액션, 우: 영수증 미리보기 패널)

8-2. 송금 처리

  1. 상세 모달의 송금 처리 섹션에서 송금일·송금 금액·메모/사유 입력
  2. "송금 완료 처리" 클릭 → 자동으로 settlement.status = PAID 동기화
  3. 기안자의 정산내역관리 화면에서 즉시 정산완료로 표시
  4. 실수했을 경우 "해제" 버튼으로 송금 처리 취소 가능 (settlement → 결재완료로 롤백)

8-3. 영수증 미리보기 (우측 패널)

  • 모달 열 때 첫 번째 영수증 자동 미리보기
  • 지출 내역 테이블의 📎 클릭 → 우측 패널에서 즉시 표시 (새 창 X)
  • 이미지는 패널 크기에 맞춰 자동 축소, PDF는 iframe으로 표시
  • 필요 시 우측 상단 "새 창" 링크로 별도 탭 열기 가능

8-4. (재고 대상) 입고 처리

구매성 지출결의서에는 추가 단계: 재고 판정 → 입고 처리 (입고일·위치·메모 입력). 개인경비 정산은 이 단계 없음.

9. 상태 필터·진행 단계

10. 자주 묻는 사용법

Q1. 카드 명세서 import 했는데 같은 거래가 또 들어와요

승인번호(approvalNo)가 있는 거래는 자동 중복 skip. 승인번호가 없는 거래(현금영수증 등)는 NULL이라 중복 체크 못함. 필요 시 수동 삭제.

Q2. 영수증 OCR이 가맹점/금액을 잘못 인식했어요

영수증 모달의 추출 정보 영역에서 직접 수정 후 "💾 저장". 또는 거래에 매칭하면 거래 정보로 자동 동기화됩니다.

Q3. 한 영수증에 여러 거래가 함께 있어요 (분할 결제)

영수증 모달 하단 "분할" 버튼으로 영역을 N개로 나눠 각각 매칭하세요. 분할 완료 시 원본은 자동 삭제됩니다.

Q4. 영수증 매칭이 후보로 안 떠요

자동 매칭 후보 기능은 비활성화되어 있습니다. 모달에서 거래 목록을 직접 보고 선택해 매칭하세요. 잘못된 자동 매칭이 결재에 섞이는 것을 방지하기 위함입니다.

Q5. 정산묶음에 거래를 추가/제거하고 싶어요

정산내역관리에서 거래의 정산분류 셀렉트를 변경. 다른 정산묶음 선택 또는 "미설정"으로 해제 가능. 단 결재 상신(SUBMITTED) 후에는 변경 불가 — 결재 취소 먼저.

Q6. 결재 반려됐어요

정산결재 상세 → 정산묶음이 REJECTED 상태로 보임. 필요 정보 보강 후 다시 결재 상신, 또는 삭제 후 다른 묶음으로 재구성.

Q7. 사적인 결제가 개인카드 명세서에 같이 들어왔어요

해당 거래의 정산분류 셀렉트에서 "제외"를 선택하세요. 거래 상태가 EXCLUDED로 변경되고 환급 대상에서 빠집니다. v1.4부터 명시적 옵션 (이전 PERSONAL 카테고리 자동 처리를 대체).

Q7-2. 법인카드 사용분은 어떻게 처리하나요?

법인카드는 경비정산이 아닌 지출결의서로 처리합니다. 법인카드 결제는 회사가 카드사에 직접 지급하므로 환급할 금액이 없습니다. 단계별 작성 방법은 §11. 지출결의서 작성을 참고하세요.

Q8. 영수증을 분실했어요

매칭된 영수증이 없는 거래가 있으면 결재 상신이 차단됩니다 ("영수증이 매칭되지 않은 거래가 N건..." 에러). 영수증 재확보 또는 해당 거래를 정산묶음에서 제외해야 상신 가능.

Q9. 결재 상신 시 프로젝트·본문을 꼭 입력해야 하나요?

모두 옵션입니다. 결재선도 비우면 부서 기본 결재라인이 자동 적용. 단, 부서 기본 결재라인이 없으면 결재선을 직접 추가해야 합니다.

Q10. 결재 완료됐는데 입금이 안 됐어요

결재 완료 후 회계 > 재무 접수에 자동 등록되고 재무팀이 송금 처리하면 정산완료로 전환됩니다. 송금 처리 전이라면 "결재완료" 단계에 머물러 있습니다. 재무팀에 문의하세요.

Q11. 정산묶음 제목을 바꾸고 싶어요

정산결재 상세 화면에서 제목 옆 ✏️ 클릭 → 직접 수정. DRAFT/REJECTED/SUBMITTED 단계까지 가능 (APPROVED/PAID 이후는 잠김).

Q12. v1.4에서 카테고리 페이지가 사라졌어요

v1.4 (2026-05-18) 부터 카테고리 도메인을 폐기했습니다. 별도의 "🏷 카테고리" 관리 페이지·셀렉트가 없어졌고, 대신 거래 행에서 사업(계약) · 구분 · 상세내역 세 컬럼으로 분류합니다.

  • 이전에 표준 카테고리(식대·교통비 등)로 분류하던 정보 → 구분 컬럼에 자유 텍스트로 입력
  • 이전에 PERSONAL(사적) 카테고리로 자동 제외되던 거래 → 정산분류 셀렉트의 "제외"를 명시적으로 선택
  • 이전 정산묶음에 남은 카테고리 코드(legacy)는 표시만 되고 신규 작업에는 영향 없음

Q13. "+ 새 정산" 했더니 제목 입력 prompt가 떠요. 자동 생성 안 되나요?

v1.6.3 (2026-05-16) 부터 자동 제목 생성을 폐기했습니다. 카테고리·날짜 기반 자동 제목이 실제 의도와 자주 맞지 않아 사용자가 직접 입력하도록 변경되었습니다. 의미 있는 제목(예: "2026-05 식대 정산", "차세대로거 출장비") 입력을 권장합니다.

11. 지출결의서 작성 (법인카드·일반 지출용)

경비정산이 개인 비용을 회사가 환급하는 도구라면, 지출결의서는 회사가 직접 지출을 결정·집행하는 결재 양식입니다. 양식 코드는 EXPENSE이며 경비정산도 같은 양식을 사용하지만 진입 경로가 다릅니다.

ℹ 경비정산은 자동으로 결제수단 "개인정산"으로 상신되고, 지출결의서는 사용자가 직접 결제수단을 선택합니다. 어느 쪽이든 결재 완료 시 동일하게 회계 > 재무 접수에 자동 등록됩니다 (FINANCE_FORWARD).

11-5. 결재선 & 본문

  • "부서 기본선 불러오기" 버튼으로 본인 부서 기본 결재라인 한 번에 로드
  • 부서·이름·역할(결재/합의) 셀렉트로 결재자 직접 추가 가능
  • 본문: 자유 텍스트 — 지출 사유·배경·집행 시점 등 결재자가 알아야 할 정보 작성
  • 결재선을 비우면 부서 기본 결재라인이 자동 적용됩니다

11-6. 상신 → 결재 → 재무 접수

  1. "결재 상신" 클릭 → 임시저장 없이 바로 결재 대기 상태
  2. 결재자들이 순차 승인 → 마지막 결재 완료 시 APPROVED
  3. 승인 즉시 자동으로 회계 > 재무 접수에 등록됨 (postApprovalAction: FINANCE_FORWARD)
  4. 재무 접수 화면에 종류 배지 "업체 송금"으로 표시됨 (개인정산은 "개인경비 송금")
  5. 법인카드 — 송금 처리 X. 회계 메모만 (이미 카드사에 지급됨)
  6. 계좌이체·현금 — 재무팀이 송금 처리 → 완료 시 송금 컬럼에 ✓ 날짜 표시
